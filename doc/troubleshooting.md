# 🛠️ Troubleshooting Log

## 🔴 Issue: PWM 제어 신호 불량으로 인한 구동 실패
* **증상:**
    * 코드는 정상적으로 업로드되었으나 모터가 전혀 회전하지 않거나, 간헐적으로 틱틱거리는 소음만 발생.
    * 테라텀(UART)으로 디버깅 시 엔코더 값은 변하지 않고 명령은 수신됨.
    * 오실로스코프/멀티미터 측정 시 모터 드라이버 입력단(PWM)에 신호가 도달하지 않음.

* **원인 분석 (Root Cause):**
    * **배선 방식의 결함:** 기존 만능기판에 납땜된 핀 헤더와 모터 드라이버를 연결하기 위해 **점퍼선(Jumper Wire)과 스크류 터미널**을 이중으로 거치는 복잡한 배선을 사용함.
    * **접촉 불량 및 신호 감쇄:** 점퍼선이 스크류 터미널에 물리는 과정에서 접촉 저항이 증가하거나, 커넥터 내부에서 단선이 발생하여 **PWM High 신호(3.3V)가 드라이버까지 전달되지 못함.**
    * 만능기판의 노후된 패턴과 임시 배선이 섞이면서 신호 무결성(Signal Integrity)이 깨짐.

## 🟢 Solution: 배선 직결 (Direct Wiring)
불안정한 중간 연결 고리(만능기판 헤더, 스크류 터미널, 점퍼선)를 제거하고 배선을 단순화함.

1.  **배선 간소화:** STM32 핀에서 모터 드라이버의 제어 핀(PWM, DIR)으로 가는 경로를 **최단 거리로 직결**.
2.  **커넥터 제거:** 스크류 터미널을 거치지 않고, 신뢰성이 높은 커넥터를 사용하거나 직접 납땜하여 연결.
3.  **결과:** PWM 신호가 깨끗하게 전달되어 모터 4개가 부드럽게 구동됨을 확인.
