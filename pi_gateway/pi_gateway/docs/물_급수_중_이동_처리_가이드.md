# 물 급수 중 이동 처리 가이드

물 급수 과정에서 로봇이 여러 위치로 이동해야 하는데, 이 이동을 어떻게 처리하는지 설명합니다.

---

## 🎯 문제 상황

물 급수 과정은 다음과 같은 이동이 필요합니다:

1. **물그릇 위치로 이동** (현재 위치 → 물그릇 위치)
2. **물그릇 집기** (로봇팔 동작, 바퀴 잠금)
3. **화장실 위치로 이동** (물그릇 위치 → 화장실 위치)
4. **물 버리기** (로봇팔 동작, 바퀴 잠금)
5. **서스펜서 위치로 이동** (화장실 위치 → 서스펜서 위치)
6. **물 받기** (로봇팔 동작, 바퀴 잠금)
7. **물그릇 위치로 이동** (서스펜서 위치 → 물그릇 위치)
8. **물그릇 두기** (로봇팔 동작, 바퀴 잠금)

**질문**: 이 이동을 어떻게 처리해야 할까요? NAV 담당자에게 계속 좌표를 받아야 하나요?

---

## ✅ 답변: Pi Gateway는 브릿지 역할만 함

**Pi Gateway는 이동 제어를 하지 않습니다!** 이동은 **젯슨이 Nav2를 통해 처리**합니다.

### 현재 구조

```
┌─────────┐         ┌──────────┐         ┌─────────────┐         ┌─────────┐
│  젯슨   │ ──────> │ Nav2     │ ──────> │ Pi Gateway  │ ──────> │  STM32  │
│ (로직)  │         │ (경로)   │         │ (브릿지)    │         │ (하드웨어)
└─────────┘         └──────────┘         └─────────────┘         └─────────┘
     │                    │                      │                     │
     │                    │                      │                     │
     │  goal_pose 발행    │  cmd_vel_auto 발행   │  cmd_vel_out 구독   │  UART 수신
     │  (목표 위치)       │  (이동 명령)         │  (최종 명령)        │  (모터 제어)
```

### 동작 흐름

#### 1. 물 급수 시작

```
젯슨:
  - arm/water (water_action=1) 발행 → Pi Gateway → STM32
  - 바퀴 잠금 (로봇팔 동작 준비)
  - 강화학습 로봇팔이 물그릇 집기 시작
```

#### 2. 물그릇 집기 완료 후 이동

```
젯슨:
  - arm/water (water_action=0) 발행 → Pi Gateway → STM32
  - 바퀴 해제 (이동 가능)
  
  - Nav2에 목표 위치 요청:
    * goal_pose 발행 (화장실 위치)
    * 또는 직접 cmd_vel 발행 (단순 이동)
  
Nav2:
  - 경로 계획 및 이동 명령 계산
  - cmd_vel_auto 발행
  
Pi Gateway:
  - cmd_vel_auto 구독
  - CmdVelMux가 cmd_vel_out으로 선택
  - RosCmdVelBridge가 UART로 전송
  
STM32:
  - cmd_vel 수신 → 모터 제어
  - 로봇이 화장실 위치로 이동
```

#### 3. 화장실 위치 도달 후 물 버리기

```
젯슨:
  - 위치 확인 (Nav2의 current_pose 또는 navigation_status)
  - arm/water (water_action=1) 발행 → Pi Gateway → STM32
  - 바퀴 잠금
  - 강화학습 로봇팔이 물 버리기 수행
```

#### 4. 이후 단계 반복

- 물 버리기 완료 → 바퀴 해제 → 서스펜서 위치로 이동
- 물 받기 → 바퀴 해제 → 물그릇 위치로 이동
- 물그릇 두기 → 완료

## 🔑 핵심 포인트

### 1. Pi Gateway의 역할

**Pi Gateway는 이동 제어를 하지 않습니다!**

- ✅ 젯슨의 명령을 STM32로 전달 (브릿지)
- ✅ 로봇팔 동작 중 바퀴 잠금 (안전 제어)
- ❌ 이동 경로 계획 (젯슨/Nav2 담당)
- ❌ 목표 위치 결정 (젯슨 담당)

### 2. 젯슨의 역할

**젯슨이 모든 이동 로직을 담당합니다:**

- 물 급수 시작/정지 신호 발행
- Nav2에 목표 위치 요청 (goal_pose)
- 또는 직접 cmd_vel 발행 (단순 이동)
- 위치 확인 (current_pose 구독)
- 이동 완료 확인 (navigation_status 구독)

### 3. Nav2의 역할

**Nav2가 경로 계획 및 이동 명령을 제공합니다:**

- 목표 위치 수신 (goal_pose)
- 경로 계획 및 장애물 회피
- cmd_vel_auto 발행 (이동 명령)

## 📋 현재 코드 구조

### PatrolLoop (현재: 단순 패턴)

```python
class PatrolLoop(Node):
    def tick(self):
        if self.mode == "auto":
            vx, wz = self.profile.step(dt)  # 단순 패턴
            out.linear.x = float(vx)
            out.angular.z = float(wz)
            self.pub_auto.publish(out)
```

**Nav2가 붙으면:**
- Nav2가 `cmd_vel_auto`를 발행
- `PatrolLoop`의 `cmd_vel_auto`는 무시됨 (Nav2가 우선)
- `CmdVelMux`가 Nav2의 `cmd_vel_auto`를 선택

### CmdVelMux (명령 선택)

```python
class CmdVelMux(Node):
    def tick(self):
        if self.mode == "auto":
            # Nav2의 cmd_vel_auto가 있으면 사용
            # 없으면 PatrolLoop의 cmd_vel_auto 사용
            self.pub_out.publish(self.last_auto)
```

## 🎯 결론

### Pi Gateway에서 할 일

**현재 구현 그대로 사용하면 됩니다!** 추가 작업 불필요합니다.

1. ✅ 물 급수 시작 신호 전달 (`arm/water` 토픽 구독 → UART 전송)
2. ✅ 바퀴 잠금/해제 처리 (로봇팔 동작 중 안전 제어)
3. ✅ Nav2의 `cmd_vel_auto`를 그대로 전달 (브릿지 역할)

**이것만 하면 됩니다!** 위치 정보, Nav2 연동 방식 등은 신경 쓸 필요 없습니다.

### 젯슨에서 할 일 (Pi Gateway와 무관)

**젯슨이 모든 이동 로직을 처리합니다:**

1. 물 급수 시작 신호 발행 (`arm/water`, `water_action=1`)
2. 물그릇 집기 완료 후 Nav2에 목표 위치 요청 (젯슨이 알아서)
3. 이동 완료 확인 후 물 버리기 신호 발행 (젯슨이 알아서)
4. 이후 단계 반복 (젯슨이 알아서)

**Pi Gateway는 젯슨이 보내는 신호만 받아서 STM32로 전달하면 됩니다!**

---

## ⚠️ 중요: Pi Gateway는 확인할 필요 없음

**위치 정보, 토픽 이름, Nav2 연동 방식은 모두 젯슨 담당입니다!**

- ❌ Pi Gateway는 위치 정보를 알 필요 없음
- ❌ Pi Gateway는 Nav2 토픽 이름을 알 필요 없음
- ❌ Pi Gateway는 Nav2 연동 방식을 알 필요 없음

**Pi Gateway는 단순히:**
- ✅ 젯슨의 `arm/water` 토픽을 구독 → UART로 전달
- ✅ Nav2의 `cmd_vel_auto` 토픽을 구독 → UART로 전달

**이것만 하면 됩니다!** 나머지는 젯슨이 알아서 처리합니다.

## 💡 요약

**Pi Gateway는 단순히 브릿지 역할만 합니다.**

- 물 급수 중 이동은 **젯슨이 Nav2를 통해 처리**
- Pi Gateway는 **젯슨의 `arm/water` 토픽과 Nav2의 `cmd_vel_auto` 토픽을 그대로 전달**
- **추가 구현 불필요** - 현재 구조가 이미 올바름
- **위치 정보, Nav2 연동 방식 등은 젯슨 담당** - Pi Gateway는 신경 쓸 필요 없음

**물 급수 중 이동은 젯슨의 책임이며, Pi Gateway는 신경 쓸 필요가 없습니다!** 🎯

**Pi Gateway 입장에서는:**
- 젯슨이 보내는 신호 받기
- Nav2가 보내는 이동 명령 받기
- 이것들을 STM32로 전달하기

**이것만 하면 됩니다!** 나머지는 젯슨과 Nav2가 알아서 처리합니다.
