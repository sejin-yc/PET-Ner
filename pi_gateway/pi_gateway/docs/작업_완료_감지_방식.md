# 작업 완료 감지 방식 설명

STM32에서 작업 완료를 어떻게 감지하는지, 각 작업별로 설명합니다.

---

## 작업 종류별 완료 감지 방식

### 1. 급식 (FEED) - STM32가 직접 제어

**STM32가 하는 일:**
- 서보모터로 개폐통을 열어서 급식
- 서보모터에 목표 위치(각도) 설정
- 서보모터가 목표 위치 도달 대기

**완료 감지:**
```
1. Pi → STM32: FEED 명령 (level=1~3)
2. STM32: 서보모터 목표 위치 설정
3. 서보모터: 목표 위치로 이동 (PWM 제어)
4. 서보모터 위치 피드백: 목표 위치 도달 확인
5. STM32: 완료 감지 → STATUS(0x01, 0x01, 0x00) 전송
```

**핵심:** 서보모터는 **위치 피드백**이 있어서 목표 위치 도달 여부를 알 수 있습니다.
- 예: 목표 각도 90도 → 현재 각도 90도 → 완료

---

### 2. 츄르 (CHURU) - STM32가 직접 제어

**STM32가 하는 일:**
- 서보모터로 주사기를 밀어서 츄르 배출
- 서보모터에 목표 위치(행정 거리) 설정

**완료 감지:**
```
1. Pi → STM32: CHURU 명령 (enable=1)
2. STM32: 서보모터 목표 위치 설정 (주사기 밀기)
3. 서보모터: 목표 위치로 이동
4. 서보모터 위치 피드백: 목표 위치 도달 확인
5. STM32: 완료 감지 → STATUS(0x01, 0x04, 0x00) 전송
```

**핵심:** 서보모터 위치 피드백으로 완료 감지

---

### 3. 변 치우기 (LITTER_CLEAN) - 젯슨이 제어

**젯슨이 하는 일:**
- 모방학습 로봇팔로 변 치우기 작업 수행
- 4구역을 자율로 처리

**STM32가 하는 일:**
- 바퀴 잠금/해제만 담당
- 로봇팔은 직접 제어 안 함

**완료 감지:**
```
1. 젯슨 → Pi → STM32: ARM_START (action_id=1) → 바퀴 잠금
2. 젯슨: 로봇팔로 변 치우기 작업 수행 (자율)
3. 젯슨: 작업 완료 판단 (예: 모든 구역 처리 완료, 로봇팔 원위치)
4. 젯슨 → Pi → STM32: ARM_START (action_id=0) → 바퀴 해제
5. 젯슨 → STM32: 완료 신호 전송 (직접 또는 Pi 경유)
   → STM32: STATUS(0x01, 0x02, 0x00) 전송
```

**핵심:** 젯슨이 작업 완료를 판단하고, STM32에 완료 신호를 보내야 합니다.

**젯슨이 완료를 판단하는 방법:**
- 로봇팔이 모든 구역 처리 완료
- 로봇팔이 원위치로 복귀
- 센서로 변 제거 확인 (선택)
- 시간 기반 (예: 30초 후 완료 가정)

---

### 4. 급수 (WATER) - 젯슨이 제어

**젯슨이 하는 일:**
- 모방학습 로봇팔로 급수 작업 수행
- 물그릇 집기 → 물 버리기 → 물 받기 → 물그릇 두기

**STM32가 하는 일:**
- 바퀴 잠금/해제만 담당
- 로봇팔은 직접 제어 안 함

**완료 감지:**
```
1. 젯슨 → Pi → STM32: ARM_WATER (water_action=1) → 물그릇 집기
2. 젯슨: 로봇팔로 물그릇 집기
3. 젯슨 → Pi → STM32: ARM_WATER (water_action=2) → 물 버리기 (바퀴 잠금)
4. 젯슨: 로봇팔로 물 버리기
5. 젯슨 → Pi → STM32: ARM_WATER (water_action=3) → 물 받기 (바퀴 잠금)
6. 젯슨: 로봇팔로 물 받기
7. 젯슨 → Pi → STM32: ARM_WATER (water_action=4) → 물그릇 두기 (바퀴 잠금)
8. 젯슨: 로봇팔로 물그릇 두기
9. 젯슨: 전체 급수 작업 완료 판단
10. 젯슨 → STM32: 완료 신호 전송
    → STM32: STATUS(0x01, 0x03, 0x00) 전송
```

**핵심:** 젯슨이 각 단계 완료를 판단하고, 최종 완료 시 STM32에 신호 전송

**젯슨이 완료를 판단하는 방법:**
- 로봇팔 위치 피드백 (목표 위치 도달)
- 센서 데이터 (물그릇 감지 등)
- 시간 기반 (예: 각 단계 N초 후 완료 가정)

---

## 요약: 완료 감지 방법

| 작업 | 제어 주체 | 완료 감지 방법 |
|------|----------|----------------|
| **급식** | STM32 | 서보모터 위치 피드백 (목표 위치 도달) |
| **츄르** | STM32 | 서보모터 위치 피드백 (목표 위치 도달) |
| **변 치우기** | 젯슨 | 젯슨이 완료 판단 → STM32에 신호 전송 |
| **급수** | 젯슨 | 젯슨이 완료 판단 → STM32에 신호 전송 |

## "모터가 더 이상 안 움직이면 완료?" - 답변

**아니요, 그게 아닙니다!**

### 서보모터 (급식, 츄르)
- 서보모터는 **위치 피드백**이 있어서 목표 위치 도달 여부를 알 수 있습니다
- 예: 목표 각도 90도 → 현재 각도 90도 → 완료
- "모터가 안 움직인다"가 아니라 "목표 위치에 도달했다"로 판단

### 로봇팔 (변 치우기, 급수)
- 젯슨이 로봇팔을 제어하므로, 젯슨이 완료를 판단해야 합니다
- 젯슨이 완료 판단 후 STM32에 신호를 보내야 합니다
- STM32는 "모터가 안 움직인다"로 판단할 수 없습니다 (젯슨이 제어하므로)

---

## 젯슨 → STM32 완료 신호 전달 방법

### 방법 1: 젯슨이 직접 STM32에 신호 전송 (권장)

```
젯슨 → STM32: 완료 신호 (직접 UART 또는 GPIO)
STM32 → Pi: STATUS(0x01, 0x02, 0x00) 전송
```

### 방법 2: 젯슨 → Pi → STM32 (현재 구조)

```
젯슨 → Pi: ROS 토픽 또는 API로 완료 알림
Pi → STM32: 완료 신호 전송 (새로운 UART 명령 필요)
STM32 → Pi: STATUS(0x01, 0x02, 0x00) 전송
```

**현재 구조에서는 방법 2가 더 자연스럽습니다:**
- 젯슨이 `arm/start` (action_id=0) 발행 → 변 치우기 완료
- Pi가 이를 받아서 STM32에 완료 신호 전송
- STM32가 STATUS로 확인 응답

---

## 결론

**작업 완료 감지:**
1. **급식/츄르**: 서보모터 위치 피드백으로 자동 감지 (STM32)
2. **변 치우기/급수**: 젯슨이 완료 판단 후 신호 전송 필요

**현재 STATUS 구조는:**
- STM32가 직접 제어하는 작업: 서보모터 피드백으로 완료 감지
- 젯슨이 제어하는 작업: 젯슨이 완료 신호를 보내야 함

**젯슨 담당자와 협의 필요:**
- 변 치우기/급수 완료 시 STM32에 신호를 어떻게 보낼지
- 직접 UART 연결 vs Pi 경유
