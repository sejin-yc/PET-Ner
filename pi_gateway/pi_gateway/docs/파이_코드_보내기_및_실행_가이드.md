# 파이로 코드 보내기 & Gateway 실행법

수정한 코드를 라즈베리파이로 보내고, Gateway를 다시 실행하는 방법입니다.

---

## 1. 파이로 수정된 코드 보내기

### 1-1. rsync (권장, 변경분만 전송)

```bash
cd pi_gateway   # 프로젝트 루트

# IP/계정/경로는 환경변수 또는 스크립트 내 설정 사용
ROBOT_USER=c203 ROBOT_IP=192.168.100.254 ROBOT_PATH=~/pi_gateway ./scripts/sync_to_robot.sh
```

**스크립트 기본값** (`scripts/sync_to_robot.sh`):
- `ROBOT_USER`: c203
- `ROBOT_IP`: 192.168.100.254
- `ROBOT_PATH`: ~/pi_gateway

**직접 지정 예:**
```bash
ROBOT_USER=pi ROBOT_IP=192.168.0.10 ROBOT_PATH=/home/pi/pi_gateway ./scripts/sync_to_robot.sh
```

- `.venv/`, `__pycache__/`, `.git/` 등 제외
- 변경된 파일만 전송

### 1-2. 특정 파일만 scp

```bash
cd pi_gateway

scp src/main.py src/uart_link.py config/params.yaml \
  c203@192.168.100.254:~/pi_gateway/
# 대상 경로를 폴더가 아니라 각 파일 맞는 위치로 넣어서 사용
# 예: scp src/main.py c203@192.168.100.254:~/pi_gateway/src/
```

### 1-3. quick_sync.sh (수정한 파일 목록 고정)

`scripts/quick_sync.sh` 안의 `FILES` 배열에 경로를 넣어두고:

```bash
./scripts/quick_sync.sh
```

동일한 계정/IP/경로를 쓰도록 `ROBOT_*` 변수가 설정돼 있어야 합니다.

---

## 2. 파이에서 Gateway 다시 실행하기

### 2-1. 스크립트로 실행 (권장)

```bash
ssh c203@192.168.100.254   # 실제 계정@IP

cd ~/pi_gateway            # ROBOT_PATH와 동일하게
chmod +x scripts/run_gateway.sh
./scripts/run_gateway.sh
```

- `.venv` 없으면 만들고, `pip install -r requirements.txt` 후 `src/main.py` 실행
- venv Python 사용

### 2-2. 수동 실행 (venv 활성화 후)

```bash
cd ~/pi_gateway
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

export PYTHONPATH="$(pwd):$(pwd)/src:$PYTHONPATH"
python3 src/main.py
```

### 2-3. 실기기(UART) 모드로 실행

```bash
cd ~/pi_gateway
export UART_ENABLED=1
export UART_PORT=/dev/ttyAMA0   # GPIO 직결. 또는 /dev/serial0, /dev/ttyUSB0
./scripts/run_gateway.sh
```

**TX 로그까지 보고 싶을 때:**
```bash
export UART_DEBUG_TX=1
./scripts/run_gateway.sh
```

### 2-4. 실행 확인

```bash
curl -s http://localhost:8000/health
# {"ok": true, ...} 나오면 정상
```

---

## 3. 한 번에 하기 (PC에서 보내고 → 파이에서 실행)

```bash
# 1) PC에서 동기화
cd pi_gateway
ROBOT_USER=c203 ROBOT_IP=192.168.100.254 ROBOT_PATH=~/pi_gateway ./scripts/sync_to_robot.sh

# 2) 파이 SSH 접속 후 실행
ssh c203@192.168.100.254 "cd ~/pi_gateway && ./scripts/run_gateway.sh"
```

백그라운드 실행이 아니라 포그라운드로 띄우는 예시입니다.  
`tmux` / `screen` 쓰면 SSH 끊어도 Gateway 유지 가능.

---

## 4. 설정 정리

| 항목 | 예시 | 비고 |
|------|------|------|
| Pi 사용자 | `c203` 또는 `pi` | `ROBOT_USER` |
| Pi IP | `192.168.100.254` | `ROBOT_IP` |
| 프로젝트 경로 | `~/pi_gateway` | `ROBOT_PATH` |
| UART 포트 | `/dev/ttyUSB0`, `/dev/serial0` | 실기기 연결 시 |

---

## 5. 자주 쓰는 순서 요약

1. **코드 수정** (PC)
2. **동기화**: `./scripts/sync_to_robot.sh` (또는 `quick_sync` / `scp`)
3. **SSH 접속**: `ssh c203@192.168.100.254`
4. **실행**: `cd ~/pi_gateway && ./scripts/run_gateway.sh`
5. **확인**: `curl -s http://localhost:8000/health`
