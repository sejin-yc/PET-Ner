# 엔코더 3개 계산 정확도 향상 방법

왼쪽 앞바퀴(fl) 고장으로 3개 엔코더만 사용할 때 정확도를 높이는 방법입니다.

---

## ✅ 적용된 개선 사항

### 1. 가중치 적용

정상 바퀴에 가중치를 적용하여 fl 바퀴의 부재를 보상:

```python
weight_fr = 1.2  # 오른쪽 앞바퀴 (fl의 대칭 위치, 높은 가중치)
weight_rl = 0.8  # 왼쪽 뒤바퀴 (단독, 낮은 가중치)
weight_rr = 1.0  # 오른쪽 뒤바퀴 (기본 가중치)
```

**이유:**
- fr은 fl의 대칭 위치이므로 fl의 기여도를 추정하는 데 유용
- rl은 단독이므로 가중치를 낮춤

### 2. 속도 필터링 (지수 이동 평균)

노이즈 제거 및 부드러운 동작:

```python
vx = alpha * vx_raw + (1 - alpha) * vx_prev
```

**파라미터:**
- `encoder_filter_alpha`: 0.0~1.0 (기본값: 0.7)
  - 높을수록 새 값에 더 민감 (빠른 반응, 노이즈에 민감)
  - 낮을수록 부드러움 (느린 반응, 노이즈에 강함)

### 3. IMU Yaw 우선 사용

엔코더보다 IMU yaw가 더 정확:

```python
if use_imu_yaw and imu_yaw is not None:
    yaw = imu_yaw  # IMU 우선 사용
    # 엔코더 yaw와 비교하여 보정
else:
    yaw = encoder_yaw  # IMU 없으면 엔코더 사용
```

**장점:**
- 회전 각도 측정 정확도 향상
- 엔코더 오차 누적 방지

### 4. 공분산 설정

3륜 모드의 불확실성을 반영:

```python
# 위치 불확실성
pose.covariance[0] = 0.1   # x
pose.covariance[7] = 0.1   # y
pose.covariance[35] = 0.2  # yaw

# 속도 불확실성
twist.covariance[0] = 0.05   # vx
twist.covariance[7] = 0.08   # vy (비대칭으로 인해 더 높음)
twist.covariance[35] = 0.1   # wz
```

## 🔧 추가 개선 방법

### 1. 칼만 필터 적용

센서 융합을 통한 정확도 향상:

```python
# IMU + 엔코더 융합
# 예측 단계: 엔코더로 위치 예측
# 업데이트 단계: IMU로 보정
```

**장점:**
- 센서 간 상호 보정
- 노이즈 제거
- 장기적 정확도 향상

### 2. 비대칭성 보정

왼쪽 앞바퀴 부재로 인한 비대칭 보정:

```python
# 좌우 이동 시 보정 계수 적용
if abs(vy) > threshold:
    vy_corrected = vy * asymmetry_factor
```

### 3. 속도 제한 및 이상치 제거

비정상적인 속도 값 필터링:

```python
# 최대 속도 제한
vx = max(-vx_max, min(vx_max, vx))
vy = max(-vy_max, min(vy_max, vy))
wz = max(-wz_max, min(wz_max, wz))

# 이상치 제거 (이전 값과 차이가 너무 크면)
if abs(vx - prev_vx) > threshold:
    vx = prev_vx  # 이전 값 유지
```

### 4. 주기적 위치 보정

외부 센서(예: 아루코 마커, GPS)로 주기적 보정:

```python
# 아루코 마커로 위치 보정
if aruco_marker_detected:
    x, y = aruco_position
    self._x, self._y = x, y  # 위치 리셋
```

## 💡 권장 설정

```python
use_imu_yaw = True          # IMU yaw 사용
encoder_filter_alpha = 0.7  # 중간 필터링
weight_fr = 1.2             # fr 가중치
weight_rl = 0.8             # rl 가중치
weight_rr = 1.0             # rr 가중치
```

### 고정밀도 요구 시

```python
use_imu_yaw = True          # IMU 필수
encoder_filter_alpha = 0.5  # 강한 필터링
weight_fr = 1.3             # fr 가중치 증가
# 칼만 필터 추가 고려
```

### 빠른 반응 요구 시

```python
use_imu_yaw = True          # IMU 사용
encoder_filter_alpha = 0.8  # 약한 필터링
weight_fr = 1.1             # 가중치 조정
```

## 📝 참고

- **가중치 합**: 항상 정규화되어야 함 (현재 자동 계산)
- **필터링**: 실시간 성능에 영향 (너무 강하면 지연 발생)
- **IMU**: 필수는 아니지만 정확도 향상에 도움
- **공분산**: SLAM/내비게이션 알고리즘이 불확실성을 고려할 수 있도록 설정
