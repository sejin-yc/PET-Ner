## 전체 구조 개요

```
Web/Backend → WebSocket/HTTP → Pi Gateway → UART → STM32
                ↓
            ROS 토픽 (ROS 모드)
```

**핵심 역할:**
- **통신 변환**: Web/Backend의 JSON → STM32의 바이너리 프로토콜
- **안전 제어**: E-STOP, 타임아웃, 로봇팔 동작 중 바퀴 잠금
- **자율 기능**: 패트롤, 자동 급식, 변 치우기 스케줄링

## 1. 패트롤 루프 (자율 순찰)

### 🎯 목적
Nav2가 연결되기 전까지 **간단한 패턴으로 자율 순찰**을 수행합니다.

### 📝 로직 흐름

```
1. auto 모드로 전환
   ↓
2. PatrolProfile이 패턴 재생
   - 구간1: 2초 전진 (vx=0.15)
   - 구간2: 1.2초 회전 (wz=0.7)
   - 반복
   ↓
3. cmd_vel_auto 토픽에 발행
   ↓
4. Mux가 cmd_vel_out으로 선택
   ↓
5. STM32로 전송
```

### 🔧 주요 함수

#### `PatrolLoop` 클래스
```python
class PatrolLoop(Node):
    """자율 순찰 패턴을 cmd_vel_auto로 발행"""
    
    def tick(self):
        """30Hz로 호출됨"""
        if mode == "auto":
            vx, wz = self.profile.step(dt)  # 패턴에서 속도 계산
            out.linear.x = vx
            out.angular.z = wz
            self.pub_auto.publish(out)  # cmd_vel_auto 발행
```

#### `PatrolProfile.step()` 함수
```python
def step(self, dt: float) -> Tuple[float, float]:
    """경과 시간에 따라 현재 구간의 속도 반환"""
    self._seg_elapsed += dt  # 시간 누적
    
    # 구간 시간이 지나면 다음 구간으로
    while self._seg_elapsed >= seg.duration_s:
        self._seg_idx = (self._seg_idx + 1) % len(segments)
    
    return seg.vx_mps, seg.wz_rps  # 현재 구간 속도 반환
```

### 💡 발표 포인트
- **Nav2 전 임시 솔루션**: Nav2가 연결되면 Nav2가 `cmd_vel_auto`를 발행
- **간단한 패턴 기반**: 복잡한 경로 계획 없이 전진/회전 반복
- **액션 스케줄러**: 순찰 중 자동 급식, 변 치우기, 급수 실행

### 📅 패트롤 액션 스케줄러

#### 변 치우기 (저녁 순찰 시 하루 1번)

def check_litter_clean_on_patrol_start(self, now: float):

## 2. 로봇팔 제어 (변 치우기, 급수)

### 🎯 목적
젯슨의 모방학습 로봇팔이 작업할 때 **바퀴를 잠가서 안전하게** 제어합니다.

### 📝 로직 흐름

#### 변 치우기
```
1. 젯슨이 arm/start 토픽 발행 (action_id=1)
   ↓
2. ArmControlBridge가 수신
   ↓
3. UART로 ID_ARM_START 전송 (action_id=1)
   ↓
4. STM32가 바퀴 모터 잠금
   ↓
5. RosCmdVelBridge가 cmd_vel 차단 (0,0,0 전송)
   ↓
6. 젯슨 로봇팔이 변 치우기 수행
   ↓
7. 완료 후 action_id=0 전송 → 바퀴 해제
```

#### 급수
```
1. 젯슨이 arm/water 토픽 발행 (water_action=0~4)
   ↓
2. ArmControlBridge가 수신
   ↓
3. UART로 ID_ARM_WATER 전송
   ↓
4. water_action에 따라 바퀴 잠금/해제:
   - 0, 1: 바퀴 해제 (이동 가능)
   - 2, 3, 4: 바퀴 잠금 (로봇팔 동작 중)
   ↓
5. RosCmdVelBridge가 바퀴 잠금 시 cmd_vel 차단
```

### 🔧 주요 함수

#### `ArmControlBridge.on_start()` 함수
```python
def on_start(self, msg: UInt8):
    """로봇팔 시작 토픽 수신"""
    action_id = int(msg.data)
    is_starting = (action_id != 0)  # 0이 아니면 시작
    
    # UART로 전송
    frame = make_arm_start_frame(action_id)
    self.uart.send(frame)
    
    # 바퀴 잠금 상태 업데이트
    self.arm_active = is_starting
    
    # RosCmdVelBridge에 알림 (cmd_vel 차단)
    if self.cmdvel_bridge is not None:
        self.cmdvel_bridge.set_arm_start_active(self.arm_active)
```

#### `RosCmdVelBridge.tick()` 함수 (바퀴 잠금 부분)
```python
def tick(self):
    """cmd_vel_out을 UART로 전송"""
    
    # 로봇팔 동작 중이면 바퀴 잠금
    if self.arm_active:  # arm_start 또는 arm_water 활성화
        self.uart.send(make_cmd_vel_frame(0.0, 0.0, 0.0))  # 정지
        return
    
    # 정상 제어
    vx, vy, wz = self.last_cmd.linear.x, ...
    self.uart.send(make_cmd_vel_frame(vx, vy, wz))
```

### 💡 발표 포인트
- **안전 우선**: 로봇팔 동작 중 바퀴가 움직이면 위험 → 자동 잠금
- **이중 잠금**: STM32 레벨 + Gateway 레벨 (cmd_vel 차단)
- **단계별 제어**: 급수는 5단계 (이동→집기→버리기→받기→두기)

## 3. 츄르 주기

### 🎯 목적
젯슨이 요청한 츄르를 **STM32 서보모터로 주사기를 밀어서** 배출합니다.

### 📝 로직 흐름

```
1. 젯슨이 feed/fill 토픽 발행 (enable=1)
   ↓
2. ChuruControlBridge가 수신
   ↓
3. UART로 ID_CHURU 전송 (enable=1)
   ↓
4. STM32가 서보모터로 주사기 밀기
   ↓
5. 츄르 배출 완료
```

### 🔧 주요 함수

#### `ChuruControlBridge.on_churu()` 함수
```python
def on_churu(self, msg: UInt8):
    """츄르 주기 토픽 수신"""
    enable = int(msg.data)  # 0=정지, 1=츄르 주기
    enable = min(1, max(0, enable))  # 0~1 범위 제한
    
    # UART로 전송
    frame = make_churu_frame(enable)
    self.uart.send(frame)
```

### 💡 발표 포인트
- **간단한 브릿지**: 젯슨 요청 → STM32 전달
- **하드웨어 제어**: STM32가 서보모터로 실제 츄르 배출

## 4. 엔코더 기반 오도메트리

### 🎯 목적
엔코더와 IMU 데이터로 **로봇의 위치와 속도**를 계산해서 `/odom` 토픽에 발행합니다.

### 📝 로직 흐름

```
1. STM32가 엔코더 틱 전송 (enc_fl, enc_fr, enc_rl, enc_rr)
   ↓
2. UART 디코딩 → telemetry/encoder 토픽 발행
   ↓
3. EncoderOdomNode가 구독
   ↓
4. 이전 틱과 비교해서 속도 계산:
   - vx = (v_fl + v_fr + v_rl + v_rr) / 4
   - vy = (-v_fl + v_fr - v_rl + v_rr) / 4
   - wz = (-v_fl + v_fr + v_rl - v_rr) / (4 * (lx + ly))
   ↓
5. IMU yaw로 body→world 변환:
   - x += (vx * cos(yaw) - vy * sin(yaw)) * dt
   - y += (vx * sin(yaw) + vy * cos(yaw)) * dt
   ↓
6. /odom 토픽에 nav_msgs/Odometry 발행
```

### 🔧 주요 함수

#### `EncoderOdomNode._on_encoder()` 함수
```python
def _on_encoder(self, msg: String):
    """엔코더 데이터 수신"""
    # JSON 파싱
    obj = json.loads(msg.data)
    fl, fr, rl, rr = obj["enc_fl"], obj["enc_fr"], ...
    
    # 이전 값과 비교
    dt = now - self._prev_t
    d_fl = (fl - self._prev[0]) * self._meters_per_tick
    
    # 속도 계산 (메카넘 휠 역기구학)
    vx = (v_fl + v_fr + v_rl + v_rr) / 4.0
    vy = (-v_fl + v_fr - v_rl + v_rr) / 4.0
    wz = (-v_fl + v_fr + v_rl - v_rr) / (4.0 * (lx + ly))
    
    # IMU yaw로 위치 적분
    if self._imu_yaw is not None:
        yaw = self._imu_yaw  # IMU 사용
    else:
        yaw = self._yaw + wz * dt  # wz 적분 (백업)
    
    # body→world 변환
    self._x += (vx * cos(yaw) - vy * sin(yaw)) * dt
    self._y += (vx * sin(yaw) + vy * cos(yaw)) * dt
    
    # /odom 발행
    odom.pose.pose.position.x = self._x
    odom.pose.pose.position.y = self._y
    odom.pose.pose.orientation = yaw_to_quat(yaw)
    self._pub.publish(odom)
```

### 💡 발표 포인트
- **메카넘 휠 역기구학**: 4륜 속도 → 로봇 속도 변환
- **IMU yaw 사용**: wz 적분 대신 IMU yaw 사용 (드리프트 방지)
- **SLAM 연동**: `/odom` 토픽을 SLAM이 구독해서 사용

## 5. 명령 선택 (Mux)

### 🎯 목적
**teleop(수동)과 auto(자율) 명령 중 하나를 선택**해서 STM32로 전달합니다.

### 📝 로직 흐름

```
1. cmd_vel_teleop (Web 입력)
   ↓
2. cmd_vel_auto (Nav2/Patrol)
   ↓
   → CmdVelMux
   ↓
3. 모드에 따라 선택:
   - teleop 모드 → cmd_vel_teleop 선택
   - auto 모드 → cmd_vel_auto 선택 (타임아웃 체크)
   ↓
4. cmd_vel_out 발행
   ↓
5. RosCmdVelBridge가 UART로 전송
```

### 🔧 주요 함수

#### `CmdVelMux.tick()` 함수
```python
def tick(self):
    """30Hz로 호출됨"""
    out = Twist()
    
    if self.estop:
        self.pub_out.publish(out)  # E-STOP이면 정지
        return
    
    now = time.time()
    
    # auto 모드이고 최근에 auto 명령이 왔으면
    if self.mode == "auto" and (now - self.last_auto_ts) <= 0.5:
        out = self.last_auto  # auto 명령 사용
    elif self.mode != "auto":
        out = self.last_teleop  # teleop 명령 사용
    
    self.pub_out.publish(out)
```

### 💡 발표 포인트
- **우선순위**: E-STOP > auto (타임아웃 체크) > teleop
- **타임아웃**: auto 명령이 0.5초 이상 안 오면 안전하게 정지
- **모드 전환**: Web에서 모드 변경 시 즉시 반영

## 6. 안전 제어

### 🎯 목적
**로봇이 위험한 상황에 빠지지 않도록** 여러 안전 장치를 구현했습니다.

### 📝 안전 기능들

#### 1. E-STOP (비상정지)
```python
# WebSocket 또는 HTTP API로 E-STOP 요청
if STATE.estop:
    uart.send(make_estop_frame(1))  # STM32에 즉시 정지 명령
    uart.send(make_cmd_vel_frame(0.0, 0.0, 0.0))  # 속도 0
```

#### 2. 명령 타임아웃
```python
# RosCmdVelBridge.tick()
stale_ms = (now - self.last_cmd_ts) * 1000.0
if stale_ms > self.cmd_timeout_ms:  # 기본 500ms
    vx, vy, wz = 0.0, 0.0, 0.0  # 타임아웃 시 정지
```

#### 3. 로봇팔 동작 중 바퀴 잠금
```python
# RosCmdVelBridge.tick()
if self.arm_active:  # 로봇팔 동작 중
    self.uart.send(make_cmd_vel_frame(0.0, 0.0, 0.0))  # 바퀴 정지
    return
```

#### 4. HEARTBEAT (워치독)
```python
# RosCmdVelBridge.tick()
if now - self._last_heartbeat_ts >= 0.2:  # 5Hz
    self.uart.send(make_heartbeat_frame())  # STM32 연결 확인
```

### 💡 발표 포인트
- **다층 안전**: Gateway 레벨 + STM32 레벨
- **타임아웃**: 명령이 안 오면 자동 정지
- **워치독**: STM32가 HEARTBEAT 안 받으면 정지

## ❓ 예상 질문과 답변

**Q: Nav2가 연결되면 PatrolLoop는 필요 없나요?**  
A: 네, Nav2가 `cmd_vel_auto`를 발행하면 PatrolLoop는 사용하지 않습니다. 하지만 Nav2 연결 전까지는 유용합니다.

**Q: 로봇팔 제어는 Pi Gateway에서 하나요?**  
A: 아니요, Pi Gateway는 브릿지 역할만 합니다. 실제 로봇팔 제어는 젯슨의 모방학습 모델이 담당합니다.

**Q: 오도메트리는 정확한가요?**  
A: 엔코더 틱과 IMU yaw를 조합해서 계산하므로, 짧은 시간 동안은 정확합니다. 장시간 사용 시 SLAM과 함께 사용하는 것이 좋습니다.

**Q: 안전 기능이 충분한가요?**  
A: E-STOP, 타임아웃, 바퀴 잠금, 워치독 등 다층 안전 장치를 구현했습니다. 추가로 필요하면 확장 가능합니다.
