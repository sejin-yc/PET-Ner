# 시스템 아키텍처 참고 (S14P11C203)

S14P11C203 프로젝트의 서버 측 아키텍처와, **Pi Gateway / MQTT 브릿지**가 어디에 붙는지 정리합니다.

---

## S14P11C203 서버 아키텍처 (5개 컨테이너)

Nginx Gateway를 중심으로 5개의 컨테이너가 연결되어 동작합니다.

| 서비스명 | 컨테이너 이름 | 역할 | 포트 (Host:Container) | 비고 |
|----------|--------------|------|------------------------|------|
| Gateway | robot_nginx | Reverse Proxy & SSL Termination | 80, 443 | 모든 트래픽의 진입점 (HTTPS) |
| Frontend | robot_client | React 웹 서버 (Nginx Upstream) | (Internal) | Nginx를 통해서만 접근 |
| Backend | robot_server | Spring Boot API & WebSocket | 8080:8080 | 비즈니스 로직, WebRTC 시그널링 |
| Broker | robot_mqtt | Eclipse Mosquitto | 1883:1883 | 로봇-서버 간 메시지 중계 (ROS2) |
| Database | robot_db | PostgreSQL 15 | 5432:5432 | 사용자, 로그, 영상 메타데이터 저장 |

## Pi Gateway / MQTT 브릿지 위치

**Pi Gateway**와 **MQTT 브릿지**는 위 Docker 컨테이너 밖에서 동작합니다.

```
[웹 브라우저]
     ↓ HTTPS
[robot_nginx] → [robot_client] (FE)
     ↓
[robot_server] (BE) ←→ [robot_mqtt] (MQTT 1883)
                            ↓
                    [MQTT 브릿지]  ← Docker 밖 (로컬 PC 또는 라즈베리파이)
                            ↓ WebSocket
                    [Pi Gateway]   ← Docker 밖 (라즈베리파이, :8000)
                            ↓ UART
                    [STM32 / 로봇]
```

- **robot_mqtt (1883):** MQTT 브릿지가 **구독/발행**하는 브로커. `MQTT_HOST` = 이 브로커 주소 (로컬이면 `localhost`, 배포 서버면 해당 도메인/IP).
- **Pi Gateway (8000):** 라즈베리파이에서 실행하는 HTTP + WebSocket 서버. MQTT 브릿지가 **연결 대상**. `PI_GATEWAY_URL` = `http://localhost:8000` 또는 `http://라즈베리파이IP:8000`.

## 로컬 vs 배포

| 환경 | FE | BE | MQTT (robot_mqtt) | Pi Gateway | MQTT 브릿지 |
|------|----|----|-------------------|------------|-------------|
| **로컬 개발** | npm run dev (5173) | Docker 8080 | Docker 1883 | 로컬 데모 :8000 또는 파이 IP:8000 | 로컬 PC 또는 파이에서 실행 |
| **서버 배포** | robot_client (Nginx 경유) | robot_server 8080 | robot_mqtt 1883 | 파이에서 실행 (:8000) | 파이 또는 서버에서 실행, PI_GATEWAY_URL=파이IP |
