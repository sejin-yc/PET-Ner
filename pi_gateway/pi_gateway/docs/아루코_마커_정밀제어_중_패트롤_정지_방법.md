# 아루코 마커 정밀 제어 중 패트롤 정지 방법

아루코 마커로 정밀 제어하는 동안 패트롤을 멈추는 방법입니다.

---

## 🎯 해결 방법

**SLAM/Nav2 담당자가 `control_mode` 토픽에 "teleop"을 발행하면 패트롤이 멈춥니다.**

---

## 📋 방법 설명

### 현재 구조

Pi Gateway는 `control_mode` 토픽을 구독하고 있습니다:

```python
# src/ros_cmdvel.py:173, 190-194
self.sub_mode = self.create_subscription(String, "control_mode", self.on_mode, 10)

def on_mode(self, msg: String):
    self.mode = "auto" if msg.data == "auto" else "teleop"
    if self.mode != "auto":
        self.profile.reset()  # 패트롤 패턴 리셋
```

**동작:**
- `control_mode = "auto"` → 패트롤 실행 (PatrolLoop가 cmd_vel_auto 발행)
- `control_mode = "teleop"` → 패트롤 멈춤 (PatrolLoop가 cmd_vel_auto 발행 안 함)

---

## 🔄 사용 방법

### 1. 아루코 마커 정밀 제어 시작 전

**SLAM/Nav2 담당자가 실행:**
```bash
ros2 topic pub /control_mode std_msgs/String "data: 'teleop'"
```

**또는 코드에서:**
```python
# SLAM/Nav2 노드에서
from std_msgs.msg import String
pub_mode = node.create_publisher(String, "control_mode", 10)
pub_mode.publish(String(data="teleop"))  # 패트롤 멈춤
```

**결과:**
- `PatrolLoop`가 `cmd_vel_auto` 발행 중단
- 패트롤 패턴 리셋
- Nav2가 `cmd_vel_auto`를 발행하면 Nav2 명령 사용 가능

---

### 2. 아루코 마커 정밀 제어 중

**Nav2가 정밀 제어 명령 발행:**
```bash
# Nav2가 cmd_vel_auto 또는 /cmd_vel에 정밀 제어 명령 발행
ros2 topic pub /cmd_vel_auto geometry_msgs/Twist "{linear: {x: 0.05, y: 0.0}, angular: {z: 0.0}}"
```

**동작:**
- `control_mode = "teleop"`이므로 PatrolLoop는 발행 안 함
- Nav2의 명령만 사용됨
- 정밀 제어 가능

---

### 3. 아루코 마커 정밀 제어 완료 후

**SLAM/Nav2 담당자가 실행:**
```bash
ros2 topic pub /control_mode std_msgs/String "data: 'auto'"
```

**또는 코드에서:**
```python
pub_mode.publish(String(data="auto"))  # 패트롤 재개
```

**결과:**
- 패트롤 모드로 전환
- PatrolLoop가 다시 `cmd_vel_auto` 발행 시작
- 또는 Nav2가 계속 발행하면 Nav2 명령 사용

---

## 📊 전체 흐름

```
1. 아루코 마커 정밀 제어 시작
   SLAM/Nav2 → control_mode = "teleop" 발행
   ↓
   PatrolLoop → 패트롤 멈춤 (cmd_vel_auto 발행 중단)
   ↓
   CmdVelMux → Nav2의 cmd_vel_auto만 사용 (PatrolLoop 없음)

2. 아루코 마커 정밀 제어 중
   Nav2 → cmd_vel_auto 발행 (정밀 제어 명령)
   ↓
   Pi Gateway → STM32로 전달
   ↓
   로봇 정밀 제어 수행

3. 아루코 마커 정밀 제어 완료
   SLAM/Nav2 → control_mode = "auto" 발행
   ↓
   PatrolLoop → 패트롤 재개 (cmd_vel_auto 발행 시작)
```

## 💡 대안: control_estop 사용

**더 강력한 정지가 필요하면:**

```bash
ros2 topic pub /control_estop std_msgs/Bool "data: true"
```

**동작:**
- 모든 이동 명령 차단 (비상 정지)
- 패트롤뿐만 아니라 모든 제어 멈춤

**주의:**
- 아루코 마커 정밀 제어도 멈춤
- 정밀 제어 중에는 사용하지 않는 것이 좋음

## ✅ 요약

**SLAM/Nav2 담당자가 할 일:**

1. **아루코 마커 정밀 제어 시작 전:**
   ```bash
   ros2 topic pub /control_mode std_msgs/String "data: 'teleop'"
   ```
   → 패트롤 멈춤

2. **아루코 마커 정밀 제어 중:**
   - Nav2가 `cmd_vel_auto`에 정밀 제어 명령 발행
   - Pi Gateway가 자동으로 전달

3. **아루코 마커 정밀 제어 완료 후:**
   ```bash
   ros2 topic pub /control_mode std_msgs/String "data: 'auto'"
   ```
   → 패트롤 재개

**Pi Gateway는:**
- `control_mode` 토픽을 구독하고 있음
- "teleop"이면 패트롤 멈춤
- "auto"이면 패트롤 재개
- **추가 코드 수정 불필요**
