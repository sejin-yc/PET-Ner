# 급식 레벨 변환 효율성 분석

레벨 1~3으로 변환하는 것이 효율적인지 분석합니다.

---

## 🔍 현재 구조

### UART 프로토콜
```python
ID_FEED = 0x05   # 급식: uint8 (level: 1~3)
make_feed_frame(level: int)  # level: 1~3만 받음
```

**프로토콜 제약:**
- UART 프레임이 `uint8` (1바이트)만 받음
- 레벨 1~3으로 정의되어 있음

---

## 📊 레벨 변환 vs 직접 전송 비교

### 방법 1: 레벨 변환 (현재 방식)

**젯슨 → Pi Gateway:**
```
그램 (정확한 값) → 퍼센트 → 레벨 (1~3) → UART
```

**장점:**
- ✅ UART 프로토콜과 일치
- ✅ STM32가 처리하기 간단 (레벨만 받으면 됨)
- ✅ 통신 데이터 작음 (1바이트)

**단점:**
- ❌ 정보 손실 (정확한 그램/퍼센트 정보 사라짐)
- ❌ 레벨 경계에서 불연속 (예: 32.9%와 33.1%는 다른 레벨)

**예시:**
- `grams = 100g` (23.4%) → 레벨 1
- `grams = 150g` (35.1%) → 레벨 2
- `grams = 200g` (46.7%) → 레벨 2
- `grams = 300g` (70.1%) → 레벨 3

---

### 방법 2: 그램 단위 직접 전송 (프로토콜 변경 필요)

**젯슨 → Pi Gateway:**
```
그램 (정확한 값) → UART (float32 또는 uint16)
```

**장점:**
- ✅ 정보 손실 없음
- ✅ 정확한 사료량 전달
- ✅ STM32가 세밀한 제어 가능

**단점:**
- ❌ UART 프로토콜 변경 필요 (`uint8` → `float32` 또는 `uint16`)
- ❌ STM32 코드 수정 필요
- ❌ 통신 데이터 증가 (1바이트 → 4바이트 또는 2바이트)

---

### 방법 3: 퍼센트 전송 (프로토콜 변경 필요)

**젯슨 → Pi Gateway:**
```
그램 → 퍼센트 (0~100) → UART (uint8)
```

**장점:**
- ✅ 레벨보다 정확함
- ✅ 통신 데이터 작음 (1바이트)
- ✅ STM32가 퍼센트에 따라 세밀한 제어 가능

**단점:**
- ❌ UART 프로토콜 변경 필요 (의미 변경)
- ❌ STM32 코드 수정 필요

---

## 💡 효율성 분석

### 현재 방식 (레벨 1~3)이 효율적인 경우

**STM32가 레벨만으로 충분한 경우:**
- 서보모터로 개폐통을 열어서 급식
- 레벨에 따라 개폐 시간만 조절하면 됨
- 예: 레벨 1 = 1초, 레벨 2 = 2초, 레벨 3 = 3초

**장점:**
- 구현 간단
- 프로토콜 변경 불필요
- STM32 코드 수정 최소화

---

### 레벨 변환이 비효율적인 경우

**STM32가 정확한 사료량이 필요한 경우:**
- 그램 단위로 정확한 급식량 제어 필요
- 퍼센트에 따라 세밀한 제어 필요
- 예: 23.4%일 때와 32.9%일 때 다른 양 급식

**단점:**
- 정보 손실
- 레벨 경계에서 불연속

---

## 🔧 권장 사항

### 현재 프로토콜 유지 (레벨 1~3)

**이유:**
1. **UART 프로토콜이 이미 정의됨**
   - `ID_FEED = 0x05`, `uint8` (1바이트)
   - 레벨 1~3으로 명시되어 있음

2. **STM32 구현 간단**
   - 레벨만 받아서 서보모터 제어
   - 복잡한 계산 불필요

3. **충분한 정확도**
   - 급식은 대략적인 양이면 충분
   - 레벨 1(적음), 2(보통), 3(많음)으로 구분 가능

---

### 프로토콜 변경이 필요한 경우

**STM32가 정확한 사료량이 필요한 경우:**
- UART 프로토콜 변경 (`uint8` → `float32` 또는 `uint16`)
- STM32 코드 수정
- Pi Gateway 코드 수정

**변경 예시:**
```python
# 기존
def make_feed_frame(level: int) -> bytes:  # level: 1~3
    payload = bytes([level & 0xFF])

# 변경 후
def make_feed_frame(grams: float) -> bytes:  # 그램 단위
    payload = struct.pack("<f", float(grams))  # float32 (4바이트)
```

## 📝 결론

### 현재 방식 (레벨 1~3) 권장

**이유:**
1. ✅ UART 프로토콜과 일치
2. ✅ STM32 구현 간단
3. ✅ 급식 목적에 충분한 정확도
4. ✅ 프로토콜 변경 불필요

**레벨 변환 로직:**
```python
if percent < 33.0:
    feed_level = 1  # 적음
elif percent < 66.0:
    feed_level = 2  # 보통
else:
    feed_level = 3  # 많음
```

**효율성:**
- 정보 손실이 있지만, 급식 목적에는 충분
- 프로토콜 변경 비용 대비 효율적
