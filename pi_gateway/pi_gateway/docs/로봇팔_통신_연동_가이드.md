## 통신 방식 분석

### 1. ROS2 Action 서버 사용

로봇팔 팀은 **ROS2 Action**을 사용하여 로봇팔 제어를 구현했습니다.

#### 액션 서버: `smolvla_action_server.py`

**액션 타입**: `catbot_interfaces/action/VlaTask`

**액션 이름**: `execute_vla_task`

**동작**:
- 자연어 프롬프트를 받아서 로봇팔 작업 수행
- 예: "Pick up the shovel from the right holder."
- LeRobot 기반 VLA (Vision-Language-Action) 모델 사용

**구독 토픽**:
- 카메라 이미지 토픽 (이미지 구독)

**발행 토픽**: 없음 (액션 피드백/결과만 사용)

---

### 2. 액션 클라이언트: `mission_controller.py`

**역할**: 액션 서버에 작업 요청

**사용 예시**:
```python
mission_controller = MissionController()
mission_controller.send_goal("Pick up the shovel from the right holder.")
```

---

### 3. 시나리오 실행: `run_toilet_scenario.py`

**역할**: 화장실 청소 시나리오를 순차적으로 실행

**작업 순서**:
1. "Pick up the shovel from the right holder."
2. "Scoop the brown snack from the center box using the shovel."
3. "Move the shovel to the left box and discard the snack."
4. "Return the shovel to the right holder."

## Pi Gateway 연동 방법

### 방법 1: ROS2 Action 클라이언트 사용 (권장)

Pi Gateway에서 로봇팔 액션 서버에 작업을 요청하는 방식입니다.

#### 구현 예시

```python
import rclpy
from rclpy.node import Node
from rclpy.action import ActionClient
from catbot_interfaces.action import VlaTask

class ArmActionClient(Node):
    def __init__(self):
        super().__init__('arm_action_client')
        self._action_client = ActionClient(self, VlaTask, 'execute_vla_task')
    
    def execute_task(self, task_prompt: str):
        """로봇팔 작업 실행"""
        # 서버 대기
        if not self._action_client.wait_for_server(timeout_sec=5.0):
            self.get_logger().error('로봇팔 액션 서버를 찾을 수 없습니다')
            return False
        
        # 목표 전송
        goal_msg = VlaTask.Goal()
        goal_msg.task_type = task_prompt
        
        send_goal_future = self._action_client.send_goal_async(goal_msg)
        rclpy.spin_until_future_complete(self, send_goal_future)
        
        goal_handle = send_goal_future.result()
        if not goal_handle.accepted:
            self.get_logger().error('작업이 거부되었습니다')
            return False
        
        # 결과 대기
        get_result_future = goal_handle.get_result_async()
        rclpy.spin_until_future_complete(self, get_result_future)
        
        result = get_result_future.result().result
        if result.success:
            self.get_logger().info(f'작업 완료: {result.message}')
            return True
        else:
            self.get_logger().error(f'작업 실패: {result.message}')
            return False
```

#### 사용 예시

```python
# Pi Gateway에서 로봇팔 작업 요청
arm_client = ArmActionClient()

# 삽 집기
arm_client.execute_task("Pick up the shovel from the right holder.")

# 똥 푸기
arm_client.execute_task("Scoop the brown snack from the center box using the shovel.")
```

### 방법 2: 토픽 기반 통신 (현재 구현)

현재 Pi Gateway의 `ArmControlBridge`는 토픽 기반 통신을 사용합니다.

**현재 구조**:
- Pi Gateway → `/arm/cmd` 토픽 발행
- UART → STM32 → 로봇팔 하드웨어 제어

**문제점**:
- 로봇팔 팀은 ROS2 Action을 사용
- 토픽 기반 통신과 불일치

**해결 방안**:
1. **방안 1**: Pi Gateway에 Action 클라이언트 추가 (권장)
2. **방안 2**: 로봇팔 팀에 토픽 기반 인터페이스 요청

## 작업 완료 신호 발행

### 현재: `/arm/done` 토픽 발행 필요

패트롤 노드가 `/arm/done` 토픽을 구독하여 작업 완료를 기다립니다.

**구현 방법**:

#### Action 클라이언트 사용 시

```python
class ArmActionClient(Node):
    def __init__(self):
        super().__init__('arm_action_client')
        self._action_client = ActionClient(self, VlaTask, 'execute_vla_task')
        # 작업 완료 신호 발행
        self.arm_done_pub = self.create_publisher(Bool, '/arm/done', 10)
    
    def execute_task(self, task_prompt: str):
        # ... 액션 요청 ...
        
        result = get_result_future.result().result
        if result.success:
            # 작업 완료 신호 발행
            done_msg = Bool()
            done_msg.data = True
            self.arm_done_pub.publish(done_msg)
            return True
```
