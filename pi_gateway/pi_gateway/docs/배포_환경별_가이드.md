# 배포 환경별 실행 가이드

## 개요

Pi Gateway 코드는 Raspberry Pi와 Jetson 두 환경에 배포됩니다. 각 환경에서 실행하는 컴포넌트가 다릅니다.

## 환경별 역할

### Raspberry Pi
- **Pi Gateway 메인 서버**: 웹 대시보드, ROS2 브릿지, UART 통신, WebRTC 시그널링
- **MQTT 브릿지**: ROS2 ↔ MQTT 변환
- **UART 디바이스 접근**: `/dev/ttyUSB0` (STM32 하드웨어 제어)

### Jetson
- **고양이 감지 서비스**: `/front_cam/compressed` 토픽 구독, 고양이 감지 및 상태 분석
- **ROS2 토픽 발행**: 고양이 감지 결과를 백엔드로 전송

## Raspberry Pi 배포

### 방법 1: Docker Compose 사용 (권장)

**장점**:
- 의존성 자동 관리
- 재시작 정책 설정 가능
- 로그 관리 용이

**실행**:
```bash
cd /home/c203/pi_gateway
docker-compose up -d
```

**docker-compose.yml 사용**: ✅ **사용**

### 방법 2: 로컬 실행

```bash
cd /home/c203/pi_gateway
source /opt/ros/humble/setup.bash
export ROS_DOMAIN_ID=1
export UART_ENABLED=1
export ROS_ENABLED=1
python3 -m src.main
```

## Jetson 배포

### 방법 1: 로컬 실행 (권장)

**이유**:
- Jetson은 GPU 사용 (CUDA, TensorRT)
- 도커에서 GPU 접근 설정이 복잡함
- 고양이 감지 서비스만 실행하므로 단순함

**실행**:
```bash
cd /home/c203/pi_gateway
source /opt/ros/humble/setup.bash
export ROS_DOMAIN_ID=1
export BE_WS_URL=wss://i14c203.p.ssafy.io/ws
python3 scripts/cat_detection_service.py \
  --ckpt models/swin_tiny_best/best \
  --yolo_pose models/yolo_pose-gy8961dp9tbgbxw9xekwdg9ome/best
```

**docker-compose.yml 사용**: ❌ **불필요** (로컬 실행 권장)

### 방법 2: Docker Compose 사용 (선택사항)

만약 Jetson에서도 도커를 사용하고 싶다면:

**주의사항**:
- NVIDIA Container Toolkit 설치 필요
- GPU 접근 설정 필요 (`--gpus all`)
- CUDA 라이브러리 마운트 필요

**docker-compose.yml 예시** (Jetson용):
```yaml
version: '3.8'

services:
  cat-detection:
    build:
      context: .
      dockerfile: Dockerfile.jetson  # Jetson 전용 Dockerfile 필요
    container_name: cat-detection
    network_mode: host
    environment:
      - ROS_DOMAIN_ID=1
      - ROS_DISTRO=humble
      - BE_WS_URL=wss://i14c203.p.ssafy.io/ws
    volumes:
      - ./models:/app/models:ro
      - ./scripts:/app/scripts:ro
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]
    command: python3 scripts/cat_detection_service.py --ckpt models/swin_tiny_best/best --yolo_pose models/yolo_pose-gy8961dp9tbgbxw9xekwdg9ome/best
```

**권장**: Jetson에서는 로컬 실행을 권장합니다.

## FAQ

### Q: Jetson에서도 docker-compose.yml을 사용해야 하나요?

**A**: 아니요. Jetson에서는 로컬 실행을 권장합니다.
- GPU 접근이 복잡함
- 고양이 감지 서비스만 실행하므로 단순함
- CUDA/TensorRT 라이브러리 접근이 도커에서 복잡함

### Q: 같은 폴더를 사용해도 되나요?

**A**: 네, 문제없습니다.
- Raspberry Pi: `docker-compose up` 실행
- Jetson: `python3 scripts/cat_detection_service.py` 실행
- 각각 다른 컴포넌트를 실행하므로 충돌 없음

### Q: Jetson에서 docker-compose를 사용하고 싶다면?

**A**: 가능하지만 추가 설정이 필요합니다:
1. NVIDIA Container Toolkit 설치
2. Jetson 전용 Dockerfile 생성 (CUDA 라이브러리 포함)
3. `--gpus all` 옵션 설정

하지만 **로컬 실행을 권장**합니다.
