# 작업 완료 신호 흐름 정리

각 작업별로 완료 신호가 어떻게 전달되는지 정리합니다.

---

## 작업별 완료 신호 흐름

### 1. 급식 (FEED) - STM32가 직접 제어 ✅

```
Pi → STM32: FEED 명령 (level=1~3)
STM32: 서보모터 제어 → 목표 위치 도달
STM32: 완료 감지 (서보모터 위치 피드백)
STM32 → Pi: STATUS(0x01, 0x01, 0x00)  # JOB_COMPLETE, 급식
Pi: 작업 이벤트 업데이트
```

**완료 감지:** STM32가 서보모터 위치 피드백으로 자동 감지

---

### 2. 츄르 (CHURU) - STM32가 직접 제어 ✅

```
Pi → STM32: CHURU 명령 (enable=1)
STM32: 서보모터 제어 → 주사기 밀기 완료
STM32: 완료 감지 (서보모터 위치 피드백)
STM32 → Pi: STATUS(0x01, 0x04, 0x00)  # JOB_COMPLETE, 츄르
Pi: 작업 이벤트 업데이트
```

**완료 감지:** STM32가 서보모터 위치 피드백으로 자동 감지

---

### 3. 변 치우기 (LITTER_CLEAN) - 젯슨이 제어 ❌

**문제:** 젯슨이 로봇팔을 제어하므로 STM32는 완료를 모름

**해결 방법:**

#### 방법 A: 젯슨이 ROS 토픽으로 완료 신호 발행 (권장)

```
젯슨: 로봇팔로 변 치우기 작업 수행
젯슨: 작업 완료 판단 (모든 구역 처리 완료, 로봇팔 원위치)
젯슨 → Pi: arm/job_complete 토픽 발행
  {"job_type": "litter_clean", "status": "success"}
Pi (ArmControlBridge): 토픽 구독 → 작업 이벤트 업데이트
```

**젯슨 코드 예시:**
```python
# 젯슨에서
from std_msgs.msg import String
import json

pub_job_complete = node.create_publisher(String, "arm/job_complete", 10)

# 변 치우기 완료 시
msg = String()
msg.data = json.dumps({
    "job_type": "litter_clean",
    "status": "success"
})
pub_job_complete.publish(msg)
```

#### 방법 B: 젯슨이 STM32에 직접 신호 (대안)

```
젯슨: 작업 완료 판단
젯슨 → STM32: 완료 신호 (GPIO 또는 UART)
STM32 → Pi: STATUS(0x01, 0x02, 0x00)  # JOB_COMPLETE, 변치우기
Pi: 작업 이벤트 업데이트
```

**현재 구현:** 방법 A (ROS 토픽) 사용

---

### 4. 급수 (WATER) - 젯슨이 제어 ❌

**문제:** 젯슨이 로봇팔을 제어하므로 STM32는 완료를 모름

**해결 방법:**

#### 방법 A: 젯슨이 ROS 토픽으로 완료 신호 발행 (권장)

```
젯슨: 로봇팔로 급수 작업 수행 (물그릇 집기 → 버리기 → 받기 → 두기)
젯슨: 전체 급수 작업 완료 판단
젯슨 → Pi: arm/job_complete 토픽 발행
  {"job_type": "water", "status": "success"}
Pi (ArmControlBridge): 토픽 구독 → 작업 이벤트 업데이트
```

**젯슨 코드 예시:**
```python
# 급수 완료 시
msg = String()
msg.data = json.dumps({
    "job_type": "water",
    "status": "success"
})
pub_job_complete.publish(msg)
```

**현재 구현:** 방법 A (ROS 토픽) 사용

## 요약

| 작업 | 제어 주체 | 완료 신호 발신자 | 신호 경로 |
|------|----------|------------------|-----------|
| **급식** | STM32 | STM32 | STM32 → Pi (STATUS) |
| **츄르** | STM32 | STM32 | STM32 → Pi (STATUS) |
| **변 치우기** | 젯슨 | 젯슨 | 젯슨 → Pi (ROS 토픽 `arm/job_complete`) |
| **급수** | 젯슨 | 젯슨 | 젯슨 → Pi (ROS 토픽 `arm/job_complete`) |

---

## 젯슨 담당자에게 전달할 내용

**변 치우기/급수 완료 시 ROS 토픽 발행:**

```python
# 젯슨 코드에 추가
from std_msgs.msg import String
import json

pub_job_complete = node.create_publisher(String, "arm/job_complete", 10)

# 변 치우기 완료 시
def on_litter_clean_complete():
    msg = String()
    msg.data = json.dumps({
        "job_type": "litter_clean",
        "status": "success"
    })
    pub_job_complete.publish(msg)

# 급수 완료 시
def on_water_complete():
    msg = String()
    msg.data = json.dumps({
        "job_type": "water",
        "status": "success"
    })
    pub_job_complete.publish(msg)

# 실패 시
def on_job_failed(job_type: str, reason: str):
    msg = String()
    msg.data = json.dumps({
        "job_type": job_type,  # "litter_clean" 또는 "water"
        "status": "failed",
        "reason": reason
    })
    pub_job_complete.publish(msg)
```

**Pi Gateway가 자동으로:**
- `arm/job_complete` 토픽 구독
- 작업 이벤트 업데이트 (`JOB_EVENTS.update_job_by_type()`)
- Backend에 작업 완료 로그 전송
- 웹 API `/robot/jobs`에서 완료 상태 확인 가능

---

## 현재 코드 상태

✅ **구현 완료:**
- `ArmControlBridge`에 `arm/job_complete` 토픽 구독 추가
- 작업 이벤트 콜백 연결
- 젯슨에서 토픽 발행하면 자동 처리

⚠️ **젯슨에서 해야 할 일:**
- 변 치우기/급수 완료 시 `arm/job_complete` 토픽 발행
- 메시지 형식: `{"job_type": "litter_clean"|"water", "status": "success"|"failed"}`
