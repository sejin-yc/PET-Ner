# 엔코더 x4 디코딩 확인 가이드

STM32가 TIM Encoder mode를 사용하는지, x4 디코딩을 적용하는지 확인하는 방법입니다.

---

## 🔍 확인 방법

### 방법 1: STM32 코드 확인 (가장 확실함)

STM32 코드에서 엔코더 초기화 부분을 확인하세요:

```c
// STM32 HAL 라이브러리 사용 시
HAL_TIM_Encoder_Init(&htim2, &sConfig);  // TIM Encoder mode 사용

// 또는 직접 레지스터 설정 시
TIM2->SMCR |= TIM_SMCR_SMS_1 | TIM_SMCR_SMS_0;  // Encoder mode 3 (x4)
```

**확인 포인트:**
- `HAL_TIM_Encoder_Init()` 사용 → TIM Encoder mode
- `TIM_SMCR_SMS` 레지스터 값:
  - `SMS = 1` (0b001): x1 디코딩 (A 채널만)
  - `SMS = 2` (0b010): x2 디코딩 (A 채널 rising/falling)
  - `SMS = 3` (0b011): **x4 디코딩** (A, B 채널 모두 rising/falling) ← **가장 흔함**

---

### 방법 2: 실측 보정 (현업에서도 사용)

**가장 정확한 방법입니다!**

1. 로봇을 정확히 1바퀴 굴립니다 (또는 N바퀴)
2. 엔코더 틱 변화량을 측정합니다
3. 계산합니다:

```
meters_per_tick = (π * D * N) / Δtick

예시:
- 바퀴 지름 D = 0.097m
- 1바퀴 굴림 (N = 1)
- 틱 변화량 Δtick = 2508
- meters_per_tick = (π * 0.097 * 1) / 2508 = 0.0001215 m/tick
```

**x4 디코딩 여부 판단:**
- 이론값 (x1): `(π * 0.097) / (11 * 57) = 627 tick/바퀴`
- 이론값 (x4): `(π * 0.097) / (11 * 57 * 4) = 2508 tick/바퀴`

**실측값이 2508에 가까우면 → x4 디코딩 사용 중**

---

### 방법 3: STM32 담당자에게 확인

**질문 예시:**
```
안녕하세요! Pi Gateway 담당입니다.

엔코더 오도메트리 계산을 위해 확인이 필요합니다:

1. STM32에서 엔코더를 읽을 때 TIM Encoder mode를 사용하시나요?
2. 만약 사용한다면, Encoder mode 3 (x4 디코딩)을 사용하시나요?
3. 아니면 다른 방식으로 엔코더를 읽으시나요?

현재 params.yaml에 설정된 값:
- meters_per_tick: 4.86e-4
- 계산식: (π × 0.097) / (11 × 57)

만약 x4 디코딩을 사용한다면:
- meters_per_tick: 1.215e-4
- 계산식: (π × 0.097) / (11 × 57 × 4)

확인 부탁드립니다!
```

## 📊 일반적인 STM32 엔코더 설정

### TIM Encoder Mode (가장 흔함)

STM32에서 엔코더를 읽을 때 가장 흔한 방법:

```c
// CubeMX에서 설정하거나 HAL 라이브러리 사용
TIM_Encoder_InitTypeDef sEncoderConfig;
sEncoderConfig.EncoderMode = TIM_ENCODERMODE_TI12;  // x4 디코딩
HAL_TIM_Encoder_Init(&htim2, &sEncoderConfig);
```

**특징:**
- A, B 채널 모두 사용
- Rising edge와 falling edge 모두 카운트
- **결과: PPR × 4 = 실제 틱 수**

### 다른 방식들

1. **외부 인터럽트 (x1 또는 x2)**
   - A 채널만 사용 → x1
   - A 채널 rising/falling → x2

2. **직접 GPIO 읽기**
   - 소프트웨어로 디코딩
   - 보통 x4 구현

## ✅ 현재 프로젝트 상황

### PinMapping.md 확인 결과

```
| Motor | Encoder A | Encoder B | Timer |
| FL    | PA0       | PA1       | TIM2  |
| FR    | PB6       | PB7       | TIM4  |
| RL    | PA15      | PB3       | TIM2  |
| RR    | PC6       | PC11      | TIM3  |
```

**관찰:**
- A, B 채널 모두 사용 중
- TIM2, TIM3, TIM4 사용
- **→ TIM Encoder mode 사용 가능성 높음**

### 결론

**확률적으로 x4 디코딩을 사용할 가능성이 높습니다.**

이유:
1. A, B 채널 모두 연결되어 있음
2. STM32 TIM Encoder mode는 기본적으로 x4 디코딩
3. 대부분의 로봇 프로젝트에서 x4 사용

**하지만 확실히 하려면:**
- STM32 코드 확인 또는
- 실측 보정 권장

## 🔧 수정 권장사항

### 현재 설정 (x4 미고려)
```yaml
encoder:
  meters_per_tick: 4.86e-4   # (π × 0.097) / (11 × 57)
```

### 수정 후 (x4 고려)
```yaml
encoder:
  meters_per_tick: 1.215e-4   # (π × 0.097) / (11 × 57 × 4)
```

**실측 후 정확한 값으로 보정하는 것을 권장합니다!**

## 📝 실측 보정 예시

```python
# 테스트 코드 예시
import time
from src.uart_link import UartLink
from src.uart_frames import decode_telemetry

uart = UartLink(...)
uart.open()

# 초기 틱 읽기
initial_ticks = None
def on_frame(msg_id, payload):
    if msg_id == 0x82:  # ENCODER
        obj = decode_telemetry(msg_id, payload)
        return obj["enc_fl"], obj["enc_fr"], obj["enc_rl"], obj["enc_rr"]

# 로봇을 정확히 1바퀴 굴림 (수동으로)
# ...

# 최종 틱 읽기
final_ticks = on_frame(...)

# 계산
delta_tick = final_ticks[0] - initial_ticks[0]  # FL 휠 기준
wheel_diameter = 0.097  # m
meters_per_tick = (3.14159 * wheel_diameter * 1) / delta_tick

print(f"실측 meters_per_tick: {meters_per_tick}")
```
