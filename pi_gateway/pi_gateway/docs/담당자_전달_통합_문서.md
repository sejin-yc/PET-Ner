# Pi Gateway 연동 - 담당자 전달 통합 문서

**작성일**: 2026.02  
**목적**: Pi Gateway 구현 완료 항목 및 백엔드·프론트 담당자 체크리스트

---

## 1. Pi Gateway 구현 완료 항목

| 항목 | 구현 | 전송 방식 | 환경 변수 |
|------|------|-----------|-----------|
| **로봇 상태·제어** | ws_pi_bridge | WebSocket (SEND /pub/robot/status, SUB /sub/robot/{userId}/control) | BE_WS_URL, PI_GATEWAY_URL, BE_USER_ID |
| **고양이 감지** | cat_detection_service | WebSocket(1) > MQTT(2) > HTTP(3) | BE_WS_URL, BE_SERVER_URL, BE_USER_ID |
| **순찰 시간** | PatrolLoop + JobEventCollector | HTTP POST /api/logs | BE_SERVER_URL, BE_USER_ID |

## 2. 백엔드 담당자 체크리스트

### 2.1 WebSocket 연동 (1883 막혀 있음 → WebSocket 443 사용)

| # | 항목 | 상세 |
|---|------|------|
| 1 | @MessageMapping("/robot/status") | Pi 상태 수신 → RobotStatus/RobotPose 저장, convertAndSend("/sub/robot/status") |
| 2 | handleControl 수정 | convertAndSend("/sub/robot/{userId}/control") 추가 (Pi 제어 수신용) |
| 3 | @MessageMapping("/robot/cat_state") | 고양이 감지 수신 → Log 저장, convertAndSend("/sub/robot/{userId}/cat_state") |
| 4 | Log 엔티티 | userId, mode, status, duration, durationNum, distance, detectionCount, details |

**참고**: `docs/WebSocket_로봇연동_백엔드_전달_문서.md`

### 2.2 순찰 시간 (POST /api/logs)

| # | 항목 | 상세 |
|---|------|------|
| 1 | POST /api/logs | **이미 존재** (LogController). Pi가 순찰 완료 시 Log 형식으로 전송. **추가 수정 없음** |
| 2 | Log 형식 | userId, mode, status, duration, durationNum(분), distance, detectionCount, details |

## 3. 프론트 담당자 체크리스트

| # | 항목 | 상세 |
|---|------|------|
| 1 | SUBSCRIBE /sub/robot/status | 배터리·위치·모드 표시 (기존 유지) |
| 2 | SUBSCRIBE /sub/robot/{userId}/cat_state | 고양이 감지 시 갤러리 갱신 |
| 3 | SEND /pub/robot/control | 수동 조작 (기존 유지) |
| 4 | GET /logs?userId= | 순찰 로그·주간 순찰 시간 (기존 유지) |

**참고**: `docs/백엔드_프론트_담당자_전달_문서.md`
