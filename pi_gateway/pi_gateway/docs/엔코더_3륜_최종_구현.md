# 엔코더 3륜 최종 구현 (SLAM+nav2 최적화)

왼쪽 앞바퀴(fl) 고장 시 SLAM+nav2 환경에 최적화된 구현입니다.

---

## ✅ 최종 구현 요약

### 핵심 원칙

1. **Pseudo FL로 4륜식 유지**
   - `v_fl = v_rr` 또는 `v_fl = 0.5 * (v_rr + v_fr)`
   - 4륜식 공식 유지로 안정성 확보

2. **vx, vy: 엔코더 기반 + EMA**
   - 엔코더로 계산 후 EMA 필터링
   - `alpha = 0.6~0.75` (nav2 지연 생기면 올리기)

3. **yaw/wz: IMU 우선**
   - `yaw`: IMU에서 가져오기
   - `wz`: IMU `gyro_z` 우선 사용 (가능하면)

---

## 📐 구현 상세

### 1. Pseudo FL 추정

```python
# 방법 1: 대칭 위치 가정 (기본값)
v_fl = v_rr

# 방법 2: 앞바퀴 평균
v_fl = 0.5 * (v_rr + v_fr)
```

**설정:**
```python
fl_estimation_method = "rr"        # v_fl = v_rr
fl_estimation_method = "fr_rr_avg" # v_fl = 0.5 * (v_rr + v_fr)
```

### 2. vx, vy 계산 (엔코더 기반 + EMA)

```python
# 4륜식 공식 유지
vx_raw = (v_fl_est + v_fr + v_rl + v_rr) / 4.0
vy_raw = (-v_fl_est + v_fr - v_rl + v_rr) / 4.0

# EMA 필터링
vx = alpha * vx_raw + (1 - alpha) * vx_prev
vy = alpha * vy_raw + (1 - alpha) * vy_prev
```

**파라미터:**
- `alpha = 0.6~0.75` (권장)
- nav2 지연 생기면 `alpha` 올리기 (예: 0.8)

### 3. yaw/wz: IMU 우선

```python
# yaw: IMU 우선
if imu_yaw is not None:
    yaw = imu_yaw  # IMU 사용
else:
    yaw = encoder_yaw  # fallback

# wz: IMU gyro_z 우선 (가능하면)
if imu_gyro_z is not None:
    wz = imu_gyro_z  # IMU 사용
else:
    wz = encoder_wz  # fallback
```

## 🔧 파라미터 설정

### 기본 설정 (권장)

```python
EncoderOdomNode(
    use_imu_yaw=True,           # IMU yaw 사용
    use_imu_gyro=True,          # IMU gyro_z 사용
    encoder_filter_alpha=0.7,    # EMA 필터링 (0.6~0.75)
    fl_estimation_method="rr"   # v_fl = v_rr
)
```

### nav2 지연 발생 시

```python
encoder_filter_alpha=0.8  # 필터 약화 (더 빠른 반응)
```

### 앞바퀴 평균 사용 시

```python
fl_estimation_method="fr_rr_avg"  # v_fl = 0.5 * (v_rr + v_fr)
```

## 📊 데이터 흐름

```
[STM32] → UART → [ros_telemetry_bridge]
                          ↓
              [EncoderOdomNode]
                  ↓         ↓
            [엔코더]    [IMU]
            fr,rl,rr    yaw,gyro_z
                  ↓         ↓
            vx,vy 계산    yaw,wz 사용
                  ↓         ↓
            EMA 필터링    직접 사용
                  ↓         ↓
              /odom 발행
                  ↓
        [robot_localization EKF]
                  ↓
        /odometry/filtered
                  ↓
            [SLAM/nav2]
```

## 💡 핵심 포인트

1. **Pseudo FL**: 4륜식 유지로 안정성 확보
2. **EMA**: 엔코더 노이즈 제거, nav2 지연 고려
3. **IMU 우선**: yaw/wz는 IMU에서 가져오기
4. **공분산**: EKF가 불확실성을 고려하도록 설정

---

## 📝 참고

- **vx, vy**: 엔코더 기반 + EMA (필터링)
- **yaw, wz**: IMU 우선 (필터링 없음, 직접 사용)
- **FL 추정**: 4륜식 공식 유지
- **alpha**: nav2 지연 생기면 올리기
