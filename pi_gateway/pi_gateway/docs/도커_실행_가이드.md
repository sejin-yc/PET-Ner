# Pi Gateway 도커 실행 가이드

## 개요

Pi Gateway 메인 서버를 도커 컨테이너로 실행하는 방법입니다.

## 사전 요구사항

- Docker 설치
- Docker Compose 설치 (선택사항, 권장)
- UART 디바이스 접근 권한 (`/dev/ttyUSB0`)

## 빠른 시작

### 1. 도커 이미지 빌드

```bash
cd /home/c203/pi_gateway
docker build -t pi-gateway:latest .
```

### 2. 도커 컴포즈로 실행

```bash
docker-compose up -d
```

### 3. 로그 확인

```bash
docker logs -f pi-gateway
```

## 상세 가이드

### Dockerfile 구조

```
FROM ros:humble-desktop          # ROS2 Humble 베이스 이미지
WORKDIR /app                     # 작업 디렉토리
COPY requirements.txt            # Python 의존성 설치
COPY src/ config/ scripts/       # 소스 코드 복사
CMD ["python3", "-m", "src.main"] # 실행 명령
```

### docker-compose.yml 설정

**주요 설정**:
- `network_mode: host`: ROS2 DDS 통신을 위해 호스트 네트워크 사용
- `devices: /dev/ttyUSB0`: UART 디바이스 마운트
- `environment`: 환경 변수 설정

### 환경 변수

| 변수명 | 기본값 | 설명 |
|--------|--------|------|
| `ROS_DOMAIN_ID` | `1` | ROS2 도메인 ID |
| `UART_ENABLED` | `1` | UART 통신 활성화 |
| `ROS_ENABLED` | `1` | ROS2 모드 활성화 |
| `WEBRTC_SIGNALING_ENABLED` | `1` | WebRTC 시그널링 클라이언트 활성화 |
| `UART_PORT` | `/dev/ttyUSB0` | UART 포트 경로 |
| `UART_BAUD` | `115200` | UART 보드레이트 |
| `HOST` | `0.0.0.0` | 웹 서버 호스트 |
| `PORT` | `8000` | 웹 서버 포트 |
| `BE_WS_URL` | `wss://i14c203.p.ssafy.io/ws` | WebRTC 시그널링 서버 URL |

### 실행 방법

#### 방법 1: Docker Compose 사용 (권장)

```bash
# 빌드 및 실행
docker-compose up -d

# 로그 확인
docker-compose logs -f

# 중지
docker-compose down

# 재시작
docker-compose restart
```

#### 방법 2: Docker 직접 실행

```bash
# 이미지 빌드
docker build -t pi-gateway:latest .

# 컨테이너 실행
docker run -d \
  --name pi-gateway \
  --network host \
  --device /dev/ttyUSB0:/dev/ttyUSB0 \
  -e ROS_DOMAIN_ID=1 \
  -e UART_ENABLED=1 \
  -e ROS_ENABLED=1 \
  -e WEBRTC_SIGNALING_ENABLED=1 \
  -e UART_PORT=/dev/ttyUSB0 \
  -e UART_BAUD=115200 \
  -e BE_WS_URL=wss://i14c203.p.ssafy.io/ws \
  -v $(pwd)/config:/app/config:ro \
  pi-gateway:latest

# 로그 확인
docker logs -f pi-gateway

# 중지
docker stop pi-gateway

# 삭제
docker rm pi-gateway
```

## UART 디바이스 설정

### 디바이스 확인

```bash
# UART 디바이스 확인
ls -l /dev/ttyUSB*

# 권한 확인
ls -l /dev/ttyUSB0
```

### 권한 설정

```bash
# 현재 사용자를 dialout 그룹에 추가
sudo usermod -a -G dialout $USER

# 재로그인 필요
# 또는
sudo chmod 666 /dev/ttyUSB0
```

### docker-compose.yml에서 디바이스 경로 변경

UART 디바이스가 `/dev/ttyUSB1`인 경우:

```yaml
devices:
  - /dev/ttyUSB1:/dev/ttyUSB1
```

환경 변수도 함께 변경:

```yaml
environment:
  - UART_PORT=/dev/ttyUSB1
```

## 네트워크 설정

### ROS2 DDS 통신

**중요**: ROS2 DDS는 멀티캐스트를 사용하므로 `network_mode: host`가 필요합니다.

```yaml
network_mode: host
```

### 포트 매핑 (host 모드 사용 시 불필요)

`network_mode: host`를 사용하면 포트 매핑이 필요 없습니다. 호스트의 포트를 직접 사용합니다.

만약 bridge 모드를 사용해야 한다면:

```yaml
ports:
  - "8000:8000"
```

하지만 ROS2 통신이 제대로 작동하지 않을 수 있으므로 **host 모드를 권장**합니다.

## 로그 관리

### 로그 확인

```bash
# 실시간 로그
docker logs -f pi-gateway

# 최근 100줄
docker logs --tail 100 pi-gateway

# 특정 시간 이후 로그
docker logs --since 10m pi-gateway

# 로그 파일로 저장
docker logs pi-gateway > gateway.log 2>&1
```

### 로그 설정 (docker-compose.yml)

```yaml
logging:
  driver: "json-file"
  options:
    max-size: "10m"    # 최대 파일 크기
    max-file: "3"      # 최대 파일 개수
```

## 문제 해결

### 문제 1: UART 디바이스 접근 불가

**증상**: `Permission denied: /dev/ttyUSB0`

**해결책**:
```bash
# 권한 확인
ls -l /dev/ttyUSB0

# 권한 변경
sudo chmod 666 /dev/ttyUSB0

# 또는 사용자를 dialout 그룹에 추가
sudo usermod -a -G dialout $USER
# 재로그인 필요
```

### 문제 2: ROS2 통신 불가

**증상**: 다른 ROS2 노드와 통신이 안 됨

**해결책**:
- `network_mode: host` 사용 확인
- `ROS_DOMAIN_ID` 환경 변수 확인 (호스트와 동일해야 함)
- 방화벽 설정 확인

### 문제 3: 컨테이너가 즉시 종료됨

**증상**: `docker ps`에 컨테이너가 없음

**해결책**:
```bash
# 로그 확인
docker logs pi-gateway

# 재시작 정책 확인
docker run --restart unless-stopped ...
```

### 문제 4: 이미지 빌드 실패

**증상**: `docker build` 중 에러 발생

**해결책**:
```bash
# 캐시 없이 빌드
docker build --no-cache -t pi-gateway:latest .

# 단계별 빌드 확인
docker build --progress=plain -t pi-gateway:latest .
```

## 프로덕션 배포

### 최적화된 Dockerfile 예시

```dockerfile
# 멀티 스테이지 빌드 사용

WORKDIR /app
COPY requirements.txt .
RUN pip3 install --user --no-cache-dir -r requirements.txt

FROM ros:humble-desktop
WORKDIR /app
COPY --from=builder /root/.local /root/.local
COPY src/ ./src/
COPY config/ ./config/
ENV PATH=/root/.local/bin:$PATH
CMD ["python3", "-m", "src.main"]
```
