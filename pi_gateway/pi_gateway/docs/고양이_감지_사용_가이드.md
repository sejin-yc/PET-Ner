# 고양이 감지 기능 사용 가이드

패트롤 중 USB 웹캠으로 고양이를 감지하여 로봇을 자동으로 정지시키는 기능 사용 가이드입니다.

---

## 🚀 빠른 시작

### 1. 모델 준비

catstate 모델 체크포인트 파일을 준비합니다:
- 기본 경로: `/home/ssafy/Downloads/S14P11C203-catstate/src/runs_stage3/.../best.pt`
- 또는 `config/params.yaml`의 `cat_detection.model_path`에 경로 지정

### 2. 설정 활성화

`config/params.yaml` 파일 수정:

```yaml
cat_detection:
  enabled: true                    # 고양이 감지 활성화
  model_path: ""                   # 비어있으면 기본 경로 시도
  camera_device: 0                  # USB 웹캠 디바이스 번호
  confidence_threshold: 0.5         # 고양이 감지 confidence 임계값
  inference_fps: 2.0               # 추론 주기 (FPS)
  use_ros_image: false             # ROS2 이미지 토픽 사용 여부
  detection_timeout: 5.0           # 고양이 사라진 후 재개까지 대기 시간 (초)
```

또는 환경변수로 활성화:

```bash
export CAT_DETECTION_ENABLED=1
python3 -m src.main
```

### 3. 실행

```bash
python3 -m src.main
```

## 🔧 동작 원리

### 1. 이미지 캡처

**USB 웹캠 직접 사용** (`use_ros_image=false`):
- OpenCV로 웹캠에서 직접 이미지 캡처
- `camera_device` 번호로 웹캠 선택

**ROS2 이미지 토픽 사용** (`use_ros_image=true`):
- `/camera/image_raw` 토픽 구독
- 다른 노드가 이미지를 발행하는 경우 사용

### 2. 고양이 감지

1. 이미지 전처리 (리사이즈, 정규화)
2. catstate 모델로 추론
3. Action과 Emotion의 confidence 확인
4. 임계값 이상이면 고양이 존재로 판단

### 3. 로봇 제어

**고양이 감지 시**:
- `cat_detection/cmd_vel_stop` 토픽에 `Twist(0,0,0)` 발행
- `CmdVelMux`가 이를 최우선으로 처리하여 로봇 정지

**고양이 사라짐**:
- `detection_timeout` 시간 동안 고양이가 감지되지 않으면
- 정지 명령 해제 → 패트롤 재개

## 🔄 통신 흐름

```
USB 웹캠
  ↓ 이미지 캡처
CatDetectionNode
  ↓ catstate 모델 추론
  ↓ 고양이 감지 여부 판단
  ↓
  ├─ 고양이 감지 → cat_detection/cmd_vel_stop 발행 (0,0,0)
  └─ 고양이 없음 → 정지 명령 해제
       ↓
CmdVelMux (우선순위: E-STOP > 고양이 감지 > mode)
  ↓ cmd_vel_out 발행
RosCmdVelBridge
  ↓ UART 전송
STM32 (모터 정지)
```

---

## ⚙️ 우선순위

`CmdVelMux`의 명령 우선순위:

1. **E-STOP** (최우선)
2. **고양이 감지 정지** (`cat_detection/cmd_vel_stop`)
3. **mode에 따른 명령** (`auto` 또는 `teleop`)

## 🐛 문제 해결

### 모델을 찾을 수 없음

**에러**: `Model checkpoint not found`

**해결**:
1. `config/params.yaml`의 `cat_detection.model_path`에 정확한 경로 지정
2. 또는 기본 경로에 모델 파일 확인:
   ```
   /home/ssafy/Downloads/S14P11C203-catstate/src/runs_stage3/.../best.pt
   ```

### 웹캠을 열 수 없음

**에러**: `Failed to open camera device`

**해결**:
1. 웹캠이 연결되어 있는지 확인: `ls /dev/video*`
2. `camera_device` 번호 확인 (0, 1, 2 등)
3. 권한 확인: `sudo chmod 666 /dev/video0`

### 추론이 너무 느림

**증상**: CPU 사용률이 높고 프레임이 느림

**해결**:
1. `inference_fps`를 낮춤 (예: 1.0 또는 0.5)
2. GPU 사용 (Jetson의 경우)
3. 모델 경량화 (TensorRT 변환 등)

### False Positive (오탐)

**증상**: 고양이가 없는데도 감지됨

**해결**:
1. `confidence_threshold`를 높임 (예: 0.7 또는 0.8)
2. 연속 프레임 확인 로직 추가 (향후 개선)

- `confidence_threshold` 튜닝
- 여러 프레임의 결과를 평균 (향후 개선)
