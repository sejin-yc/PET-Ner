# 고양이 감지와 실시간 스트리밍 동작 설명

## 개요

로봇 카메라는 두 가지 용도로 사용됩니다:
1. **고양이 감지 서비스**: 지속적으로 실행되어 고양이를 탐지하고 갤러리에 저장
2. **실시간 스트리밍**: 사용자가 버튼을 눌렀을 때만 WebRTC로 실시간 영상 송출

## 핵심 답변

**Q: 실시간 스트리밍 중에도 고양이가 탐지되면 어떻게 되나요?**

**A: 별개로 동작합니다. 실시간 스트리밍 중에도 고양이 감지는 계속 작동하며, 탐지되면 갤러리에 저장되고 알림이 뜹니다.**

---

## 동작 구조

### ROS2 토픽 구조

```
카메라 노드 (젯슨)
    ↓
/front_cam/compressed 토픽 발행
    ↓
    ├─→ 고양이 감지 서비스 (Jetson) ─→ 웹 갤러리 저장 + 알림
    │
    └─→ WebRTC 스트리밍 (robot_webrtc.py) ─→ 실시간 영상 송출
```

### 핵심 원리

**ROS2 토픽의 특성**:
- 하나의 토픽에 여러 구독자가 동시에 구독 가능
- 각 구독자는 독립적으로 메시지를 받음
- 서로 영향을 주지 않음 (독립 실행)

## 시나리오별 동작

### 시나리오 1: 평상시 (실시간 스트리밍 OFF)

**동작**:
1. 고양이 감지 서비스가 `/front_cam/compressed` 토픽을 계속 구독
2. 고양이 탐지 → 웹 갤러리에 저장
3. 탐지 결과 알림 표시

**WebRTC 스트리밍**: 실행되지 않음

---

### 시나리오 2: 실시간 스트리밍 ON (고양이 없음)

**동작**:
1. 사용자가 "영상 시작" 버튼 클릭
2. Pi Gateway가 `robot_webrtc.py` 시작
3. `robot_webrtc.py`가 `/front_cam/compressed` 토픽 구독 시작
4. WebRTC로 실시간 영상 송출
5. **고양이 감지 서비스는 계속 실행 중** (별도 프로세스)

**결과**:
- 실시간 영상이 프론트엔드에 표시됨
- 고양이 감지 서비스는 계속 작동 중 (단, 고양이가 없으므로 저장/알림 없음)

---

### 시나리오 3: 실시간 스트리밍 ON + 고양이 탐지 ⭐

**동작**:
1. 실시간 스트리밍이 진행 중
2. `robot_webrtc.py`가 영상을 WebRTC로 스트리밍
3. **동시에** 고양이 감지 서비스가 같은 토픽을 구독 중
4. 고양이 탐지 발생
5. **두 가지가 동시에 발생**:
   - 실시간 스트리밍: 계속 진행 (영상이 계속 송출됨)
   - 고양이 감지: 웹 갤러리에 저장 + 알림 표시

**결과**:
- ✅ 실시간 영상은 계속 송출됨
- ✅ 고양이 감지 결과가 갤러리에 저장됨
- ✅ 알림이 표시됨
- ✅ **별개로 동작하며 서로 영향을 주지 않음**

## 코드 구조

### 고양이 감지 서비스 (Jetson)

```python
# scripts/cat_detection_service.py
class CatDetectionService(Node):
    def __init__(self):
        # /front_cam/compressed 토픽 구독
        self.subscription = self.create_subscription(
            CompressedImage,
            '/front_cam/compressed',
            self.image_callback,
            qos_profile
        )
    
    def image_callback(self, msg):
        # 고양이 감지 로직
        if cat_detected:
            # 갤러리에 저장
            save_to_gallery(frame)
            # 알림 전송
            send_notification(result)
```

### WebRTC 스트리밍 (robot_webrtc.py)

```python
# scripts/robot_webrtc.py
class ImageSubscriber(Node):
    def __init__(self):
        # /front_cam/compressed 토픽 구독
        self.subscription = self.create_subscription(
            CompressedImage,
            '/front_cam/compressed',
            self.listener_callback,
            qos_profile
        )
    
    def listener_callback(self, msg):
        # WebRTC 스트리밍용 프레임 저장
        self.latest_frame = frame
```

**중요**: 둘 다 같은 토픽을 구독하지만, **독립적으로 동작**합니다.

## 실행 프로세스

### 프로세스 구조

```
Jetson:
├─ cat_detection_service.py (항상 실행 중)
│   └─ /front_cam/compressed 구독 → 고양이 감지 → 갤러리 저장
│
└─ robot_webrtc.py (버튼 클릭 시 시작)
    └─ /front_cam/compressed 구독 → WebRTC 스트리밍
```

### 동시 실행 시

```
/front_cam/compressed 토픽
    │
    ├─→ cat_detection_service.py (프로세스 1)
    │   └─ 고양이 감지 → 갤러리 저장 + 알림
    │
    └─→ robot_webrtc.py (프로세스 2)
        └─ WebRTC 스트리밍 → 프론트엔드 표시
```

**결과**: 두 프로세스가 동시에 같은 토픽을 구독해도 문제없음

## FAQ

### Q1: 실시간 스트리밍 중에도 고양이 감지가 작동하나요?

**A**: 네, 작동합니다. 고양이 감지 서비스는 별도 프로세스로 계속 실행 중이며, 실시간 스트리밍과 무관하게 동작합니다.

### Q2: 실시간 스트리밍 중 고양이가 탐지되면 어떻게 되나요?

**A**: 
- 실시간 영상은 계속 송출됩니다
- 고양이 감지 결과가 갤러리에 저장됩니다
- 알림이 표시됩니다
- **별개로 동작하며 서로 영향을 주지 않습니다**

### Q3: 두 기능이 충돌하지 않나요?

**A**: 충돌하지 않습니다. ROS2 토픽은 여러 구독자를 지원하며, 각 구독자는 독립적으로 메시지를 받습니다.

### Q4: 실시간 스트리밍을 끄면 고양이 감지도 멈추나요?

**A**: 아니요. 고양이 감지 서비스는 별도 프로세스로 실행되며, 실시간 스트리밍과 무관하게 계속 작동합니다.

### Q5: 성능에 영향을 주지 않나요?

**A**: 
- ROS2 토픽은 효율적으로 메시지를 배포합니다
- 각 구독자는 필요한 만큼만 처리합니다
- 일반적으로 성능 문제는 없습니다
- 다만 Jetson의 GPU/CPU 부하가 높을 수 있으므로 모니터링 권장

## 실제 동작 예시

### 상황: 사용자가 실시간 스트리밍을 켜고, 그 사이 고양이가 나타남

**타임라인**:
```
00:00 - 사용자가 "영상 시작" 버튼 클릭
00:01 - robot_webrtc.py 시작 → 실시간 스트리밍 시작
00:05 - 고양이 감지 서비스가 고양이 탐지
00:05 - 갤러리에 이미지/동영상 저장
00:05 - 웹 대시보드에 알림 표시
00:10 - 실시간 스트리밍은 계속 진행 중
00:15 - 사용자가 "영상 종료" 버튼 클릭
00:16 - robot_webrtc.py 종료 → 실시간 스트리밍 종료
00:20 - 고양이 감지 서비스는 계속 실행 중 (별도 프로세스)
```

**결과**:
- ✅ 실시간 스트리밍: 00:01 ~ 00:16 (15초간)
- ✅ 고양이 감지: 00:05에 탐지 및 저장
- ✅ 알림: 00:05에 표시
- ✅ 두 기능이 독립적으로 동작함
