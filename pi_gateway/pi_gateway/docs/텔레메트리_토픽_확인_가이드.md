# 텔레메트리 토픽 확인 가이드

`/telemetry/imu` 토픽이 발행되지 않을 때 해결 방법입니다.

---

## 🔍 문제 상황

```bash
ros2 topic echo /telemetry/imu
WARNING: topic [/telemetry/imu] does not appear to be published yet
```

---

## ✅ 해결 방법

### 1단계: Pi Gateway 실행 (ROS2 활성화)

```bash
cd /home/ssafy/Downloads/pi_gateway_spec_aligned/pi_gateway
export PYTHONPATH="$(pwd):$PYTHONPATH"

# ROS2 환경 설정 (필수!)
source /opt/ros/humble/setup.bash

# ROS2 활성화하여 실행
export DEMO_MODE=1
export ROS_ENABLED=1
python3 src/main.py
```

**확인:**
- 콘솔에 `[ROS] started.` 메시지 확인
- `TelemetryRosPublisher initialized` 메시지 확인

### 2단계: ROS2 토픽 확인

**새 터미널에서:**
```bash
# ROS2 환경 설정
source /opt/ros/humble/setup.bash

# 토픽 목록 확인
ros2 topic list

# 예상 토픽:
# /telemetry/battery
# /telemetry/imu
# /telemetry/encoder
# /telemetry/raw
# /odom
```

### 3단계: IMU 토픽 확인

```bash
# IMU 토픽 확인
ros2 topic echo /telemetry/imu

# 토픽 정보 확인
ros2 topic info /telemetry/imu
ros2 topic hz /telemetry/imu
```

## 🔧 문제 해결

### ROS2 환경이 설정되지 않았을 때

```bash
# ROS2 환경 설정
source /opt/ros/humble/setup.bash

# ROS2 데몬 확인
ros2 daemon status

# ROS2 데몬 재시작 (필요시)
ros2 daemon stop
ros2 daemon start
```

### Pi Gateway가 실행되지 않았을 때

```bash
# 프로세스 확인
ps aux | grep main.py

# 실행 중이 아니면 실행
cd /home/ssafy/Downloads/pi_gateway_spec_aligned/pi_gateway
export PYTHONPATH="$(pwd):$PYTHONPATH"
source /opt/ros/humble/setup.bash
export DEMO_MODE=1
export ROS_ENABLED=1
python3 src/main.py
```

### ROS_ENABLED가 False일 때

```bash
# 환경 변수 확인
echo $ROS_ENABLED

# ROS2 활성화하여 실행
export ROS_ENABLED=1
python3 src/main.py
```

### 데모 모드에서 텔레메트리가 없을 때

데모 모드에서는 `start_fake_telemetry`가 가짜 데이터를 생성합니다.
하지만 ROS2 모드가 아니면 토픽이 발행되지 않습니다.

**해결:**
```bash
# ROS2 활성화 필수
export ROS_ENABLED=1
python3 src/main.py
```

```
    ↓

[UART 데이터 수신 또는 가짜 데이터 생성]
    ↓
[TelemetryRosPublisher.publish 호출]
    ↓
[/telemetry/* 토픽 발행]
```

```json
  "data": "{\"yaw\": 1.57, \"pitch\": 0.0, \"roll\": 0.0, \"accx\": 0.0, \"accy\": 0.0, \"accz\": 9.8, \"gyro_z\": 0.0}"
```

```bash

# 또는 Python으로 파싱
ros2 topic echo /telemetry/imu | python3 -m json.tool
```

## 📝 참고

- **텔레메트리 브릿지**: `main.py`에서 자동 시작됨 (`TelemetryRosPublisher`)
- **가짜 데이터**: 데모 모드에서 `start_fake_telemetry`가 생성
- **실제 데이터**: UART 연결 시 STM32에서 수신
- **ROS2 필수**: 토픽 발행을 위해 `ROS_ENABLED=1` 필요
