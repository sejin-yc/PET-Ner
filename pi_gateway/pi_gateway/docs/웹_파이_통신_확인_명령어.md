# 웹 ↔ 파이 통신 확인 명령어

웹 UI ↔ Pi Gateway 연결이 제대로 되는지 단계별로 확인하는 명령어입니다.

---

## 1. Pi Gateway 확인 (파이에서)

Gateway가 떠 있는 상태에서 **같은 파이** 터미널에서:

```bash
# 헬스 체크 (JSON 응답 나오면 정상)
curl -s http://localhost:8000/robot/health | python3 -m json.tool

# 텔레메트리 (배터리, IMU 등)
curl -s http://localhost:8000/telemetry/latest | python3 -m json.tool
```

- `resultCode: "SUCCESS"`, `data` 안에 `mode`, `estop`, `battery` 등이 있으면 Gateway 동작 정상.

---

## 2. MQTT 브로커 확인 (1883 접근 가능한 곳에서)

서버(i14c203.p.ssafy.io) MQTT(1883)에 접속 가능할 때:

```bash
# 제어 메시지 구독 (웹에서 조이스틱 움직이면 여기로 메시지 옴)
mosquitto_sub -h i14c203.p.ssafy.io -p 1883 -t "/pub/robot/control" -v

# 로봇 상태 구독 (브릿지가 주기적으로 발행)
mosquitto_sub -h i14c203.p.ssafy.io -p 1883 -t "/sub/robot/status" -v
```

- 웹에서 조이스틱/키보드 제어 시 `/pub/robot/control` 에 `MOVE`, `STOP`, `MODE` 등이 보이면 **웹 → 백엔드 → MQTT** 구간 정상.
- `/sub/robot/status` 에 주기적으로 JSON이 보이면 **브릿지 → MQTT** 발행 정상.

`mosquitto_sub` 가 없으면: `sudo apt install mosquitto-clients` (Ubuntu/Debian) 또는 Docker 등으로 Mosquitto 클라이언트 설치.

---

## 3. 브릿지 로그 확인 (파이 터미널 2)

`python3 scripts/mqtt_pi_bridge.py` 실행 시:

- `[MQTT] ✅ 연결 성공! 토픽 구독: /pub/robot/control` → MQTT 연결·구독 정상.
- `[MQTT] 상태 발행 중...` (주기적) → Pi Gateway에서 상태 가져와서 MQTT로 발행 중.

웹에서 조이스틱을 움직여도 브릿지 쪽에 별도 로그가 없을 수 있음 (코드에 print 없음). **실제 로봇이 움직이면** 브릿지가 제대로 전달하는 것.

---

## 4. 한 번에 확인 (파이에서)

```bash
# 1) Gateway 살아 있는지
curl -s http://localhost:8000/robot/health | head -5

# 2) 브릿지가 Gateway에 접속 가능한지 (브릿지 실행 중일 때)
curl -s http://localhost:8000/robot/health | head -5
```

둘 다 JSON이 나오면, 파이 위에서는 Gateway·브릿지 경로까지 정상.

---

## 5. 흐름 요약

| 확인 대상 | 명령어 / 방법 |
|-----------|----------------|
| Pi Gateway 동작 | `curl http://localhost:8000/robot/health` (파이에서) |
| 웹 → MQTT | 웹에서 조이스틱 동작 + `mosquitto_sub -h i14c203.p.ssafy.io -t "/pub/robot/control"` |
| 브릿지 → MQTT | `mosquitto_sub -h i14c203.p.ssafy.io -t "/sub/robot/status"` (주기적 메시지) |
| 끝단 동작 | 웹에서 조이스틱 → 로봇(바퀴) 반응 |

---

## 6. 문제 나올 때

- **Gateway 응답 없음**: 도커(Pi Gateway)가 떠 있는지, 8000 포트 사용 중인지 확인.
- **브릿지 `[MQTT] 연결 실패`**: 서버 1883 포트 개방 여부, `MQTT_HOST`/`MQTT_PORT` 값 확인.
- **웹 조이스틱해도 MQTT에 안 옴**: 백엔드(robot_server)와 MQTT(robot_mqtt) 연결, 웹 STOMP 연결 확인.
- **MQTT에는 오는데 로봇 안 움직임**: 브릿지가 Pi Gateway(localhost:8000)에 붙는지, UART/STM32 연결 확인.
