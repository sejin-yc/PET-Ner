# 수동조작 구조 재분석

## 개요

`gateway_sj.cpp`의 역할과 수동조작 노드의 관계를 재분석합니다.

---

## gateway_sj.cpp의 역할

### 현재 동작
- **구독**: `/cmd_vel_out` 토픽
- **변환**: `Twist` 메시지 → 1바이트 문자 (w/x/a/d/q/e/s)
- **전송**: UART로 STM32에 전송

### 명령 매핑
```
w: 전진 (vx > 0)
x: 후진 (vx < 0)
a: 좌측 횡이동 (vy > 0)
d: 우측 횡이동 (vy < 0)
q: 좌회전 (wz > 0)
e: 우회전 (wz < 0)
s: 정지 (모든 값이 0에 가까움)
```

**의미**: 키보드 입력과 유사한 수동조작 명령!

---

## 가능한 구조

### 시나리오 1: ROS2 폴더에 수동조작 노드가 있음

```
수동조작 노드 (키보드/조이스틱)
    └─ /cmd_vel_out 발행
        ↓
gateway_sj.cpp (구독)
    └─ UART → STM32
```

**이 경우**:
- ROS2 폴더의 수동조작 노드가 `/cmd_vel_out` 발행
- `gateway_sj.cpp`가 UART로 전송
- Pi Gateway와는 별개 경로

### 시나리오 2: Pi Gateway가 /cmd_vel_out 발행

```
Pi Gateway CmdVelMux
    └─ /cmd_vel_out 발행 (원래 구조)
        ↓
gateway_sj.cpp (구독)
    └─ UART → STM32
```

**현재 상태**:
- Pi Gateway는 `/cmd_vel_joy` 발행 (변경됨)
- `gateway_sj.cpp`는 `/cmd_vel_out` 구독
- **불일치 발생!**

---

## 문제점

### 현재 구조의 불일치

**Pi Gateway**:
- `/cmd_vel_joy` 발행 (twist_mux 입력)

**gateway_sj.cpp**:
- `/cmd_vel_out` 구독

**결과**: `gateway_sj.cpp`가 Pi Gateway의 명령을 받지 못함!

---

## 해결 방안

### 방안 1: Pi Gateway가 /cmd_vel_out도 발행

**구조**:
```
Pi Gateway CmdVelMux
    ├─ /cmd_vel_joy 발행 (twist_mux 입력)
    └─ /cmd_vel_out 발행 (gateway_sj.cpp 입력)
```

**장점**:
- `gateway_sj.cpp`가 Pi Gateway 명령을 받을 수 있음
- twist_mux와도 통합 가능

**단점**:
- 두 개의 토픽 발행 필요

### 방안 2: gateway_sj.cpp가 /cmd_vel 구독

**구조**:
```
Pi Gateway
    └─ /cmd_vel_joy 발행
        ↓
twist_mux
    └─ /cmd_vel 발행
        ↓
gateway_sj.cpp (구독 토픽 변경)
    └─ UART 전송
```

**장점**:
- twist_mux 통합 경로 사용
- 단일 경로

**단점**:
- `gateway_sj.cpp` 수정 필요 (또는 파라미터 변경)

### 방안 3: ROS2 폴더 수동조작 노드 사용

**구조**:
```
ROS2 폴더 수동조작 노드
    └─ /cmd_vel_out 발행
        ↓
gateway_sj.cpp
    └─ UART 전송

Pi Gateway
    └─ /cmd_vel_joy 발행
        ↓
twist_mux
    └─ /cmd_vel 발행
        ↓
twist_to_stm_uart_bridge.py
    └─ UART 전송
```

**장점**:
- 각자 독립적으로 동작

**단점**:
- 두 개의 UART 브릿지가 동시에 실행되면 충돌 가능

---

## 권장 방안

### 방안 2 (gateway_sj.cpp가 /cmd_vel 구독) 권장

**이유**:
1. twist_mux 통합 경로 사용
2. 우선순위 관리 자동화
3. 단일 UART 브릿지 사용 가능

**구현**:
- `gateway_sj.cpp`의 `cmd_topic` 파라미터를 `/cmd_vel`로 변경
- 또는 `twist_to_stm_uart_bridge.py`만 사용하고 `gateway_sj.cpp`는 사용 안 함

---

## 현재 구조 정리

### Pi Gateway
- `/cmd_vel_joy` 발행 (twist_mux 입력)

### ROS2 폴더
- `twist_mux`: `/cmd_vel_joy` 구독 → `/cmd_vel` 발행
- `twist_to_stm_uart_bridge.py`: `/cmd_vel` 구독 → UART 전송
- `gateway_sj.cpp`: `/cmd_vel_out` 구독 → UART 전송

### 문제
- `gateway_sj.cpp`가 Pi Gateway의 명령을 받지 못함
- `/cmd_vel_out`을 발행하는 노드가 없음

2. **현재 구조 불일치**: Pi Gateway가 `/cmd_vel_joy`를 발행하지만, `gateway_sj.cpp`는 `/cmd_vel_out`을 구독
