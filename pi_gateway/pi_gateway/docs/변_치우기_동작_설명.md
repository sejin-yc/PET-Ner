# 변 치우기 동작 설명

`arm/start`의 `action_id`가 변 치우기와 어떻게 연결되는지, 그리고 순찰 중에 어떻게 실행되는지 설명합니다.

---

## 🎯 action_id의 의미

### action_id = 1: 변 치우기 시작

**의미**: 로봇팔이 "변 치우기" 작업을 시작합니다.

**동작**:
1. 젯슨이 `arm/start` 토픽 발행 (`action_id = 1`)
2. Pi Gateway가 STM32로 `ID_ARM_START` (action_id=1) 전송
3. STM32가 바퀴 모터 잠금
4. **로봇팔 강화학습 모델이 자율로 변 치우기 작업 수행**
   - 화장실 4구역 (상좌, 상우, 하좌, 하우) 자동 처리
   - 로봇팔이 알아서 변을 퍼서 버리는 동작 수행
5. 작업이 완료될 때까지 로봇팔 동작 중

---

### action_id = 0: 로봇팔 정지

**의미**: 변 치우기 작업이 완료되어 로봇팔을 정지합니다.

**동작**:
1. 젯슨이 `arm/start` 토픽 발행 (`action_id = 0`)
2. Pi Gateway가 STM32로 `ID_ARM_START` (action_id=0) 전송
3. STM32가 바퀴 모터 해제
4. 로봇팔 정지
5. 로봇 이동 가능

---

## 🔄 전체 흐름

### 순찰 중 자동 실행

**파일**: `src/ros_cmdvel.py` - `PatrolActionScheduler`

변 치우기는 **순찰 중에만** 자동으로 실행됩니다.

```
1. 순찰 모드 시작 (mode = "auto")
   → PatrolLoop.tick() 호출
   → mode == "auto"일 때만 action_scheduler.tick() 실행

2. 순찰 시작 시점 체크
   → 순찰 모드로 전환될 때 (mode = "auto") 체크
   → 저녁 시간대(예: 18:00 이후)인지 확인
   → 오늘 이미 실행했는지 확인 (날짜 기반)
   → 조건 만족 시 변 치우기 시작

3. 변 치우기 시작
   → PatrolActionScheduler가 make_arm_start_frame(1) 전송
   → STM32: ID_ARM_START (action_id=1) 수신
   → 바퀴 잠금
   → 로봇팔 강화학습 모델이 변 치우기 작업 수행 (자율)

4. 변 치우기 작업 중
   로봇팔이 화장실 4구역을 자동으로 처리
   (젯슨의 강화학습 모델이 제어)

5. 변 치우기 완료
   젯슨 → arm/start (action_id=0) 발행
   → Pi Gateway → STM32: ID_ARM_START (action_id=0)
   → 바퀴 해제
   → 로봇팔 정지
   → 순찰 계속 진행
```

### 수동 실행 (웹 API 또는 젯슨)

순찰 중이 아니어도 수동으로 실행 가능:

```
1. 웹 API: POST /arm/start {"action_id": 1}
   또는
   젯슨: arm/start 토픽 발행 (action_id=1)

2. 변 치우기 시작
   → 동일한 흐름
```

---

## 💡 핵심 포인트

### action_id는 "동작 종류"가 아니라 "시작/정지" 신호

- **action_id = 1**: "변 치우기" 작업 시작 신호
- **action_id = 0**: 작업 완료/정지 신호

### 실제 변 치우기 작업은 젯슨이 담당

- Pi Gateway는 시작/정지 신호만 전달
- 실제 변 치우기 동작 (어떤 구역을 어떻게 치울지)은 젯슨의 강화학습 모델이 담당
- STM32는 서보 모터를 제어하여 로봇팔을 움직임

## 📝 요약

### action_id 의미

| action_id | 의미 | 바퀴 상태 |
|-----------|------|---------|
| **1** | 변 치우기 시작 (로봇팔이 변 치우기 작업 시작) | 잠금 |
| **0** | 변 치우기 완료/정지 (로봇팔 정지) | 해제 |

**action_id = 1**: "지금부터 변 치우기 시작해!"  
**action_id = 0**: "변 치우기 끝났어, 로봇팔 정지해!"

### 순찰 중 실행

**중요**: 변 치우기는 **순찰 중에만** 자동으로 실행됩니다!

- **순찰 모드 (mode = "auto")**: `PatrolActionScheduler`가 시간 간격에 따라 자동 실행
- **수동 모드 (mode = "teleop")**: 자동 실행 안 됨 (웹 API나 젯슨에서 수동 실행 가능)

**설정**: `config/params.yaml`
```yaml
patrol_actions:
  litter_clean_daily_hour: 18      # 변치우기: 저녁 순찰 시간 (시, 0~23). -1이면 비활성화. 예: 18 = 저녁 6시 이후 순찰 시 실행
  litter_clean_daily_minute: 0     # 변치우기: 저녁 순찰 시간 (분, 0~59)
```

**동작 방식**:
- 저녁 시간대(예: 18:00 이후)에 순찰이 시작될 때 한 번만 실행
- 하루에 한 번만 실행 (같은 날에는 다시 실행되지 않음)
- 순찰 모드로 전환될 때(`mode = "auto"`) 체크
