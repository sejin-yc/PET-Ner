## 2. left, right가 좌회전 우회전?

### ✅ 답변: 네, left/right가 좌회전/우회전입니다!

**코드 확인**: `src/main.py:49-52`

```python
if "left" in pressed:  wz += cfg.angular_speed   # 좌회전
if "right" in pressed: wz -= cfg.angular_speed    # 우회전
if "rot_l" in pressed: wz += cfg.angular_speed   # 좌회전 (rot_l도 지원)
if "rot_r" in pressed: wz -= cfg.angular_speed   # 우회전 (rot_r도 지원)
```

**의미**:
- `left` = **좌회전** (반시계방향, 양수 `wz`)
- `right` = **우회전** (시계방향, 음수 `wz`)
- `rot_l` = **Rotate Left** (좌회전, `left`와 동일, 호환성 지원)
- `rot_r` = **Rotate Right** (우회전, `right`와 동일, 호환성 지원)

**키보드 입력**:
- FE에서는 **4개 키**만 사용합니다:
  - `W`: 전진 (`linear += 1.0`)
  - `S`: 후진 (`linear -= 1.0`)
  - `A`: 좌회전 (`angular += 1.0`)
  - `D`: 우회전 (`angular -= 1.0`)

**FE 코드 확인**: `RobotContext.jsx:375-376`

```javascript
if (keysPressed.current['w']) linear += 1.0;  // 전진
if (keysPressed.current['s']) linear -= 1.0;  // 후진
if (keysPressed.current['a']) angular += 1.0;  // 좌회전
if (keysPressed.current['d']) angular -= 1.0;  // 우회전
```

**MQTT 브릿지 변환**: `mqtt_pi_bridge.py:127-141`

```python
if t == "MOVE":
    linear = float(payload.get("linear", 0.0))  # W/S
    angular = float(payload.get("angular", 0.0))  # A/D
    presses = [
        ("up", linear > 0),      # W → up
        ("down", linear < 0),    # S → down
        ("left", angular > 0),   # A → left (좌회전)
        ("right", angular < 0),  # D → right (우회전)
        ("rot_l", False),        # 사용 안 함 (left로 통일)
        ("rot_r", False),        # 사용 안 함 (right로 통일)
    ]
```

**Pi Gateway 지원**:
- Pi Gateway는 `left`, `right`를 **좌회전/우회전**으로 처리합니다 (`wz` 제어)
- `rot_l`, `rot_r`도 지원하지만 `left`, `right`와 동일하게 동작 (호환성)
- FE에서는 실제로 **4개 키만 사용** (W, S, A, D)

## 4. 20Hz로 보낸다는 건 뭘 20Hz로 보낸다는 거지?

### ✅ 답변: cmd_vel 명령을 초당 20번 전송합니다!

**코드 확인**: `src/main.py:79-106`

```python
def start_demo_teleop_loop(uart: UartLink, tele_cfg: TeleopConfig, tx_rate_hz: float = 20.0, ...):
    period = 1.0 / max(1.0, float(tx_rate_hz))  # 1/20 = 0.05초
    
    def loop():
        while True:
            now = time.time()
            # ... (생략) ...
            
            if STATE.mode == "teleop":
                vx, vy, wz = state_to_cmd(STATE, tele_cfg, cat_detector)
                uart.send(make_cmd_vel_frame(vx, vy, wz))  # ← 이걸 20Hz로 전송
            
            time.sleep(period)  # 0.05초 대기 (20Hz)
```

**의미**:
- **20Hz** = 초당 20번 = 0.05초마다 한 번
- **전송 내용**: `make_cmd_vel_frame(vx, vy, wz)` (이동 명령)
- **이유**: STM32가 주기적으로 명령을 받아야 모터를 부드럽게 제어할 수 있음

**예시**:
```
시간: 0.00초 → cmd_vel(vx=0.2, vy=0.0, wz=0.0) 전송
시간: 0.05초 → cmd_vel(vx=0.2, vy=0.0, wz=0.0) 전송
시간: 0.10초 → cmd_vel(vx=0.2, vy=0.0, wz=0.0) 전송
...
(1초에 20번 전송)
```

**가짜 텔레옵 패킷?**
- 아니요! **실제 UART 통신**입니다.
- `uart.send()`가 실제로 STM32로 전송합니다.
- 단, `uart.enabled = false`이면 hex만 출력 (dry-run 모드)

## 5. linear_speed, angular_speed가 왜 필요한거야?

### ✅ 답변: 사용자 입력을 실제 물리 속도로 변환하기 위해 필요합니다!

**코드 확인**: `src/main.py:37-52`

```python
def state_to_cmd(state, cfg: TeleopConfig, cat_detector: Optional[CatDetector] = None):
    if getattr(state, "joy_active", False):
        vx = float(getattr(state, "joy_x", 0.0)) * cfg.linear_speed  # ← 여기서 곱하기!
        vy = float(getattr(state, "joy_y", 0.0)) * cfg.linear_speed
        wz = 0.0
    else:
        pressed = getattr(state, "pressed", set()) or set()
        if "up" in pressed:    vx += cfg.linear_speed      # ← 여기서 사용!
        if "down" in pressed:  vx -= cfg.linear_speed
        if "left" in pressed:  vy += cfg.linear_speed
        if "right" in pressed: vy -= cfg.linear_speed
        if "rot_l" in pressed: wz += cfg.angular_speed    # ← 여기서 사용!
        if "rot_r" in pressed: wz -= cfg.angular_speed
```

**의미**:
- `linear_speed = 0.20 m/s`: 전진/후진/좌우 이동 속도 (물리 단위)
- `angular_speed = 0.80 rad/s`: 회전 속도 (물리 단위)

**왜 필요한가?**
1. **웹 입력은 정규화된 값** (-1.0 ~ 1.0)
   - 조이스틱: `joy_x = 0.5` (50% 밀기)
   - 키보드: `"up"` 키 누름 (100% 전진)

2. **STM32는 물리 속도가 필요**
   - `vx = 0.2 m/s` (초당 0.2미터)
   - `wz = 0.8 rad/s` (초당 0.8라디안 회전)

3. **변환 공식**:
   ```
   vx = joy_x * linear_speed
   wz = (rot_l ? 1 : 0) * angular_speed
   ```

**예시**:
- 조이스틱을 50% 밀었을 때: `vx = 0.5 * 0.20 = 0.10 m/s`
- `rot_l` 키를 눌렀을 때: `wz = 1.0 * 0.80 = 0.80 rad/s`

**설정 파일**: `config/params.yaml:30-32`
```yaml
teleop:
  linear_speed: 0.20   # m/s (초당 0.2미터)
  angular_speed: 0.80  # rad/s (초당 0.8라디안)
```

## 6. 타이머: 30Hz로 tick() 호출이 무슨 뜻이야?

### ✅ 답변: 초당 30번 `tick()` 함수를 호출합니다!

**코드 확인**: `src/ros_cmdvel.py:177`

```python
self.timer = self.create_timer(1.0 / float(self.cfg.rate_hz), self.tick)
# rate_hz = 30.0이면
# 1.0 / 30.0 = 0.0333초마다 tick() 호출
# = 초당 30번 호출 = 30Hz
```

**의미**:
- **30Hz** = 초당 30번 = 0.0333초마다 한 번
- **호출 함수**: `PatrolLoop.tick()`
- **역할**: 패트롤 패턴에서 속도를 계산하고 `cmd_vel_auto` 토픽 발행

**코드 확인**: `src/ros_cmdvel.py:208-225`

```python
def tick(self):
    now = time.time()
    dt = now - self._last_ts  # 경과 시간 계산
    self._last_ts = now
    
    out = Twist()
    if self.mode == "auto":
        vx, wz = self.profile.step(dt)  # 패턴에서 속도 가져오기

    self.pub_auto.publish(out)  # ROS 토픽 발행
```

**왜 30Hz인가?**
- ROS2 표준 주파수 (일반적으로 10~50Hz 사용)
- 부드러운 제어를 위해 충분히 빠른 주기
- CPU 부하를 고려한 적절한 주파수

## 7. PatrolProfile 패턴 재생에서 구간이 뭐야? 시간이야?

### ✅ 답변: 구간은 시간과 속도의 조합입니다!

**코드 확인**: `src/ros_cmdvel.py:24-30`

```python
@dataclass
class PatrolSegment:
    """패턴 한 구간. duration_s 동안 (vx_mps, wz_rps) 유지."""
    duration_s: float      # 구간 지속 시간 (초)
    vx_mps: float         # 전진 속도 (m/s)
    wz_rps: float         # 회전 속도 (rad/s)
```

**구간의 의미**:
- **구간(Segment)** = 특정 시간 동안 특정 속도로 이동하는 단위
- 예: "2초 동안 0.15 m/s로 전진"

**기본 패턴**: `src/ros_cmdvel.py:49-52`

```python
segments = [
    PatrolSegment(duration_s=2.0, vx_mps=0.15, wz_rps=0.0),  # 구간 1: 앞으로 2초
    PatrolSegment(duration_s=1.2, vx_mps=0.0, wz_rps=0.7),   # 구간 2: 회전 1.2초
]
```

**동작 흐름**: `src/ros_cmdvel.py:69-78`

```python
def step(self, dt: float) -> Tuple[float, float]:
    self._seg_elapsed += dt  # 경과 시간 누적
    
    seg = self.cfg.segments[self._seg_idx]  # 현재 구간
    
    # 구간 시간이 지나면 다음 구간으로 순환
    while self._seg_elapsed >= seg.duration_s and seg.duration_s > 0:
        self._seg_elapsed -= seg.duration_s
        self._seg_idx = (self._seg_idx + 1) % len(self.cfg.segments)  # 순환
        seg = self.cfg.segments[self._seg_idx]
    
    return seg.vx_mps, seg.wz_rps  # 현재 구간의 속도 반환
```

**예시**:
```
시간: 0.0초 → 구간 1 시작 (vx=0.15, wz=0.0)
시간: 0.5초 → 구간 1 진행 중 (vx=0.15, wz=0.0)
시간: 2.0초 → 구간 1 완료 → 구간 2로 전환 (vx=0.0, wz=0.7)
시간: 3.2초 → 구간 2 완료 → 구간 1로 순환 (vx=0.15, wz=0.0)
시간: 5.2초 → 구간 1 완료 → 구간 2로 순환 (vx=0.0, wz=0.7)
... (무한 반복)
```

**왜 속도를 반환하나?**
- `PatrolLoop.tick()`에서 `cmd_vel_auto` 토픽을 발행하기 위해
- 매 프레임마다 현재 구간의 속도를 가져와서 ROS2 토픽으로 발행
- STM32가 받아서 모터 제어

## 8. self.pub_auto.publish(out)에서 out은 내가 설정한 이름인가?

### ✅ 답변: 네, 변수 이름은 제가 설정한 것입니다!

**코드 확인**: `src/ros_cmdvel.py:212-225`

```python
def tick(self):
    now = time.time()
    dt = now - self._last_ts
    self._last_ts = now
    out = Twist()  # ← 변수 이름은 자유롭게 설정 가능
    if self.estop:
        self.pub_auto.publish(out)
        return
    if self.mode == "auto":
        vx, wz = self.profile.step(dt)
        out.linear.x = float(vx)      # ← ROS2 표준 메시지 형식
        out.angular.z = float(wz)     # ← ROS2 표준 메시지 형식
    
    self.pub_auto.publish(out)  # ← publish()는 ROS2 표준 메서드
```

**설명**:
- `out`: 변수 이름 (자유롭게 변경 가능, 예: `msg`, `twist`, `cmd`)
- `Twist()`: ROS2 표준 메시지 타입 (`geometry_msgs/msg/Twist`)
- `publish()`: ROS2 표준 메서드 (변경 불가)

**ROS2 표준 형식**:
```python
from geometry_msgs.msg import Twist

out = Twist()
out.linear.x = 0.15   # 전진 속도
out.linear.y = 0.0    # 횡이동 속도
out.linear.z = 0.0    # 상승 속도 (사용 안 함)
out.angular.x = 0.0   # 롤 (사용 안 함)
out.angular.y = 0.0   # 피치 (사용 안 함)
out.angular.z = 0.7   # 회전 속도
```

**결론**: `out`은 변수 이름이지만, `Twist()`와 `publish()`는 ROS2 표준입니다.

## 9. state_to_cmd()는 뭐하는 애야?

### ✅ 답변: 웹 입력 상태를 물리 속도 명령으로 변환하는 함수입니다!

**코드 확인**: `src/main.py:37-61`

```python
def state_to_cmd(state, cfg: TeleopConfig, cat_detector: Optional[CatDetector] = None):
    """
    WebState를 받아서 (vx, vy, wz) 물리 속도로 변환
    
    Args:
        state: WebState 객체 (pressed, joy_x, joy_y, estop 등)
        cfg: TeleopConfig (linear_speed, angular_speed)
        cat_detector: CatDetector (고양이 탐지 시 속도 감소)
    
    Returns:
        (vx, vy, wz): 물리 속도 (m/s, m/s, rad/s)
    """
    # 1. 비상정지 체크
    if getattr(state, "estop", False):
        return 0.0, 0.0, 0.0
    
    # 2. 조이스틱 입력 처리
    if getattr(state, "joy_active", False):
        vx = float(getattr(state, "joy_x", 0.0)) * cfg.linear_speed
        vy = float(getattr(state, "joy_y", 0.0)) * cfg.linear_speed
        wz = 0.0
    else:
        # 3. 키보드 입력 처리
        pressed = getattr(state, "pressed", set()) or set()
        vx = vy = wz = 0.0
        if "up" in pressed:    vx += cfg.linear_speed
        if "down" in pressed:  vx -= cfg.linear_speed
        if "left" in pressed:  vy += cfg.linear_speed
        if "right" in pressed: vy -= cfg.linear_speed
        if "rot_l" in pressed: wz += cfg.angular_speed
        if "rot_r" in pressed: wz -= cfg.angular_speed
    
    # 4. 고양이 탐지 시 속도 감소
    if cat_detector:
        multiplier = cat_detector.get_speed_multiplier()  # 0.5 또는 1.0
        vx *= multiplier
        vy *= multiplier
        wz *= multiplier
    
    return vx, vy, wz
```

**역할**:
1. **입력 변환**: 웹 입력 → 물리 속도
2. **우선순위 처리**: 비상정지 > 조이스틱 > 키보드
3. **안전 기능**: 고양이 탐지 시 속도 감소

**사용 위치**: `src/main.py:105`
```python
vx, vy, wz = state_to_cmd(STATE, tele_cfg, cat_detector)
uart.send(make_cmd_vel_frame(vx, vy, wz))
```

## 10. 웹 API는 왜 필요한거야?

### ✅ 답변: FE/BE가 Pi Gateway를 제어하기 위해 필요합니다!

**웹 API 목적**:
1. **FE → Pi Gateway 직접 통신**: HTTP/WebSocket으로 로봇 제어
2. **BE → Pi Gateway 통신**: MQTT 브릿지를 통한 간접 통신
3. **다른 시스템 연동**: 젯슨, 외부 서비스 등

**현재 구현된 API**: `src/web_ws_server.py`

**로봇팔 제어 API**:
- `POST /arm/start`: 로봇팔 동작 시작/정지
- `POST /arm/position_correct`: 로봇 위치 보정
- `POST /arm/water`: 급수 동작 제어

**왜 필요한가?**

**시나리오 1: FE에서 직접 제어**
```
사용자가 웹 UI에서 "로봇팔 시작" 버튼 클릭
  ↓
FE → HTTP POST /arm/start
  ↓
Pi Gateway → UART 전송
  ↓
STM32 → 로봇팔 동작
```

**시나리오 2: 젯슨에서 제어**
```
젯슨 AI 모델이 로봇팔 제어 필요
  ↓
젯슨 → ROS2 토픽 발행: arm/start
  ↓
ArmControlBridge → UART 전송
  ↓
STM32 → 로봇팔 동작
```

**시나리오 3: BE 서버를 통한 제어**
```
사용자가 모바일 앱에서 제어
  ↓
모바일 앱 → BE 서버
  ↓
BE 서버 → MQTT → MQTT 브릿지 → Pi Gateway WebSocket
  ↓
Pi Gateway → UART 전송
```

**결론**: 웹 API는 다양한 경로로 Pi Gateway를 제어할 수 있게 해주는 인터페이스입니다!

## 16. PatrolLoop.tick() (30Hz)가 무슨 뜻이야?

### ✅ 답변: PatrolLoop의 tick() 함수를 초당 30번 호출한다는 뜻입니다!

**코드 확인**: `src/ros_cmdvel.py:177`

```python
self.timer = self.create_timer(1.0 / float(self.cfg.rate_hz), self.tick)
# rate_hz = 30.0이면
# 1.0 / 30.0 = 0.0333초마다 tick() 호출
# = 초당 30번 = 30Hz
```

**의미**:
- **30Hz** = 초당 30번 호출
- **호출 함수**: `PatrolLoop.tick()`
- **주기**: 0.0333초마다 한 번

**tick() 함수 역할**: `src/ros_cmdvel.py:208-225`

```python
def tick(self):
    # 1. 경과 시간 계산
    now = time.time()
    dt = now - self._last_ts
    self._last_ts = now
    
    # 2. 패턴에서 속도 가져오기
    if self.mode == "auto":
        vx, wz = self.profile.step(dt)  # 패턴 재생
        out.linear.x = float(vx)
        out.angular.z = float(wz)
        
        # 3. 패트롤 중 액션 스케줄러 실행
        if self.action_scheduler is not None:
            self.action_scheduler.tick(now)
    
    # 4. ROS2 토픽 발행
    self.pub_auto.publish(out)
```

**왜 30Hz인가?**
- 부드러운 제어를 위해 충분히 빠른 주기
- ROS2 표준 주파수 범위 내
- CPU 부하를 고려한 적절한 주파수
