# 고양이 탐지 결과 웹 전달 - 백엔드·프론트 담당자 전달 문서

**작성일**: 2026.02  
**목적**: Pi Gateway 고양이 감지 결과를 백엔드·프론트에서 처리

---

## 1. 전체 흐름

**WebSocket 사용 시** (권장):
```
[Pi cat_detection_service]  BE_WS_URL 설정
    └─ SEND /pub/robot/cat_state → [백엔드 @MessageMapping] → Log 저장 + /sub/robot/{userId}/cat_state
```

**HTTP fallback** (BE_WS_URL 없을 때):
```
[Pi cat_detection_service]
    └─ POST /api/robot/cat_state (443)
                │
                ▼
[백엔드]  RobotController 수신
    │
    ├─ Log 저장 (주간 감지 리포트)
    └─ WebSocket 전달 → /sub/robot/{userId}/cat_state
                │
                ▼
[프론트]  STOMP 구독 → 갤러리 갱신, 로그 갱신
```

---

## 2. 전체 상황 요약

| 구간 | 완료 | 미완료 |
|------|------|--------|
| **Pi (당신)** | POST /api/videos, SEND /pub/robot/cat_state(WS) 또는 POST /api/robot/cat_state(HTTP) | - |
| **백엔드** | - | **POST /api/robot/cat_state** 추가, Log 저장, POST/GET /api/videos |
| **프론트** | - | cat_state 구독, 갤러리 초기 로드 |

**※ 주간 감지 리포트**: LogsPage "주간 감지 리포트"는 Log 엔티티 `detectionCount` 사용. 고양이 감지 시 **Log 저장** 필요.

## 3. 백엔드 수정

**※ WebSocket 사용 시**: `@MessageMapping("/robot/cat_state")` 사용 (WebSocket_로봇연동_백엔드_전달_문서 참고).  
**HTTP fallback용**: 아래 `POST /api/robot/cat_state` 추가.

### 3.1 RobotController - POST /api/robot/cat_state (HTTP fallback)

**파일**: `server/src/main/java/com/ssafy/robot_server/controller/RobotController.java`

**추가할 코드** (POST /api/robot/status 근처에 추가):

```java
/**
 * Pi cat_detection_service가 HTTP로 고양이 감지 결과 전송 (1883 막혀 있어서 HTTP 사용)
 */
@PostMapping("/cat_state")
public ResponseEntity<?> receiveCatState(@RequestBody Map<String, Object> payload) {
    try {
        Long userId = Long.valueOf(String.valueOf(payload.getOrDefault("userId", 1L)));
        // Log 저장 → "주간 감지 리포트"에 표시
        Log logEntry = Log.builder()
                .userId(userId)
                .rentId(0L).vehicleId(0L)
                .mode("auto").status("completed")
                .duration("0분").durationNum(0).distance(0)
                .detectionCount(1)
                .details("고양이 감지: " + payload.getOrDefault("action_ko", "") + " / " + payload.getOrDefault("emotion_ko", ""))
                .build();
        logRepository.save(logEntry);
        // WebSocket 전달 → 프론트 갤러리·로그 갱신
        messagingTemplate.convertAndSend("/sub/robot/" + userId + "/cat_state", payload);
        return ResponseEntity.ok().build();
    } catch (Exception e) {
        System.err.println("POST /api/robot/cat_state 에러: " + e.getMessage());
        return ResponseEntity.badRequest().build();
    }
}
```

**필요 사항**: `LogRepository`, `SimpMessagingTemplate` 주입. Log 엔티티에 `createdAt` 자동 설정이면 `.createdAt(...)` 생략 가능.

## 4. 프론트 수정

### 4.1 RobotContext - /cat_state 구독

**파일**: `client/src/contexts/RobotContext.jsx`

**위치**: `client.subscribe(\`/sub/${user.id}/peer/ice\`, ...)` 블록 **바로 다음**, `});` (onConnect 끝) **앞에**

**추가할 코드**:

```javascript
        // 고양이 감지 결과
        client.subscribe(`/sub/${user.id}/robot/cat_state`, (message) => {
          try {
            const payload = JSON.parse(message.body);
            api.get('/videos').then(res => setVideos(res.data || [])).catch(() => {});
            if (queryClient) queryClient.invalidateQueries(['logs']);
          } catch (e) {
            console.error("cat_state 파싱 에러:", e);
          }
        });
```

**주의**: `queryClient` 없으면 `invalidateQueries` 줄 제거. STOMP 구독 경로는 기존 `/status`와 동일 패턴 사용.

---

### 4.2 갤러리 초기 로드

**위치**: `fetchInitialState` useEffect **바로 아래**, 같은 `}, [user]);` 블록 **밖에**

**추가할 코드**:

```javascript
  // 갤러리 영상 초기 로드
  useEffect(() => {
    if (!user?.id) return;
    api.get('/videos').then(res => setVideos(res.data || [])).catch(() => {});
  }, [user?.id]);
```
