# 패트롤 노드 연동 가이드

## 개요

팀원이 패트롤을 담당하므로, Pi Gateway는 패트롤 노드와의 통신만 처리하면 됩니다.

## 역할 분담

### 팀원 (패트롤 담당)
- ✅ 패트롤 경로 계획 및 실행
- ✅ `/cmd_vel_nav` 토픽 발행 (자동 주행 명령)
- ✅ `/gateway/allow_strafe` 토픽 발행 (횡이동 허용 여부)
- ✅ 작업 완료 신호 대기 (`/arm/done`, `/dock/done`, `/servo/done`)

### Pi Gateway (통신 담당)
- ✅ `/cmd_vel_nav` 토픽 구독 (패트롤 명령 수신)
- ✅ `/gateway/allow_strafe` 토픽 구독 (횡이동 허용 여부 수신)
- ✅ 작업 완료 신호 발행 (`/arm/done`, `/dock/done`, `/servo/done`)
- ✅ 웹 대시보드 명령과 패트롤 명령 통합 (`/cmd_vel` 또는 twist_mux)

---

## 통신 구조

### 패트롤 노드 → Pi Gateway

```
패트롤 노드 (팀원)
    │
    ├─→ /cmd_vel_nav (geometry_msgs/Twist)
    │   └─ 자동 주행 속도 명령
    │
    └─→ /gateway/allow_strafe (std_msgs/Bool)
        └─ 횡이동 허용 여부
            ↓
Pi Gateway
    ├─ /cmd_vel_nav 구독
    ├─ /gateway/allow_strafe 구독
    └─ 웹 명령(/cmd_vel_teleop)과 통합
        ↓
    /cmd_vel (최종 명령)
        ↓
UART 브릿지 → STM32
```

### Pi Gateway → 패트롤 노드

```
Pi Gateway
    │
    ├─→ /arm/done (std_msgs/Bool)
    │   └─ 로봇팔 작업 완료 신호
    │
    ├─→ /dock/done (std_msgs/Bool)
    │   └─ 도킹 완료 신호
    │
    └─→ /servo/done (std_msgs/Bool)
        └─ 서보 작업 완료 신호
            ↓
패트롤 노드 (팀원)
    └─ 작업 완료 신호 수신 후 다음 단계 진행
```

## Pi Gateway 구현 상태

### ✅ 이미 구현됨

#### 1. `/cmd_vel_nav` 구독

**위치**: `src/ros_cmdvel.py` - `CmdVelMux` 클래스

```python
# 패트롤 노드의 자동 주행 명령 구독
self.sub_nav = self.create_subscription(
    Twist, 
    "/cmd_vel_nav", 
    self.on_cmd_vel_nav, 
    10
)
```

**동작**:
- 패트롤 노드가 발행하는 `/cmd_vel_nav` 토픽 구독
- 웹 대시보드 명령(`/cmd_vel_teleop`)과 통합
- 최종 명령을 `/cmd_vel` 또는 `/cmd_vel_out`으로 발행

#### 2. `/gateway/allow_strafe` 구독

**위치**: `src/ros_cmdvel.py` - `CmdVelMux` 클래스

```python
# 횡이동 허용 여부 구독
self.sub_allow_strafe = self.create_subscription(
    Bool,
    "/gateway/allow_strafe",
    self.on_allow_strafe,
    10
)
```

**동작**:
- 패트롤 노드가 발행하는 횡이동 허용 여부 수신
- 횡이동 허용 시 `vy` (횡이동 속도) 사용 가능

#### 3. 작업 완료 신호 발행 (일부 구현됨)

**위치**: `src/ros_cmdvel.py` - `ArmControlBridge`, `FeedControlBridge` 등

**현재 상태**:
- 로봇팔 작업 완료: `ArmControlBridge`에서 처리 (확인 필요)
- 도킹 완료: 구현 필요
- 서보 작업 완료: 구현 필요

## 통합 방식

### 현재 구현: CmdVelMux 사용

```
웹 대시보드
    ↓
/cmd_vel_teleop (geometry_msgs/Twist)
    ↓
패트롤 노드
    ↓
/cmd_vel_nav (geometry_msgs/Twist)
    ↓
    ┌─────────────┐
    │ CmdVelMux   │ ← Pi Gateway 내부
    │ (통합)      │
    └─────────────┘
         ↓
    /cmd_vel (최종 명령)
         ↓
UART 브릿지 → STM32
```

**장점**:
- Pi Gateway 내부에서 통합 관리
- 우선순위 제어 가능 (teleop vs nav)
- twist_mux 없이도 동작 가능

## 작업 완료 신호 발행 가이드

### 1. 로봇팔 작업 완료 (`/arm/done`)

**발행 시점**:
- 로봇팔 작업이 완료되었을 때
- STM32에서 작업 완료 신호를 받았을 때

**구현 위치**: `src/ros_cmdvel.py` - `ArmControlBridge`

**예시 코드**:
```python
# 로봇팔 작업 완료 시
arm_done_msg = Bool()
arm_done_msg.data = True
self.arm_done_pub.publish(arm_done_msg)
```

### 2. 도킹 완료 (`/dock/done`)

**발행 시점**:
- 도킹 작업이 완료되었을 때
- 아루코 마커 정렬 완료 시

**구현 위치**: `src/ros_cmdvel.py` - 새로 추가 필요

**예시 코드**:
```python
# 도킹 완료 시
dock_done_msg = Bool()
dock_done_msg.data = True
self.dock_done_pub.publish(dock_done_msg)
```

### 3. 서보 작업 완료 (`/servo/done`)

**발행 시점**:
- 서보 작업이 완료되었을 때
- STM32에서 서보 완료 신호를 받았을 때

**구현 위치**: `src/ros_cmdvel.py` - 새로 추가 필요

**예시 코드**:
```python
# 서보 작업 완료 시
servo_done_msg = Bool()
servo_done_msg.data = True
self.servo_done_pub.publish(servo_done_msg)
```

## 패트롤 노드와의 통신 확인

### 토픽 확인 명령어

```bash
# ROS2 도메인 설정
export ROS_DOMAIN_ID=1

# 패트롤 노드가 발행하는 토픽 확인
ros2 topic echo /cmd_vel_nav
ros2 topic echo /gateway/allow_strafe

# Pi Gateway가 발행하는 토픽 확인
ros2 topic echo /arm/done
ros2 topic echo /dock/done
ros2 topic echo /servo/done

# 최종 명령 확인
ros2 topic echo /cmd_vel
```

### 통신 상태 확인

```bash
# 토픽 발행 빈도 확인
ros2 topic hz /cmd_vel_nav
ros2 topic hz /gateway/allow_strafe

# 토픽 목록 확인
ros2 topic list | grep -E "(cmd_vel|allow_strafe|done)"
```

## 주의사항

### 1. 토픽명 일치 확인

패트롤 노드가 사용하는 토픽명과 Pi Gateway가 구독/발행하는 토픽명이 일치해야 합니다:

| 토픽명 | 패트롤 노드 | Pi Gateway | 상태 |
|--------|------------|-----------|------|
| `/cmd_vel_nav` | 발행 | 구독 | ✅ |
| `/gateway/allow_strafe` | 발행 | 구독 | ✅ |
| `/arm/done` | 구독 | 발행 | ⚠️ 확인 필요 |
| `/dock/done` | 구독 | 발행 | ⚠️ 구현 필요 |
| `/servo/done` | 구독 | 발행 | ⚠️ 구현 필요 |

### 2. QoS 설정 일치

ROS2 토픽의 QoS 설정이 일치해야 합니다:

```python
# 패트롤 노드와 동일한 QoS 사용
from rclpy.qos import QoSProfile, ReliabilityPolicy, HistoryPolicy

qos_profile = QoSProfile(
    reliability=ReliabilityPolicy.RELIABLE,  # 또는 BEST_EFFORT
    history=HistoryPolicy.KEEP_LAST,
    depth=10
)
```

### 3. 통합 우선순위

웹 대시보드 명령과 패트롤 명령의 우선순위:

- **teleop 모드**: 웹 대시보드 명령 우선
- **auto 모드**: 패트롤 명령 우선

현재 `CmdVelMux`에서 모드에 따라 우선순위를 관리합니다.
