## 🔧 프레임 구조

모든 UART 프레임은 다음 구조를 따릅니다:

```
[STX1] [STX2] [MSG_ID] [LENGTH] [PAYLOAD...] [CHECKSUM]
```

| 필드 | 크기 | 설명 | 값 |
|------|------|------|-----|
| STX1 | 1 byte | 시작 바이트 1 | `0xAA` |
| STX2 | 1 byte | 시작 바이트 2 | `0x55` |
| MSG_ID | 1 byte | 메시지 ID | 아래 표 참조 |
| LENGTH | 1 byte | Payload 길이 (바이트) | 0~255 |
| PAYLOAD | N bytes | 실제 데이터 | 메시지별 상이 |
| CHECKSUM | 1 byte | XOR 체크섬 | 아래 계산식 참조 |

**프레임 최소 크기**: 5 bytes (payload 없을 때)
**프레임 최대 크기**: 260 bytes (payload 최대 255 bytes)

## 📤 SBC → STM32 명령

### 1. ID_CMD_VEL (0x01) - 이동 명령

**목적**: 로봇 이동 속도 제어 (메카넘 휠)

**Payload 형식**:
```
[float32: vx] [float32: vy] [float32: wz]
```

**총 크기**: 12 bytes (3 × float32)

**변수 설명**:
| 변수 | 타입 | 범위 | 단위 | 설명 |
|------|------|------|------|------|
| `vx` | float32 | -1.0 ~ 1.0 | m/s | 전진/후진 속도 (양수=앞, 음수=뒤) |
| `vy` | float32 | -1.0 ~ 1.0 | m/s | 좌우 이동 속도 (양수=왼쪽, 음수=오른쪽) |
| `wz` | float32 | -2.0 ~ 2.0 | rad/s | 회전 속도 (양수=반시계, 음수=시계) |

**엔디안**: Little-endian (`<fff`)

**예제 값**:
- 전진: `vx=0.15, vy=0.0, wz=0.0`
- 좌측 이동: `vx=0.0, vy=0.15, wz=0.0`
- 제자리 회전: `vx=0.0, vy=0.0, wz=0.7`
- 정지: `vx=0.0, vy=0.0, wz=0.0`

**STM32 처리**:
- 메카넘 휠 역기구학으로 4륜 속도 계산
- 모터 PWM 제어

---

### 2. ID_FEED (0x05) - 급식

**목적**: 사용자가 웹 UI에서 직접 급식을 하는 기능 (서보모터로 개폐통 열어서 급식)

**Payload 형식**:
```
[uint8: level]
```

**총 크기**: 1 byte

**변수 설명**:
| 변수 | 타입 | 범위 | 값 | 설명 |
|------|------|------|-----|------|
| `level` | uint8 | 1~3 | 1 | 급식 레벨 1 |
| | | | 2 | 급식 레벨 2 |
| | | | 3 | 급식 레벨 3 |

**STM32 처리**:
- 서보모터로 개폐통을 열어서 급식
- 완료 시 **STATUS(0x84, JOB_COMPLETE, 급식)** 전송 권장 (Pi → BE 리포트용)

---

### 3. ID_ARM_START (0x06) - 로봇팔 동작 시작/정지

**목적**: 로봇팔 동작 시작 또는 정지 신호

**Payload 형식**:
```
[uint8: action_id]
```

**총 크기**: 1 byte

**변수 설명**:
| 변수 | 타입 | 범위 | 값 | 설명 |
|------|------|------|-----|------|
| `action_id` | uint8 | 0~255 | 0 | 로봇팔 정지 (변 치우기 완료 또는 중지) |
| | | | 1 | 변 치우기 시작 (젯슨의 모방학습 로봇팔이 자율로 4구역 처리) |
| | | | 2+ | 기타 로봇팔 동작 시작 (예비) |

**동작 흐름**:
1. `action_id = 1` 수신 → **바퀴 모터 잠금** (로봇팔 동작 중 이동 방지)
2. 젯슨의 모방학습 로봇팔이 변 치우기 작업 수행 (자율 제어)
3. `action_id = 0` 수신 → **바퀴 모터 해제** (이동 가능)

**STM32 처리**:
- `action_id != 0`: 바퀴 모터 잠금 (PWM 출력 차단 또는 제로 속도 고정)
- `action_id = 0`: 바퀴 모터 해제 (정상 제어 재개)
- **주의**: STM32는 로봇팔을 직접 제어하지 않음 (젯슨이 모방학습 모델로 제어)
- 변 치우기/급수 완료는 **젯슨이 ROS `arm/job_complete`** 로 Pi에 알림 (STM32 STATUS 아님)

---

### 4. ID_ARM_POSITION_CORRECT (0x07) - 로봇 위치 보정

**목적**: 로봇팔 동작 중 위치가 어긋났을 때 미세 조정

**Payload 형식**:
```
[float32: dx] [float32: dy] [float32: dz]
```

**총 크기**: 12 bytes (3 × float32)

**변수 설명**:
| 변수 | 타입 | 범위 | 단위 | 설명 |
|------|------|------|------|------|
| `dx` | float32 | -0.5 ~ 0.5 | m | x축 이동 거리 (양수=앞, 음수=뒤) |
| `dy` | float32 | -0.5 ~ 0.5 | m | y축 이동 거리 (양수=왼쪽, 음수=오른쪽) |
| `dz` | float32 | -0.5 ~ 0.5 | m | z축 이동 거리 (양수=위, 음수=아래, 현재 사용 안 함) |

**엔디안**: Little-endian (`<fff`)

**예제 값**:
- 앞으로 5cm: `dx=0.05, dy=0.0, dz=0.0`
- 왼쪽으로 3cm: `dx=0.0, dy=0.03, dz=0.0`

**STM32 처리**:
- `dx`, `dy`는 바퀴 모터로 이동
- `dz`는 현재 사용 안 함 (로봇팔 제어는 STM32가 하지 않음)

---

### 5. ID_CHURU (0x08) - 츄르 주기

**목적**: 사용자가 웹 UI에서 직접 츄르를 주는 기능 (주사기 밀기)

**Payload 형식**:
```
[uint8: enable]
```

**총 크기**: 1 byte

**변수 설명**:
| 변수 | 타입 | 범위 | 값 | 설명 |
|------|------|------|-----|------|
| `enable` | uint8 | 0~1 | 0 | 정지 |
| | | | 1 | 츄르 주기 (주사기 밀기) |

**동작 흐름**:
1. 사용자가 웹 UI에서 츄르 주기 버튼 클릭
2. Pi Gateway가 `ID_CHURU` 전송 (`enable=1`)
3. STM32가 서보모터로 주사기를 밀어서 츄르 배출
4. 완료 시 **STATUS(0x84, JOB_COMPLETE, 츄르)** 전송 권장 (Pi → BE 리포트용)
5. 정지 시 `enable=0` 전송 (또는 자동 정지)

**STM32 처리**:
- `enable = 1`일 때 주사기를 밀어서 츄르 배출
- `enable = 0`일 때 정지

---

### 6. ID_ARM_WATER (0x09) - 급수 동작

**목적**: 급수 단계별 제어 신호 (모방학습 로봇팔이 자율로 물 뜨고 버리는 과정 수행)

**Payload 형식**:
```
[uint8: water_action]
```

**총 크기**: 1 byte

**변수 설명**:
| 변수 | 타입 | 범위 | 값 | 설명 |
|------|------|------|-----|------|
| `water_action` | uint8 | 0~4 | 0 | 위치 이동/정지 (바퀴 해제, 이동 가능) |
| | | | 1 | 물그릇 집기 (바퀴 해제 - 위치 이동 중) |
| | | | 2 | 물 버리기 (바퀴 잠금 - 아루코 마커로 정밀 위치 조정 완료 후) |
| | | | 3 | 물 받기 (바퀴 잠금 - 아루코 마커로 정밀 위치 조정 완료 후) |
| | | | 4 | 물그릇 두기 (바퀴 잠금 - 로봇팔 동작 중) |

**동작 흐름** (젯슨의 모방학습 로봇팔이 자율로 수행):
1. `water_action = 0` 수신 → 물그릇 위치로 이동 (바퀴 해제)
2. `water_action = 1` 수신 → 물그릇 집기 (바퀴 해제 - 위치 이동 중)
3. `water_action = 0` 수신 → 화장실 위치로 이동 (바퀴 해제)
4. 화장실 도착 → 아루코 마커로 정밀 위치 조정 (바퀴 해제)
5. 정밀 위치 조정 완료 → `water_action = 2` 수신 → 물 버리기 (바퀴 잠금)
6. `water_action = 0` 수신 → 서스펜서 위치로 이동 (바퀴 해제)
7. 서스펜서 도착 → 아루코 마커로 정밀 위치 조정 (바퀴 해제)
8. 정밀 위치 조정 완료 → `water_action = 3` 수신 → 물 받기 (바퀴 잠금)
9. `water_action = 0` 수신 → 물그릇 위치로 이동 (바퀴 해제)
10. `water_action = 4` 수신 → 물그릇 두기 (바퀴 잠금)
11. `water_action = 0` 수신 → 완료 (바퀴 해제)

**STM32 처리**:
- `water_action = 2, 3, 4`: 바퀴 모터 잠금 (PWM 출력 차단 또는 제로 속도 고정)
- `water_action = 0, 1`: 바퀴 모터 해제 (정상 제어 재개)
- **주의**: STM32는 로봇팔을 직접 제어하지 않음. 급수 완료는 **젯슨이 `arm/job_complete`** 로 Pi에 알림.

**목적**: 비상 상황 시 즉시 정지

**Payload 형식**:
```
[uint8: value]
```

**총 크기**: 1 byte

**변수 설명**:
| 변수 | 타입 | 범위 | 값 | 설명 |
|------|------|------|-----|------|
| `value` | uint8 | 0~1 | 0 | 비상정지 해제 |
| | | | 1 | 비상정지 활성화 |

**STM32 처리**:
- `value = 1`: 모든 모터 즉시 정지 (PWM 출력 차단)

## 📥 STM32 → SBC 텔레메트리

### 1. ID_BATTERY (0x81) - 배터리 상태

**목적**: 배터리 전압, 잔량, 충전 상태 전송

**Payload 형식**:
```
[uint16: vbat_mV] [uint8: soc] [uint8: charging] [uint8: error_code]
```

**총 크기**: 5 bytes

**변수 설명**:
| 변수 | 타입 | 범위 | 단위 | 설명 |
|------|------|------|------|------|
| `vbat_mV` | uint16 | 0~65535 | mV | 배터리 전압 (밀리볼트) |
| `soc` | uint8 | 0~100 | % | 배터리 잔량 (State of Charge) |
| `charging` | uint8 | 0~1 | - | 충전 중 여부 (0=충전 안 함, 1=충전 중) |
| `error_code` | uint8 | 0~255 | - | 에러 코드 (0=정상) |

**엔디안**: Little-endian (`<HBBB`)

---

### 2. ID_ENCODER (0x82) - 엔코더 틱

**목적**: 메카넘 휠 4륜의 누적 엔코더 틱 전송

**Payload 형식**:
```
[int32: enc_fl] [int32: enc_fr] [int32: enc_rl] [int32: enc_rr]
```

**총 크기**: 16 bytes (4 × int32)

**변수 설명**:
| 변수 | 타입 | 설명 |
|------|------|------|
| `enc_fl` | int32 | 앞왼쪽 (Front Left) 누적 틱 |
| `enc_fr` | int32 | 앞오른쪽 (Front Right) 누적 틱 |
| `enc_rl` | int32 | 뒤왼쪽 (Rear Left) 누적 틱 |
| `enc_rr` | int32 | 뒤오른쪽 (Rear Right) 누적 틱 |

**엔디안**: Little-endian (`<iiii`)

---

### 3. ID_IMU (0x83) - IMU 센서 데이터

**목적**: 자이로스코프/가속도계 데이터 전송

**Payload 형식**:
```
[float32: yaw] [float32: pitch] [float32: roll] [float32: acc_x] [float32: acc_y] [float32: acc_z]
```

**총 크기**: 24 bytes (6 × float32)

**변수 설명**:
| 변수 | 타입 | 범위 | 단위 | 설명 |
|------|------|------|------|------|
| `yaw` | float32 | -π ~ π | rad | yaw 각도 (수평 회전, 0=북쪽) |
| `pitch` | float32 | -π/2 ~ π/2 | rad | pitch 각도 (앞뒤 기울기) |
| `roll` | float32 | -π/2 ~ π/2 | rad | roll 각도 (좌우 기울기) |
| `acc_x` | float32 | -9.8 ~ 9.8 | m/s² | x축 가속도 |
| `acc_y` | float32 | -9.8 ~ 9.8 | m/s² | y축 가속도 |
| `acc_z` | float32 | -9.8 ~ 9.8 | m/s² | z축 가속도 |

**엔디안**: Little-endian (`<ffffff`)

---

### 4. ID_STATUS (0x84) - 상태/작업 완료·실패·에러

**목적**: 작업 완료·실패·에러·상태 변경을 Pi에 알림 (BE 리포트/로그용)

**Payload 형식**:
```
[uint8: status_type] [uint8: status_code] [uint8: flags]
```

**총 크기**: 3 bytes

**변수 설명**:

| 변수 | 타입 | 설명 |
|------|------|------|
| `status_type` | uint8 | 상태 타입 (아래 표) |
| `status_code` | uint8 | 상태/작업 코드 (status_type별 상이) |
| `flags` | uint8 | 상태 플래그 bitmask (아래 표) |

**status_type (상태 타입)**:

| 값 | 이름 | 설명 |
|----|------|------|
| `0x01` | JOB_COMPLETE | 작업 완료 |
| `0x02` | JOB_FAILED | 작업 실패 |
| `0x03` | ERROR | 에러 발생 |
| `0x04` | STATE | 상태 변경 |

**status_code (status_type = 0x01 작업 완료)**:

| 값 | 작업 | STM32 제어 여부 |
|----|------|----------------|
| `0x01` | 급식 완료 | ✅ STM32가 서보 제어 → 완료 감지 가능 |
| `0x02` | 츄르 주기 완료 | ✅ STM32가 서보 제어 → 완료 감지 가능 |

**참고**: 변 치우기/급수는 젯슨이 로봇팔 제어하므로 STM32는 완료를 알 수 없음 → 젯슨이 `arm/job_complete`로 Pi에 알림

**status_code (status_type = 0x02 작업 실패)**:

| 값 | 작업 | STM32 제어 여부 |
|----|------|----------------|
| `0x01` | 급식 실패 | ✅ STM32가 감지 가능 |
| `0x02` | 츄르 주기 실패 | ✅ STM32가 감지 가능 |

**참고**: 변 치우기/급수는 젯슨이 제어하므로 STM32는 실패를 알 수 없음 → 젯슨이 `arm/job_complete`로 Pi에 알림

**status_code (status_type = 0x03 에러)**:

| 값 | 에러 타입 |
|----|----------|
| `0x01` | 모터 오류 |
| `0x02` | 센서 오류 |
| `0x03` | 통신 타임아웃 |
| `0x04` | 과부하 |

**flags (상태 플래그, bitmask)**:

| 비트 | 플래그 | 설명 |
|------|--------|------|
| `0x01` | FLAG_ARM_ACTIVE | 로봇팔 동작 중 |
| `0x02` | FLAG_WHEELS_LOCKED | 바퀴 잠금 |
| `0x04` | FLAG_EMERGENCY_STOP | 비상정지 활성화 |

**STM32가 STATUS를 보내는 작업**:
- **급식(ID_FEED)**: STM32가 서보모터로 개폐통 제어 → 완료 시 `STATUS(0x01, 0x01, 0x00)` 전송 권장.
- **츄르(ID_CHURU)**: STM32가 서보모터로 주사기 제어 → 완료 시 `STATUS(0x01, 0x02, 0x00)` 전송 권장.

**STM32가 STATUS를 보내지 않는 작업**:
- **변 치우기**, **급수**: 젯슨이 로봇팔 제어 → 완료는 **젯슨이 ROS `arm/job_complete`** 로 Pi에 알림. STM32는 완료를 알 수 없으므로 STATUS를 보내지 않음.

**Pi Gateway 처리**:
- `status_type = 0x01`: 해당 job_type을 success로 JOB_EVENTS 업데이트 → BE `/api/logs` 전송
- `status_type = 0x02`: 해당 job_type을 failed로 업데이트 → BE 전송
- `status_type = 0x03`: 에러 로그 출력
- `status_type = 0x04`: flags 기반 상태 참고용

**예제 Payload**:
- 급식 완료: `[0x01, 0x01, 0x00]` (STM32가 서보 제어 완료 감지)
- 츄르 완료: `[0x01, 0x02, 0x00]` (STM32가 서보 제어 완료 감지)
- 급식 실패: `[0x02, 0x01, 0x00]` (STM32가 에러 감지)
- 츄르 실패: `[0x02, 0x02, 0x00]` (STM32가 에러 감지)

**주의**: 변 치우기/급수는 젯슨이 로봇팔 제어하므로 STM32는 완료를 알 수 없음 → 젯슨이 `arm/job_complete`로 Pi에 알림. STATUS 메시지에는 사용하지 않음.

## 🔐 체크섬 계산

체크섬은 **XOR 연산**을 사용합니다.

**계산식**:
```c
uint8_t checksum = (msg_id & 0xFF) ^ (length & 0xFF);
for (int i = 0; i < length; i++) {
    checksum ^= payload[i];
}
checksum = checksum & 0xFF;
```

## ⚠️ 주의사항

1. **바퀴 잠금**:
   - `ID_ARM_START` (action_id ≠ 0) 수신 시 바퀴 모터 잠금
   - `ID_ARM_WATER` (water_action = 2, 3, 4) 수신 시 바퀴 모터 잠금
   - `action_id = 0` 또는 `water_action = 0` 수신 시 바퀴 모터 해제

2. **엔디안**:
   - 모든 다중 바이트 데이터는 **Little-endian** 형식

3. **체크섬 검증**:
   - 체크섬이 맞지 않으면 프레임을 무시하고 다음 프레임 대기

4. **프레임 동기화**:
   - STX1 (0xAA) 다음에 STX2 (0x55)가 오지 않으면 처음부터 다시 시작

5. **타임아웃**:
   - 프레임 수신 중 타임아웃 발생 시 파서 초기화

6. **STATUS (0x84)**:
   - **STM32가 직접 제어하는 작업만** STATUS로 보냄: **급식(0x01)**, **츄르(0x02)**
   - **변 치우기**, **급수**는 젯슨이 로봇팔 제어하므로 STM32는 완료를 알 수 없음 → 젯슨이 `arm/job_complete`로 Pi에 알림
