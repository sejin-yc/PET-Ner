# 구조 1 적용 완료 가이드

## 개요

ROS2 폴더의 UART 브릿지를 사용하도록 변경을 완료했습니다.

---

## 변경 사항 요약

### 1. CmdVelMux 변경 ✅

**변경 내용**:
- 발행 토픽: `cmd_vel_out` → `/cmd_vel_joy` (twist_mux 입력)
- 구독 토픽: `/cmd_vel` → `/cmd_vel_nav` (패트롤 노드 명령)

**이유**:
- ROS2 폴더의 `twist_mux`가 `/cmd_vel_joy`를 구독하여 통합
- `twist_mux`가 최종적으로 `/cmd_vel`을 발행
- `twist_to_stm_uart_bridge.py`가 `/cmd_vel`을 구독하여 UART 전송
- **충돌 방지**: Pi Gateway와 ROS2 폴더를 동시에 실행해도 문제 없음

---

### 2. RosCmdVelBridge 변경 ✅

**변경 내용**:
- `/cmd_vel_out` 구독 제거
- cmd_vel UART 전송 로직 제거

**유지된 기능**:
- ✅ Heartbeat 전송 (STM32 워치독)
- ✅ Feed 명령 전송
- ✅ Estop 처리
- ✅ 로봇팔 동작 중 바퀴 잠금 (cmd_vel=0 직접 전송)

---

### 3. HomingController 변경 ✅

**변경 내용**:
- 발행 토픽: `/cmd_vel` → `/cmd_vel_nav`

**이유**:
- CmdVelMux가 통합하여 `/cmd_vel`로 발행
- 패트롤 노드와 동일한 토픽 사용

---

## 최종 통신 구조

```
┌─────────────────┐
│  웹 대시보드     │
└────────┬────────┘
         │ WebSocket
         ↓
┌─────────────────┐
│  Pi Gateway     │
│  (CmdVelMux)    │
│  ├─ /cmd_vel_teleop 구독 (웹)
│  ├─ /cmd_vel_nav 구독 (패트롤 + 홈 이동)
│  └─ /cmd_vel_joy 발행 (twist_mux 입력)
└────────┬────────┘
         │ ROS2 토픽
         ↓
┌─────────────────┐
│  ROS2 폴더      │
│  ├─ twist_mux   │
│  │  ├─ /cmd_vel_nav 구독 (패트롤, priority 10)
│  │  ├─ /cmd_vel_joy 구독 (Pi Gateway, priority 50)
│  │  ├─ /cmd_vel_aruco 구독 (아루코, priority 80)
│  │  └─ /cmd_vel 발행 (통합, 우선순위 적용)
│  └─ twist_to_stm_uart_bridge
│     ├─ /cmd_vel 구독
│     └─ UART 전송
└────────┬────────┘
         │ UART
         ↓
┌─────────────────┐
│  STM32          │
└─────────────────┘
```

**우선순위**:
- 아루코 마커 (80) > 웹 대시보드 (50) > 패트롤 (10)

## Pi Gateway의 역할

### ✅ 담당하는 것

1. **웹 대시보드 통신**
   - WebSocket 서버 운영
   - 수동 조작 명령 수신

2. **명령 통합 (Mux)**
   - 웹 명령 (`/cmd_vel_teleop`)
   - 패트롤 명령 (`/cmd_vel_nav`)
   - 홈 이동 명령 (`/cmd_vel_nav`)
   - → `/cmd_vel` 발행 (통합)

3. **UART 제어 (cmd_vel 제외)**
   - Heartbeat 전송
   - Feed 명령 전송
   - Estop 처리
   - 로봇팔 바퀴 잠금 (cmd_vel=0)

4. **WebRTC 스트리밍 제어**
   - 프론트엔드 요청 수신
   - 스트리밍 프로세스 관리

5. **MQTT 브릿지**
   - 백엔드와 통신

6. **작업 완료 신호 발행**
   - `/arm/done`, `/dock/done`, `/servo/done`

### ❌ 담당하지 않는 것

1. **cmd_vel UART 전송**
   - ROS2 폴더의 `twist_to_stm_uart_bridge.py`가 담당

## 실행 방법

### 1. ROS2 폴더 브릿지 실행 (필수)

```bash
# ROS2 워크스페이스에서
cd /path/to/S14P11C203-ROS2/[Local] mission
source /opt/ros/humble/setup.bash
export ROS_DOMAIN_ID=1
ros2 run priority twist_to_stm_uart_bridge
```

### 2. Pi Gateway 실행

```bash
# 도커 실행
docker-compose up -d

# 또는 로컬 실행
python3 -m src.main
```

### 3. 통신 확인

```bash
# Pi Gateway가 발행하는 토픽 확인
ros2 topic echo /cmd_vel

# ROS2 폴더 브릿지가 구독하는지 확인
ros2 topic hz /cmd_vel
```

## 주의사항

### 1. ROS2 폴더 브릿지 실행 필수

**중요**: ROS2 폴더의 `twist_to_stm_uart_bridge.py`가 실행되어야 합니다.

**없으면**:
- `/cmd_vel` 토픽은 발행되지만 UART로 전송되지 않음
- STM32가 명령을 받지 못함

### 2. 토픽명 확인

**Pi Gateway**:
- `/cmd_vel` 발행 (통합된 명령)
- `/cmd_vel_nav` 구독 (패트롤 + 홈 이동)

**ROS2 폴더 브릿지**:
- `/cmd_vel` 구독 (Pi Gateway가 발행)

**일치 확인**:
- ROS2 폴더 브릿지의 `twist_topic` 파라미터가 `/cmd_vel`인지 확인

### 3. 로봇팔 바퀴 잠금

**현재 구현**:
- 로봇팔 동작 중 Pi Gateway가 직접 `cmd_vel=0`을 UART로 전송
- ROS2 폴더 브릿지를 통하지 않고 직접 전송

**이유**:
- 즉시 바퀴를 잠가야 하므로 직접 전송 필요
