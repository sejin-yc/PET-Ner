# 카메라 스트림 i14c203.p.ssafy.io 배포 가이드

**목적**: 젯슨 MJPEG → i14c203.p.ssafy.io 프록시 → 웹 대시보드

---

## 0. 제일 중요한 주의점 3개

### 1) Mixed Content (HTTPS 페이지 + HTTP 스트림)

대시보드가 `https://i14c203.p.ssafy.io` 인데 스트림을 `http://i14c203.p.ssafy.io:8889/cam`로 걸면 **브라우저가 막을 수 있음** (특히 iframe).

- **해결**: 같은 HTTPS(443) 아래 경로로 프록시하는 게 제일 안전함.
- **권장 URL**: `https://i14c203.p.ssafy.io/api/robot/stream`
- **비권장**: `http://i14c203.p.ssafy.io:8889/cam` (HTTP)

8889를 꼭 써야 하면, 그 포트도 TLS(https)로 열어야 함. 그럴 바엔 443 path로 붙이는 게 더 쉬움.

### 2) Nginx 프록시에서 버퍼링 끄기

MJPEG는 "긴 응답을 계속 흘리는" 방식이라 Nginx가 버퍼링/타임아웃 걸면 **중간에 끊김**.

- **해결**: `proxy_buffering off`, `proxy_read_timeout` 크게 설정 필요.

### 3) 서버(i14c203)가 Jetson(192.168.100.253)로 라우팅 가능해야 함

i14c203 서버가 `192.168.100.253:8080`으로 직접 curl이 돼야 프록시가 동작.

```bash
# i14c203 서버에서 실행
curl -v http://192.168.100.253:8080/stream.mjpeg
```

이게 먼저 성공해야 함.

## 1. Jetson (192.168.100.253) Flask MJPEG 서버

핵심은 **0.0.0.0 바인딩** (외부에서 접근 가능).

`cat_detection_service.py` 또는 `jetson_mjpeg_server.py` 실행:

```bash
# 젯슨
cd /path/to/pi_gateway
pip install flask opencv-python
export STREAM_PORT=8080
python3 scripts/cat_detection_service.py --ckpt ... --yolo_pose ...
# 또는
python3 scripts/jetson_mjpeg_server.py
```

- 라우트: `/stream.mjpeg`
- 포트: 8080 (기본)
- 젯슨 방화벽: 8080 열려 있어야 함

## 2. i14c203 서버 Nginx 프록시 설정 (권장: 443 path)

`/api/robot/stream` 으로 프록시. Nginx server 블록 안에 location 추가:

```nginx
location /api/robot/stream {
    proxy_pass http://192.168.100.253:8080/stream.mjpeg;

    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header Connection "";

    # MJPEG는 버퍼링/캐시 끄는 게 중요
    proxy_buffering off;
    proxy_cache off;

    # 끊김 방지
    proxy_read_timeout 3600s;
    proxy_send_timeout 3600s;

    add_header Cache-Control "no-store";
}
```

프론트:

```tsx
const STREAM_URL = "https://i14c203.p.ssafy.io/api/robot/stream";
<img src={STREAM_URL} alt="실시간 카메라" />
```

## 3. 8889/cam → 젯슨 (내부 데모용)

가능은 한데, 8889 포트가 TLS(https) 아니면 Mixed Content로 막힐 수 있음.

```nginx
server {
    listen 8889;
    server_name i14c203.p.ssafy.io;

    location /cam {
        proxy_pass http://192.168.100.253:8080/stream.mjpeg;
        proxy_buffering off;
        proxy_read_timeout 3600s;
    }
}
```

- 대시보드가 HTTPS면 브라우저에서 이슈 날 수 있음.
- **권장**: 443 path 방식 (`/api/robot/stream`).

## 4. 점검 체크리스트 (순서대로)

| # | 확인 | 명령 |
|---|------|------|
| 1 | 젯슨에서 MJPEG 동작 | `curl -v http://127.0.0.1:8080/stream.mjpeg` |
| 2 | i14c203 서버에서 젯슨 접근 | `curl -v http://192.168.100.253:8080/stream.mjpeg` |
| 3 | Nginx 프록시 적용 후 확인 | `curl -vk https://i14c203.p.ssafy.io/api/robot/stream` |
| 4 | 브라우저에서 img로 확인 | `<img src="https://i14c203.p.ssafy.io/api/robot/stream" />` |

MJPEG에는 **iframe보다 img가 더 안정적**.

## 5. ROS 토픽은 "선택"

- `PUBLISH_CAMERA_TOPIC=1`: `/camera/image_compressed` 발행 (Pi 경유 필요할 때)
