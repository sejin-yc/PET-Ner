# 고양이 탐지 → 상태 분석 → 웹 전송 로직 설계

라즈베리 파이 카메라에서 고양이 탐지 시 액션/감정 분석 후 웹으로 전송하고, 15초 영상 기록 기능을 위한 설계 문서입니다.

### 1.3 infer_cat.py

| 항목 | 내용 |
|------|------|
| **위치** | `pi_gateway/src/infer_cat.py` |
| **역할** | 비디오 파일 입력 → YOLO 포즈 + Swin 액션/감정 → 결과 오버레이 영상 출력 (cat_detection_service에서 import) |
| **핵심 흐름** | 1) YOLO로 고양이 탐지 2) K프레임 버퍼로 Swin 분류 3) cls_hz(기본 5회/초)마다 액션/감정 추론 |

**주요 인자**:
- `--ckpt`: Swin 체크포인트 (action/emotion)
- `--yolo_pose`: YOLO pose 모델 (.pt)
- `--video`: 입력 비디오
- `--K`: 프레임 버퍼 크기 (기본 8)
- `--cls_hz`: 초당 분류 횟수 (기본 5)

## 2. 아키텍처 설계

### 2.1 전체 흐름

```
[ Pi 카메라 ]
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│  CatDetectionService (별도 프로세스 또는 ROS 노드)            │
│                                                             │
│  1. 실시간 캡처 (OpenCV VideoCapture)                        │
│  2. YOLO pose → 고양이 탐지 (bbox, keypoints)                │
│  3. 탐지 시 → K프레임 버퍼 채움 → Swin 액션/감정 분류        │
│  4. 결과 → MQTT /sub/robot/cat_state 발행                    │
│  5. 탐지 시 → 15초 영상 버퍼 시작/저장 → 웹/스토리지 전송    │
└─────────────────────────────────────────────────────────────┘
      │
      ├─ MQTT /sub/robot/cat_state ──► Spring Backend ──► 웹 대시보드
      │
      └─ 15초 영상 ──► (파일 저장 또는 HTTP 업로드) ──► 웹에서 재생
```

### 2.2 Pi Gateway와의 관계

| 컴포넌트 | 역할 | Pi Gateway 연동 |
|----------|------|-----------------|
| **Pi Gateway** | 제어, 텔레메트리, MQTT 브릿지 | MQTT 브릿지가 `/sub/robot/status` 등 발행 |
| **CatDetection** | 고양이 탐지 + 상태 분석 | **별도 프로세스** 또는 **Pi Gateway에 HTTP API 추가** |
| **연동 방식** | | CatDetection → MQTT 직접 발행 또는 Pi Gateway API → MQTT 브릿지 |

**권장**: CatDetection을 **별도 Python 프로세스**로 두고, 결과를 **MQTT로 직접 발행**  
- Spring/웹은 이미 MQTT 구독 구조  
- Pi Gateway와 결합도 낮게 유지

## 3. MQTT 메시지 스펙 (웹 전송)

### 3.1 고양이 상태 토픽

**토픽**: `/sub/robot/cat_state` (또는 팀 합의 토픽)

**페이로드 예시** (JSON):
```json
{
  "timestamp": 1738000000.123,
  "detected": true,
  "action": "LOAF_SIT",
  "action_ko": "발을 숨기고 웅크리고 앉음",
  "emotion": "CALM",
  "emotion_ko": "편안/안정",
  "confidence": {
    "action": 0.85,
    "emotion": 0.72
  },
  "bbox": [100, 150, 400, 450],
  "video_clip_url": "https://..." 
}
```

- `detected: false` → 고양이 미탐지
- `video_clip_url` → 15초 영상 업로드 후 URL (선택)

### 3.2 발행 주기

| 상황 | 발행 주기 |
|------|-----------|
| 고양이 **탐지** | 즉시 + 상태 변경 시 |
| 고양이 **미탐지** | 1~2초마다 `detected: false` (선택) |

## 4. 15초 영상 녹화 로직

### 4.1 구현 방식

```python
# 탐지 시점을 기준으로 앞뒤 버퍼
RECORD_SEC = 15
FPS = 15  # 또는 캡처 FPS에 맞춤

# 탐지됐을 때
if cat_detected and not recording:
    ring_buffer = deque(maxlen=RECORD_SEC * FPS)  # 이미 저장된 최근 15초
    recording = True
    record_start = time.time()

# 녹화 중
if recording:
    ring_buffer.append(frame_copy)
    if time.time() - record_start >= RECORD_SEC:
        # 15초 분량 저장
        save_video(ring_buffer, f"cat_{timestamp}.mp4")
        # 업로드 또는 로컬 경로 전달
        recording = False
```

### 4.2 선행 버퍼 (선택)

탐지 **이전** 5초도 포함하려면:
- 항상 최근 5초를 `deque`에 유지
- 탐지 시 → 5초(기존) + 10초(추가) = 15초 저장

### 4.3 영상 전송 경로

| 방식 | 설명 |
|------|------|
| **A. 로컬 저장** | Pi에 `/var/cat_clips/cat_xxx.mp4` 저장, URL은 Nginx 등으로 제공 |
| **B. Spring 업로드** | HTTP POST `/api/robot/cat-video` → Spring이 저장 후 URL 반환 |
| **C. S3/객체스토리지** | Pi에서 직접 업로드 → presigned URL 등 사용 |

## 5. 구현 단계 제안

### Phase 1: 탐지 + 상태 분석 (MQTT)

1. **CatDetection 프로세스**
   - `infer_video_cat_pose_action_emotion.py` 기반
   - `cv2.VideoCapture(0)` → 실시간 캡처 (파일 대신)
   - YOLO + Swin 추론 로직 재사용
   - 결과를 MQTT `/sub/robot/cat_state` 발행

2. **모델 경로 설정**
   - `--ckpt`: `swin_tiny_best/best.pt` 또는 `swin_tiny_best/best` (폴더인 경우)
   - `--yolo_pose`: `yolo_pose-gy8961dp9tbgbxw9xekwdg9ome.pt`

3. **실행**
   ```bash
   python scripts/cat_detection_service.py \
     --ckpt /path/to/swin_tiny_best/best.pt \
     --yolo_pose /path/to/yolo_pose-gy8961dp9tbgbxw9xekwdg9ome.pt \
     --camera 0 \
     --mqtt_host i14c203.p.ssafy.io
   ```

### Phase 2: 15초 영상 녹화

1. 탐지 시 `deque` 버퍼로 15초 분량 수집
2. `cv2.VideoWriter`로 mp4 저장
3. 저장 경로를 MQTT 페이로드의 `video_path` 또는 `video_clip_url`에 포함

### Phase 3: 웹 전송

1. Spring에서 `/sub/robot/cat_state` 구독
2. 대시보드에 고양이 상태(액션/감정) 표시
3. 15초 영상은 URL로 링크 또는 플레이어에 임베드

```bash

./scripts/run_cat_detection.sh

python scripts/cat_detection_service.py \
  --ckpt /path/to/swin_tiny_best/best.pt \
```

### 9.4 Spring 연동 (갤러리 저장)

- `BE_SERVER_URL` 설정 시 `POST /api/videos` 호출
- **Pi Gateway가 /cat_clips 정적 제공**: 영상은 Pi에 저장, Pi Gateway(8000)가 `/cat_clips/xxx.mp4`로 서빙
- **URL 설정**: `PI_GATEWAY_PUBLIC_URL=http://<Pi-IP>:8000` (갤러리에서 영상 재생 시 이 주소로 접근)
- 웹에서 Pi에 접근하려면 같은 네트워크 또는 포트포워딩 필요
