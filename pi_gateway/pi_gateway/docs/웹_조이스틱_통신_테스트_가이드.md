# 웹 조이스틱 통신 테스트 가이드

FE 웹에서 조이스틱을 눌러서 Pi Gateway와 통신이 되는지 테스트하는 방법입니다.

---

## 🔗 통신 경로

```
[FE 웹] 
  ↓ STOMP/WebSocket
  wss://i14c203.p.ssafy.io/ws
  ↓
[BE 서버]
  ↓ MQTT
  /pub/robot/control
  ↓
[MQTT 브릿지]
  ↓ WebSocket
  ws://라즈베리파이IP:8000/ws/teleop
  ↓
[Pi Gateway]
  ↓ UART (실기기 연결 시)
  [STM32]
```

---

## 📋 테스트 준비

### 1단계: Pi Gateway 실행

```bash
cd /home/ssafy/Downloads/pi_gateway_spec_aligned/pi_gateway

# 데모 모드로 실행 (UART 없이)
python3 src/main.py

# 또는 실기기 연결 시
export DEMO_MODE=0
export ROS_ENABLED=1
export UART_ENABLED=1
export UART_PORT=/dev/serial0
python3 src/main.py
```

**확인:**
- 콘솔에 `[WEB] starting uvicorn on 0.0.0.0:8000` 메시지 확인
- 브라우저에서 `http://localhost:8000/robot/health` 접속 테스트

### 2단계: MQTT 브릿지 실행 (BE와 통신하려면)

**새 터미널에서:**
```bash

export PI_GATEWAY_URL=http://localhost:8000
export MQTT_HOST=localhost  # 또는 인프라 서버 주소
export MQTT_PORT=1883

python3 scripts/mqtt_pi_bridge.py
```

**확인:**
- 콘솔에 `[MQTT] ✅ 연결 성공!` 메시지 확인

### 3단계: BE 서버 실행

```bash
cd /home/ssafy/Downloads/S14P11C203-FE,BE,Infra(2)/S14P11C203-FE,BE,Infra/server
```

**확인:**

### 4단계: FE 웹 실행

## 🎮 조이스틱 테스트 방법

### 방법 1: FE 웹에서 직접 테스트

1. **FE 웹 접속**
   - `http://localhost:5173` 접속
   - 로그인
   - 대시보드에서 "수동 제어" 패널 확인

2. **조이스틱 조작**
   - 조이스틱을 위로 드래그 → 전진 명령
   - 조이스틱을 아래로 드래그 → 후진 명령
   - 조이스틱을 좌우로 드래그 → 회전 명령

3. **로그 확인**

   **FE 브라우저 콘솔 (F12):**
   ```javascript
   // 조이스틱 조작 시 로그 확인
   ```

   **BE 서버 콘솔:**
   ```
   🕹️ 명령 수신: MOVE
   ```

   **MQTT 브릿지 콘솔:**
   ```
   [MQTT] 메시지 수신: /pub/robot/control
   ```

   **Pi Gateway 콘솔:**
   ```
   [UART SEND] aa55010c...  (cmd_vel 프레임 hex 출력)
   ```

### 방법 2: 직접 Pi Gateway WebSocket 테스트

**Python 스크립트로 테스트:**
```bash
cd /home/ssafy/Downloads/pi_gateway_spec_aligned/pi_gateway
python3 scripts/ws_send.py
```

**또는 curl로 HTTP API 테스트:**
```bash
# 모드 전환
curl -X POST http://localhost:8000/robot/mode \
  -H "Content-Type: application/json" \
  -d '{"mode": "teleop"}'

# 비상정지
curl -X POST http://localhost:8000/robot/estop \
  -H "Content-Type: application/json" \
  -d '{"enabled": true}'

# 상태 확인
curl http://localhost:8000/robot/state
```

## ⚠️ 문제 해결

### 조이스틱을 눌렀는데 반응이 없을 때

1. **FE 연결 확인**
   - 브라우저 콘솔에서 STOMP 연결 상태 확인
   - `✅ STOMP(데이터) 연결 성공!` 메시지 확인

2. **BE 서버 확인**
   - BE 서버가 실행 중인지 확인
   - `http://localhost:8080` 접속 가능한지 확인

3. **MQTT 브로커 확인**
   - `docker-compose ps`로 MQTT 브로커 실행 확인
   - MQTT 브릿지가 연결되었는지 확인

4. **Pi Gateway 확인**
   - `http://localhost:8000/robot/health` 접속 가능한지 확인
   - WebSocket 연결 확인

### 직접 Pi Gateway에 연결해서 테스트

BE를 거치지 않고 직접 Pi Gateway WebSocket에 연결:

```javascript
// 브라우저 콘솔에서 실행
const ws = new WebSocket('ws://localhost:8000/ws/teleop');
ws.onopen = () => console.log('✅ 연결 성공!');
ws.onmessage = (e) => console.log('📩 수신:', e.data);

// 조이스틱 명령 전송
ws.send(JSON.stringify({
  type: 'joy',
  joy_x: 0.5,  // 전진
  joy_y: 0.0,
  joy_active: true,
  timestamp: Date.now() / 1000
}));
```

## ✅ 성공 확인

조이스틱을 조작했을 때 다음 로그가 보이면 성공:

1. **FE 콘솔**: 조이스틱 이벤트 전송
2. **BE 콘솔**: `🕹️ 명령 수신: MOVE`
3. **MQTT 브릿지**: 메시지 수신 및 전송
4. **Pi Gateway**: `[UART SEND]` 로그 (실기기 연결 시)

이 모든 로그가 보이면 통신이 정상적으로 작동하는 것입니다!
