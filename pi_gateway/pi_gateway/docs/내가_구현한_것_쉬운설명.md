# 내가 구현한 것 - 쉬운 설명

**목적**: 담당자랑 말할 때, 내가 뭘 했는지 설명할 수 있도록 정리

---

## 1. 내 역할 (한 줄 요약)

**로봇(라즈베리파이) ↔ 웹 대시보드 사이에서 데이터를 옮겨주는 다리 역할**을 했습니다.

---

## 2. 전체 그림 (아주 단순하게)

```
[로봇/STM32]  ←→  [라즈베리파이 (Pi Gateway)]  ←→  [웹 대시보드]
      UART              ↑ 여기를 내가 만듦              HTTP/WebSocket
```

**내가 한 일**: 라즈베리파이에서 돌아가는 Pi Gateway + mqtt_bridge (또는 http_bridge)

---

## 3. 데이터가 어떻게 오가는지

### ① 로봇 → 웹 (상태 전달)

- **보내는 것**: 배터리, 위치(x,y), 수동/자동 모드
- **방법**: Pi Gateway가 상태를 모아서 → MQTT(또는 HTTP)로 → 백엔드에 전달
- **결과**: 웹 대시보드에 "배터리 78%", "현재 위치", "수동 모드" 등이 표시됨

### ② 웹 → 로봇 (수동 조작)

- **보내는 것**: "앞으로 가", "멈춰", "자동 모드로 전환" 같은 명령
- **방법**: 웹에서 버튼/조이스틱 → 백엔드 → MQTT(또는 HTTP) → Pi Gateway → UART → 로봇
- **결과**: 로봇이 움직이거나 멈춤

---

## 4. MQTT / HTTP / WebSocket 뭐가 다른지 (쉽게)

| 이름 | 비유 | 특징 |
|------|------|------|
| **HTTP** | 편지 한 통씩 보내기 | 요청 → 응답 끝. 한 번에 하나씩 |
| **WebSocket** | 전화 통화 | 연결 유지하면서 계속 주고받기 |
| **MQTT** | 게시판에 글 올리기 | 누가 "게시판"에 쓰면, 구독한 사람이 다 받음. 중간에 브로커가 있음 |

**내가 쓰는 것**:
- **MQTT**: Pi → 브로커에 "상태" 올림. 백엔드가 그걸 구독해서 웹에 전달
- **HTTP**: MQTT 대신 쓸 때, Pi가 백엔드 API에 직접 "상태" POST로 보냄

---

## 5. 내가 구현한 것 (구체적으로)

| 파일/역할 | 하는 일 |
|-----------|---------|
| **Pi Gateway** | 로봇(UART), ROS 연결. 웹 요청 받아서 로봇에 전달 |
| **mqtt_pi_bridge** | Pi Gateway 상태를 주기적으로 MQTT 브로커에 올림. MQTT로 들어온 제어 명령을 Pi Gateway에 전달 |

**흐름**:
1. Pi Gateway: `/robot/health`, `/telemetry/latest` API 제공 (상태 조회)
2. mqtt_bridge: 이걸 0.2초마다 읽어서 MQTT `/sub/robot/status`에 발행
3. mqtt_bridge: MQTT `/pub/robot/control` 구독 → 명령 들어오면 Pi Gateway `POST /robot/control`로 전달
4. Pi Gateway: `/robot/control` 받으면 → UART(또는 ROS)로 로봇에 전달

## 6. 담당자한테 말할 때 (멘트 예시)

### 인프라 담당자에게

> "로봇 상태랑 수동 조작은 **MQTT 1883 포트**로 보내고 받아요.  
> 지금 1883이 막혀 있어서 연결이 안 되는데, UFW에서 `sudo ufw allow 1883/tcp` 해주시면 돼요."

### 백엔드 담당자에게 (1883 막혀 있을 때)

> "MQTT 1883 포트가 막혀 있어서 HTTP로 대신 보내고 싶어요.  
> `POST /api/robot/status`랑 `GET /api/robot/control` API 하나씩만 추가해주시면, 제가 그쪽으로 연결하도록 수정할게요.  
> 형식은 `docs/HTTP_로봇연동_백엔드_전달_문서.md`에 적어뒀어요."

### 내가 한 일 요약해서 말할 때

> "로봇(STM32)이랑 웹 대시보드 사이 **데이터 다리** 만든 거예요.  
> 라즈베리파이에서 Pi Gateway 돌리고, 상태는 MQTT(또는 HTTP)로 보내고, 수동 조작 명령은 MQTT(또는 HTTP)로 받아서 로봇에 전달해요."

---

## 7. 정리

- **내 역할**: 로봇 ↔ 웹 사이 데이터 전달
- **방식**: MQTT (기본) 또는 HTTP (1883 막혀 있을 때 대안)
- **담당자 요청**: 1883 포트 열기 **또는** HTTP API 추가
