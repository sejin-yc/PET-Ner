# 젯슨 발행/구독 토픽 정리

**라즈베리파이**와 **젯슨 Orin Nano** 역할을 나누고, 젯슨이 **발행하는 토픽**·**구독해야 할 토픽**을 정리합니다.  
같은 ROS2 도메인( Pi ↔ 젯슨 공유 )에서 동작한다고 가정합니다.

---

## 0. 보드별 역할 (Pi vs 젯슨)

| 보드 | 담당 |
|------|------|
| **라즈베리파이** | Pi Gateway(UART↔STM32), **SLAM**, **Nav2**, **LiDAR**, 엔코더 오도메트리(`/odom`), `cmd_vel` Mux |
| **Jetson Orin Nano** | **로봇팔** 제어, **TTS**(텍스트→음성) |

- **cmd_vel_auto**는 **Pi에서 동작하는 Nav2**가 발행. 젯슨이 발행하지 않음.  
- **/odom**, **telemetry/*** 는 **Pi**가 발행. SLAM·Nav2·LiDAR도 Pi에서 구독.  
- **arm/***, **feed/fill** 은 **젯슨**이 발행. Pi Gateway가 구독 후 UART로 STM32 전달.

---

## 1. 젯슨이 **발행**하는 토픽 (젯슨 → Pi)

Pi Gateway(ArmControlBridge, ChuruControlBridge 등)가 **구독**합니다.

| 토픽 이름 | 메시지 타입 | 용도 | 비고 |
|----------|-------------|------|------|
| **arm/start** | `std_msgs/UInt8` | 로봇팔 동작 (0=정지, 1=변 치우기, 2+=기타) | ArmControlBridge 구독 → UART 전달 |
| **arm/position_correct** | `geometry_msgs/Vector3` | 로봇 위치 보정 (dx, dy, dz) | 아루코 마커 정밀 제어 등 |
| **arm/water** | `std_msgs/UInt8` | 급수 단계 (0=이동/정지, 1=집기, 2=버리기, 3=받기, 4=두기) | 단계별 바퀴 잠금/해제 |
| **feed/fill** | `std_msgs/UInt8` | 츄르 주기 (0=정지, 1=츄르) | ChuruControlBridge 구독 → UART 전달 |

- **cmd_vel_auto**는 Pi의 Nav2가 발행. 젯슨은 발행하지 않음.  
- TTS: 젯슨에서 동작. Pi와 ROS 토픽 연동은 현재 없음. (필요 시 `tts/speak` 등 설계 가능)

---

## 2. 젯슨이 **구독**해야 할 토픽 (Pi → 젯슨)

**라즈베리파이**(Pi Gateway, EncoderOdomNode 등)가 **발행**합니다.

| 토픽 이름 | 메시지 타입 | 용도 | 비고 |
|----------|-------------|------|------|
| **/odom** | `nav_msgs/Odometry` | 오도메트리 (pose, twist) | 로봇팔 제어 시 위치/자세 참고용 (필요 시) |
| **telemetry/encoder** | `std_msgs/String` (JSON) | 엔코더 raw | 필요 시 가공용 |
| **telemetry/imu** | `std_msgs/String` (JSON) | IMU (yaw, pitch, roll, gyro_z 등) | 필요 시 가공용 |
| **telemetry/battery** | `std_msgs/String` (JSON) | 배터리 (vbat_V, 잔량 등) | 모니터링·안전용 |

- SLAM·Nav2·LiDAR는 **Pi**에서 구독. 젯슨은 로봇팔·TTS만 담당하므로 `/odom`·telemetry는 **로봇팔 로직에서 필요할 때만** 구독하면 됨.

---

## 3. 요약

| 구분 | 토픽 | 메시지 타입 |
|------|------|-------------|
| **젯슨 발행** | `arm/start` | `std_msgs/UInt8` |
| | `arm/position_correct` | `geometry_msgs/Vector3` |
| | `arm/water` | `std_msgs/UInt8` |
| | `feed/fill` | `std_msgs/UInt8` |
| **젯슨 구독** | `/odom` | `nav_msgs/Odometry` |
| | `telemetry/encoder` | `std_msgs/String` (JSON) |
| | `telemetry/imu` | `std_msgs/String` (JSON) |
| | `telemetry/battery` | `std_msgs/String` (JSON) |

**Pi 전용 (젯슨 미관여):** `cmd_vel_auto` 발행(Nav2), `/odom`·`telemetry/*` 발행, SLAM·Nav2·LiDAR 구독.

---

## 4. Pi 쪽 Nav2·SLAM 담당자 공유용

- **Nav2**(Pi): `cmd_vel_auto`에 `geometry_msgs/Twist` 발행. 표준 `/cmd_vel`만 쓰면 토픽 리매핑 필요.  
- **SLAM**(Pi): `/odom` 구독. `nav_msgs/Odometry`, frame_id=`odom`, child_frame_id=`base_link`.  
- **LiDAR**(Pi): 스캔 토픽 발행. SLAM·Nav2가 구독.

자세한 건 `SLAM_Nav2_담당자_전달_멘트.md`, `Nav2_연동_방식_설명.md` 참고.
