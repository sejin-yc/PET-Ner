# 순찰 시간 저장 구현 가이드

**목적**: 주간 순찰 시간(주간 순찰 리포트)에 데이터가 표시되도록 Pi → Backend 연동

---

## 1. 구현 방식 (현재)

### 방식 A: control_mode 구독 (구현됨)

- **PatrolLoop**가 `control_mode` 토픽 구독
- `auto` → `teleop` 전환 시 순찰 종료로 판단
- duration = `(종료 시각 - 시작 시각)` → Log로 전송

**흐름**:
```
[control_mode] "auto" 수신 → _patrol_start_ts 기록
[control_mode] "teleop" 수신 → duration = now - _patrol_start_ts
  → job_event_callback("patrol", "success", duration_sec=duration_sec)
  → JOB_EVENTS.add_job → _send_to_be → POST /api/logs
```

---

## 2. 방식 B: cmd_vel 구독 (선택, 고급)

실제 이동 시간만 누적하려면:

1. **PatrolTimeTracker** 노드 추가
2. `cmd_vel_out` 또는 `/cmd_vel` 구독
3. `control_mode`가 `auto`일 때만 `|vx| + |vy| + |wz| > threshold`인 구간만 시간 누적
4. `teleop` 전환 시 누적 시간을 duration으로 사용

**예시 (미구현)**:
```python
# ros_cmdvel.py에 PatrolTimeTracker 클래스
class PatrolTimeTracker(Node):
    def __init__(self, job_event_callback=None):
        self.sub_cmd = self.create_subscription(Twist, "cmd_vel_out", self._on_cmd, 10)
        self.sub_mode = self.create_subscription(String, "control_mode", self._on_mode, 10)
        self._mode = "teleop"
        self._moving_accumulated = 0.0  # 이동 시간 누적(초)
        self._last_ts = None
        self._threshold = 0.01  # 속도 임계값
```

---

## 3. 백엔드 필요 사항

**JobEventCollector**는 `POST {BE_SERVER_URL}/api/logs`로 전송:

```json
{
  "userId": 1,
  "mode": "auto",
  "status": "completed",
  "duration": "5분",
  "durationNum": 5,
  "distance": 0,
  "detectionCount": 0,
  "details": "patrol 작업 - success"
}
```

- **durationNum**: 분 단위 (주간 순찰 시간 차트에 사용)
- Backend에 `POST /api/logs` 엔드포인트가 있어야 함
- Log 엔티티에 `duration`, `durationNum` 저장 필요

---

## 4. Pi Gateway 환경 변수

순찰 로그를 Backend로 보내려면:

```bash
export BE_SERVER_URL=https://i14c203.p.ssafy.io
export BE_USER_ID=1
```

Docker 실행 예:
```bash
docker run ... -e BE_SERVER_URL=https://i14c203.p.ssafy.io -e BE_USER_ID=1 ...
```

---

## 5. 흐름 요약

| 단계 | 담당 | 동작 |
|------|------|------|
| 1 | PatrolLoop | `control_mode` auto→teleop 시 duration 계산 |
| 2 | job_event_callback | JOB_EVENTS.add_job(..., duration_sec=...) 호출 |
| 3 | JobEventCollector | duration_sec → durationNum(분) 변환, POST /api/logs |
| 4 | Backend | Log 저장 → 주간 순찰 시간 집계 |
| 5 | 프론트 | LogsPage에서 주간 순찰 시간 표시 |
