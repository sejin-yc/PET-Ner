## 젯슨 담당자에게 전달할 내용

### 1. 필요한 ROS2 토픽 추가

**구독할 토픽:**
- `feed/request` (`std_msgs/String`): Pi Gateway가 급식 요청 시 발행
  - 메시지 형식: `{"waypoint_id": "feed_zone", "timestamp": 1737700000.123}`

**발행할 토픽:**
- `feed/amount` (`std_msgs/UInt8`): 사료 량 계산 완료 후 발행
  - 값: 1~3 (사료 량 레벨)
  - 계산 방법: `grams / full_grams * 100` → 레벨 변환

### 2. 레벨 변환 로직

```python
# config.yaml의 full_grams 사용 (예: 427.8g)
full_grams = 427.8

# 모델로 사료 량 계산 (그램)
grams = predict_grams(model, tf, roi, device, amp)

# 퍼센트 계산
percent = (grams / full_grams) * 100.0
percent = clamp(percent, 0.0, 100.0)

# 레벨 변환 (1~3)
if percent < 33.0:
    feed_level = 1  # 적음
elif percent < 66.0:
    feed_level = 2  # 보통
else:
    feed_level = 3  # 많음
```

# FeedAINode 클래스 추가 (간단)
class FeedAINode(Node):
    def __init__(self):
        super().__init__("feed_ai_node")
        self.sub_feed_request = self.create_subscription(
            String, "feed/request", self.on_feed_request, 10
        )
        self.pub_feed_amount = self.create_publisher(UInt8, "feed/amount", 10)
    
    def on_feed_request(self, msg: String):
        """Pi Gateway가 급식 요청 시 최신 추론 결과 발행"""
        try:
            import json
            data = json.loads(msg.data)
            waypoint_id = data.get("waypoint_id", "")
            self.get_logger().info(f"Feed requested for waypoint: {waypoint_id}")
            
            # 기존 infer_loop가 계산한 최신 결과 사용
            with FEED_LEVEL_LOCK:
                feed_level = LATEST_FEED_LEVEL if LATEST_FEED_LEVEL is not None else 2
            
            msg = UInt8()
            msg.data = feed_level
            self.pub_feed_amount.publish(msg)
            self.get_logger().info(f"Published feed/amount: {feed_level}")
        except Exception as e:
            self.get_logger().error(f"Failed to process feed request: {e}")

## 통신 흐름

```
1. [Nav2] waypoint 도착 → patrol/waypoint_reached 발행
   {"waypoint_id": "feed_zone", "waypoint_type": "feed"}

2. [Pi Gateway] 신호 받음 → 아루코 정렬 대기 (필요한 경우)

3. [Nav2] 아루코 정렬 완료 → patrol/aruco_aligned 발행

4. [Pi Gateway] 급식 요청 → feed/request 발행
   {"waypoint_id": "feed_zone", "timestamp": 1737700000.123}

5. [젯슨] feed/request 구독 → 이미지 받아서 추론
   - YOLO로 그릇 세그멘테이션
   - Regression 모델로 사료 량 계산 (그램)
   - 퍼센트 계산 → 레벨 변환 (1~3)

6. [젯슨] feed/amount 발행
   feed_level = 2 (예시)

7. [Pi Gateway] feed/amount 구독 → UART로 STM32 전송
   make_feed_frame(feed_level=2)

8. [STM32] 서보모터 제어 → 완료 신호 (STATUS)

9. [Pi Gateway] patrol/action_complete 발행
   {"action_type": "feed", "waypoint_id": "feed_zone", "status": "success"}

10. [Nav2] 다음 waypoint로 이동
```
