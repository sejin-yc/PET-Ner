# 수동조작 코드 확인 가이드

## 개요

ROS2 폴더에 수동조작 코드가 있는지 확인하고, Pi Gateway와의 관계를 정리합니다.

---

## ROS2 폴더 확인 결과

### ✅ 확인된 파일들

1. **`twist_mux.yaml`** (설정 파일)
   - `/cmd_vel_joy` 토픽을 구독 (priority 50)
   - 수동조작 입력으로 설정됨

2. **`gateway_sj.cpp`**
   - `/cmd_vel_out` 토픽을 구독
   - UART로 전송하는 브릿지 (C++ 버전)

3. **`twist_to_stm_uart_bridge.py`**
   - `/cmd_vel` 토픽을 구독
   - UART로 전송하는 브릿지 (Python 버전)

### ❓ 확인되지 않은 것

**`/cmd_vel_joy`를 발행하는 노드**:
- ROS2 폴더에서 명시적으로 `/cmd_vel_joy`를 발행하는 노드를 찾지 못함
- 조이스틱이나 키보드 입력을 받는 노드가 있을 수 있음
- 또는 다른 패키지/노드에서 발행할 수 있음

---

## 가능한 시나리오

### 시나리오 1: ROS2 폴더에 수동조작 노드가 있음

```
조이스틱/키보드 입력
    ↓
ROS2 폴더 수동조작 노드
    └─ /cmd_vel_joy 발행
        ↓
twist_mux
    └─ /cmd_vel 발행 (통합)
        ↓
twist_to_stm_uart_bridge
    └─ UART 전송
```

**이 경우**:
- Pi Gateway의 `/cmd_vel_joy` 발행과 충돌 가능
- 두 개가 동시에 발행하면 마지막 메시지가 적용됨

### 시나리오 2: ROS2 폴더에 수동조작 노드가 없음

```
웹 대시보드
    ↓
Pi Gateway
    └─ /cmd_vel_joy 발행
        ↓
twist_mux
    └─ /cmd_vel 발행 (통합)
        ↓
twist_to_stm_uart_bridge
    └─ UART 전송
```

**이 경우**:
- Pi Gateway만 `/cmd_vel_joy` 발행
- 충돌 없음

---

## Pi Gateway의 역할

### 현재 구조

**Pi Gateway CmdVelMux**:
- 웹 대시보드 명령 수신 (`/cmd_vel_teleop`)
- 패트롤 명령 구독 (`/cmd_vel_nav`)
- 통합하여 `/cmd_vel_joy` 발행

**의미**:
- Pi Gateway가 웹 대시보드를 통한 수동조작을 담당
- ROS2 폴더에 수동조작 노드가 있어도, Pi Gateway의 `/cmd_vel_joy`가 우선순위 50으로 동작

---

## 우선순위 (twist_mux.yaml)

```
아루코 마커 (priority 80) > 웹 대시보드 (priority 50) > 패트롤 (priority 10)
```

**의미**:
- 아루코 마커 제어 중: 다른 모든 명령 무시
- 웹 대시보드 수동조작: 패트롤 명령 무시
- 패트롤: 다른 명령이 없을 때만 동작

---

## 권장 사항

### 1. ROS2 폴더에 수동조작 노드가 있는 경우

**옵션 A: Pi Gateway만 사용 (권장)**
- ROS2 폴더의 수동조작 노드는 실행하지 않음
- Pi Gateway의 웹 대시보드만 사용

**옵션 B: 둘 다 사용**
- ROS2 폴더: 조이스틱/키보드 입력
- Pi Gateway: 웹 대시보드 입력
- 둘 다 `/cmd_vel_joy` 발행 → 마지막 메시지가 적용됨
- ⚠️ 충돌 가능성 있음

### 2. ROS2 폴더에 수동조작 노드가 없는 경우

**현재 구조 유지**:
- Pi Gateway가 `/cmd_vel_joy` 발행
- twist_mux가 통합
- 문제 없음 ✅

## 확인 방법

### ROS2 토픽 확인

```bash
# /cmd_vel_joy를 발행하는 노드 확인
ros2 topic info /cmd_vel_joy

# /cmd_vel_joy 메시지 확인
ros2 topic echo /cmd_vel_joy

# 발행자 확인
ros2 topic hz /cmd_vel_joy
```

### ROS2 노드 확인

```bash
# 실행 중인 노드 확인
ros2 node list

# 특정 노드의 토픽 확인
ros2 node info <node_name>
```

## 결론

### 현재 상태

1. **`twist_mux.yaml`**: `/cmd_vel_joy` 구독 설정됨
2. **Pi Gateway**: `/cmd_vel_joy` 발행 (웹 대시보드)
3. **ROS2 폴더**: `/cmd_vel_joy` 발행 노드 확인 안 됨

### 권장 사항

- **Pi Gateway의 `/cmd_vel_joy` 발행 유지**
- ROS2 폴더에 수동조작 노드가 있다면, 실행하지 않거나 다른 토픽명 사용 권장
- 또는 Pi Gateway와 ROS2 폴더의 수동조작을 다른 우선순위로 분리
