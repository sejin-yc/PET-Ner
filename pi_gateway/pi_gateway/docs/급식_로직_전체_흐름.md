# 급식 로직 전체 흐름

Nav2 → Pi Gateway → 젯슨 → Pi Gateway → STM32까지의 전체 급식 로직입니다.

---

## 🔄 전체 통신 흐름

```
[Nav2] waypoint 도착
    ↓ patrol/waypoint_reached 발행
[Pi Gateway - PatrolLoop] 신호 받음
    ↓ 아루코 정렬 대기 (필요한 경우)
[Nav2] 아루코 정렬 완료
    ↓ patrol/aruco_aligned 발행
[Pi Gateway - PatrolLoop] 신호 받음
    ↓ execute_action("feed", waypoint_id) 호출
[Pi Gateway - PatrolActionScheduler] 급식 요청
    ↓ feed/request 발행
[젯슨] feed/request 구독
    ↓ FEED_AI 모델로 사료 량 계산
[젯슨] feed/amount 발행 (1~3)
    ↓
[Pi Gateway - FeedControlBridge] feed/amount 구독
    ↓ UART 프레임 생성
[STM32] UART 수신
    ↓ 서보모터 제어
[STM32] 완료 신호 (STATUS)
    ↓
[Pi Gateway] patrol/action_complete 발행
    ↓
[Nav2] 다음 waypoint로 이동
```

---

```json

  "waypoint_id": "feed_zone",

  "timestamp": 1737700000.123
}
```

**발행 위치:** Nav2 담당자 코드

### 2단계: Pi Gateway - PatrolLoop가 신호 받음

**파일:** `src/ros_cmdvel.py`  
**메서드:** `PatrolLoop.on_waypoint_reached()`

```python
def on_waypoint_reached(self, msg: String):
    data = json.loads(msg.data)
    waypoint_id = data.get("waypoint_id", "")
    waypoint_type = data.get("waypoint_type", "")
    
    # 급식도 아루코 마커 필요할 수 있음
    if waypoint_type in ["litter_clean", "water", "feed"]:
        # 아루코 정렬 대기
        self.action_scheduler._waiting_for_aruco = True
        self.action_scheduler._pending_action = waypoint_type
        self.action_scheduler._pending_waypoint_id = waypoint_id
```

**동작:**
- `waypoint_type == "feed"` 확인
- 아루코 정렬 대기 상태로 설정

### 4단계: Pi Gateway - PatrolLoop가 아루코 정렬 신호 받음

**메서드:** `PatrolLoop.on_aruco_aligned()`

```python
def on_aruco_aligned(self, msg: String):
    data = json.loads(msg.data)
    waypoint_id = data.get("waypoint_id", "")
    waypoint_type = data.get("waypoint_type", "")
    
    # 대기 중인 액션 실행
    if self.action_scheduler._waiting_for_aruco:
        self.action_scheduler.execute_action(
            waypoint_type, 
            waypoint_id,
            action_complete_callback=self._publish_action_complete
        )
```

**동작:**
- 아루코 정렬 완료 확인
- `execute_action("feed", waypoint_id)` 호출

### 6단계: Pi Gateway - FeedControlBridge가 젯슨에 요청

**메서드:** `FeedControlBridge.request_feed()`

```python
def request_feed(self, waypoint_id: str):
    msg = String()
    msg.data = json.dumps({
        "waypoint_id": waypoint_id,
        "timestamp": time.time()
    })
    self.pub_request.publish(msg)  # feed/request 토픽 발행
```

**토픽:** `feed/request` (`std_msgs/String`)  
**메시지:** `{"waypoint_id": "feed_zone", "timestamp": 1737700000.123}`

### 7단계: 젯슨이 feed/request 구독 및 사료 량 계산

**젯슨 담당자가 구현해야 할 코드:**

```python
def on_feed_request(self, msg: String):
    """Pi Gateway가 급식 요청 시 호출"""
    # 1. 카메라 이미지 받기
    frame = get_latest_image()  # ROS2 토픽: /camera/image_raw
    
    # 2. YOLO 세그멘테이션으로 그릇 찾기
    result = yolo.predict(source=frame, imgsz=640, conf=0.60)[0]
    idx, reason = choose_best_instance_by_mask_area(result)
    
    if idx is None:
        feed_level = 2  # 기본값
    else:
        # 3. ROI 추출
        roi, bbox = get_roi_from_seg_result(result, frame, idx)
        
        # 4. Regression 모델로 사료 량 계산 (그램)
        grams = predict_grams(reg_model, reg_tf, roi, device="cuda")
        
        # 5. 퍼센트 계산
        percent = (grams / 427.8) * 100.0  # full_grams = 427.8
        percent = clamp(percent, 0.0, 100.0)
        
        # 6. 레벨 변환 (1~3)
        if percent < 33.0:
            feed_level = 1  # 적음
        elif percent < 66.0:
            feed_level = 2  # 보통
        else:
            feed_level = 3  # 많음
    
    # 7. feed/amount 토픽 발행
    msg = UInt8()
    msg.data = feed_level
    self.pub_feed_amount.publish(msg)
```

**사용 모델:**
- `bowl_seg_best.pt` (YOLO 세그멘테이션)
- `ckpt_feedg_roi_fromtxt__effv2s__224__e80__bs32__lr3e-4__seed42.pt` (Regression)

### 8단계: Pi Gateway - FeedControlBridge가 사료 량 받음

**메서드:** `FeedControlBridge.on_feed_amount()`

```python
def on_feed_amount(self, msg: UInt8):
    """젯슨이 계산한 사료 량 수신 → UART로 STM32에 전송"""
    feed_level = int(msg.data)  # 1~3
    
    if feed_level < 1 or feed_level > 3:
        feed_level = 2  # 기본값
    
    # UART 프레임 생성 및 전송
    frame = make_feed_frame(feed_level)
    self.uart.send(frame)
    
    # 액션 완료 콜백 호출
    if self.action_complete_callback:
        waypoint_id = self.action_scheduler._pending_feed_waypoint_id
        self.action_complete_callback("feed", waypoint_id, "success")
        self.action_scheduler._waiting_for_feed_amount = False
```

**동작:**
- `feed_level` (1~3) 받음
- UART 프레임 생성 (`make_feed_frame(feed_level)`)
- STM32에 전송

---

### 9단계: STM32가 UART 수신 및 서보모터 제어

**UART 프레임:**
```
STX: 0xAA 0x55
MSG_ID: 0x84 (ID_FEED)
LENGTH: 0x01
PAYLOAD: feed_level (1~3)
CHECKSUM: XOR
```

**STM32 동작:**
- UART 프레임 파싱
- 서보모터로 급식통 열기/닫기 제어
- 완료되면 STATUS 메시지 전송

### 10단계: Pi Gateway가 완료 신호 발행

**메서드:** `PatrolLoop._publish_action_complete()`

```python
def _publish_action_complete(self, action_type: str, waypoint_id: str, status: str):
    msg = String()
    msg.data = json.dumps({
        "action_type": action_type,
        "waypoint_id": waypoint_id,
        "status": status,
        "timestamp": time.time()
    })
    self.pub_action_complete.publish(msg)  # patrol/action_complete 발행
```

**토픽:** `patrol/action_complete` (`std_msgs/String`)  
**메시지:** `{"action_type": "feed", "waypoint_id": "feed_zone", "status": "success", "timestamp": 1737700000.789}`

## 🔑 핵심 포인트

### 1. 아루코 마커 지원
- 급식도 아루코 마커가 필요할 수 있음
- `waypoint_type in ["litter_clean", "water", "feed"]` → 아루코 정렬 대기

### 2. 젯슨 연동
- Pi Gateway는 계산하지 않음
- 젯슨이 FEED_AI 모델로 사료 량 계산
- `feed/request` → `feed/amount` 통신

### 3. 레벨 변환
- 젯슨: 그램 → 퍼센트 → 레벨 (1~3)
- Pi Gateway: 레벨 그대로 UART 전송

### 4. 완료 신호
- STM32가 서보모터 제어 완료
- Pi Gateway가 `patrol/action_complete` 발행
- Nav2가 다음 waypoint로 이동
