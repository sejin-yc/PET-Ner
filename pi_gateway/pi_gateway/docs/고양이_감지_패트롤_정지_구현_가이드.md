패트롤 중 USB 웹캠으로 고양이를 인식하여 로봇을 정지시키는 기능 구현 가이드입니다.

- 고양이가 사라지면 패트롤 재개

## 🔍 모델 분석

### catstate 모델의 역할

**MultiHeadClassifier**:
- **action**: 고양이의 행동 분류 (걷기, 그루밍, 꼬리 흔들기 등)
- **emotion**: 고양이의 감정 분류 (공격성, 공포, 불안, 편안, 행복, 화남 등)

**중요**: 이 모델은 **고양이가 이미 프레임에 있을 때** 액션/감정을 분류합니다.  
고양이 **존재 여부**를 판단하려면 추가 로직이 필요합니다.

---

## 💡 구현 방법

### 방법 1: Confidence 기반 고양이 감지 (추천)

**아이디어**: catstate 모델의 confidence가 낮으면 고양이가 없는 것으로 판단

```python
# 추론 결과
action_label, action_conf, action_probs = model_infer(frame)
emotion_label, emotion_conf, emotion_probs = model_infer(frame)

# 고양이 존재 판단
cat_detected = (action_conf > 0.5) or (emotion_conf > 0.5)
```

**장점**:
- 추가 모델 불필요
- 기존 catstate 모델 재사용

**단점**:
- Confidence 임계값 튜닝 필요
- 배경에 고양이 같은 패턴이 있으면 오탐 가능

---

### 방법 2: YOLO Detection + catstate 분류 (더 정확)

**아이디어**: YOLO로 고양이를 먼저 감지한 후, 감지된 영역에 대해 catstate 모델 적용

```python
# 1. YOLO로 고양이 감지
yolo_results = yolo_model(frame)
cat_bboxes = [r for r in yolo_results if r.class == "cat"]

if len(cat_bboxes) > 0:
    # 2. 고양이가 감지되면 catstate 모델로 액션/감정 분류
    for bbox in cat_bboxes:
        cropped = crop_image(frame, bbox)
        action, emotion = catstate_model(cropped)
```

**장점**:
- 고양이 존재 여부 판단이 정확함
- catstate 모델은 감지된 고양이 영역에만 적용 (효율적)

**단점**:
- YOLO 모델 추가 필요 (또는 cat_pose_yolo 사용)

## 🛠️ 구현 구조

### ROS2 노드: `CatDetectionNode`

**역할**:
1. USB 웹캠에서 이미지 캡처 (또는 ROS2 이미지 토픽 구독)
2. catstate 모델로 추론
3. 고양이 감지 시 `cmd_vel`을 0으로 설정하는 토픽 발행
4. 고양이가 사라지면 패트롤 재개

**토픽**:
- 구독: `/camera/image_raw` (또는 웹캠 직접 캡처)
- 발행: `cat_detection/state` (`std_msgs/Bool` - 고양이 감지 여부)
- 발행: `/cmd_vel` (고양이 감지 시 `Twist(0,0,0)` 발행)

### 2. CmdVelMux 수정 (고양이 감지 우선순위)

**현재 구조**:
```
cmd_vel_teleop / /cmd_vel → CmdVelMux → cmd_vel_out
```

**수정 후**:
```
cmd_vel_teleop / /cmd_vel / cat_detection/cmd_vel_stop → CmdVelMux → cmd_vel_out
```

**CmdVelMux에 고양이 감지 우선순위 추가**:

```python

        """고양이 감지 시 정지 명령"""
```

## ⚠️ 주의사항

### 1. 성능 최적화

**문제**: catstate 모델 추론이 느릴 수 있음

**해결책**:
- 프레임 샘플링 (예: 2 FPS로 추론)
- 모델 경량화 (TensorRT 변환 등)
- GPU 사용 (Jetson의 경우)

### 2. False Positive 방지

**문제**: 배경 패턴이 고양이로 오인식될 수 있음

**해결책**:
- Confidence 임계값 튜닝
- 연속 프레임에서 일관성 확인 (예: 3프레임 연속 감지 시만 정지)
- YOLO Detection 추가 (방법 2)

### 3. 패트롤 재개 조건

**문제**: 고양이가 사라졌을 때 언제 패트롤을 재개할지

**해결책**:
- 일정 시간(예: 5초) 동안 고양이가 감지되지 않으면 재개
- 또는 Nav2가 자동으로 재개하도록 설정
