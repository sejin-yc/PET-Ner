## 🎯 로봇팔 제어 개요

### 역할 구분

- **젯슨**: 로봇팔 동작 로직 제어 (순서, 타이밍, 각도 등)
- **Pi Gateway**: 젯슨의 명령을 STM32로 전달하는 브릿지 + 바퀴 잠금
- **STM32**: 실제 하드웨어 제어 (서보 모터, 바퀴 잠금)

### Pi Gateway의 역할

1. **브릿지**: 젯슨의 ROS2 토픽을 UART 프레임으로 변환하여 STM32로 전송
2. **안전 제어**: 로봇팔 동작 중 바퀴 모터 잠금

## 📡 UART 프레임 정의

**파일**: `src/uart_frames.py`

### 1. ID_ARM_START (0x06) - 로봇팔 동작 시작/정지

```python
ID_ARM_START = 0x06

def make_arm_start_frame(action_id: int) -> bytes:
    """
    로봇팔 동작 시작/정지 프레임 생성.
    
    Args:
        action_id: 
            - 0 = 로봇팔 정지 (변 치우기 완료 또는 중지, 바퀴 해제)
            - 1 = 변 치우기 시작 (바퀴 잠금)
            - 2+ = 기타 로봇팔 동작 시작 (바퀴 잠금)
    """
    payload = bytes([action_id & 0xFF])
    return make_frame(ID_ARM_START, payload)
```

**사용 예시**:
- 변 치우기 시작: `action_id = 1` → 로봇팔이 변 치우기 작업 시작 (강화학습 모델이 자율로 처리)
- 변 치우기 완료/정지: `action_id = 0` → 로봇팔 정지, 바퀴 해제

**동작 흐름**:
1. `action_id = 1`: 변 치우기 시작 신호 → 로봇팔이 변 치우기 작업 수행
2. 작업 중: 젯슨의 강화학습 모델이 로봇팔 제어 (화장실 4구역 자동 처리)
3. `action_id = 0`: 작업 완료 신호 → 로봇팔 정지

---

### 2. ID_ARM_POSITION_CORRECT (0x07) - 로봇 위치 보정

```python
ID_ARM_POSITION_CORRECT = 0x07

def make_arm_position_correct_frame(dx: float, dy: float, dz: float) -> bytes:
    """
    로봇 위치 보정 프레임 생성.
    
    Args:
        dx: x축 이동 거리 (m), 양수=앞, 음수=뒤
        dy: y축 이동 거리 (m), 양수=왼, 음수=오
        dz: z축 이동 거리 (m), 양수=위, 음수=아래
    """
    payload = struct.pack("<fff", float(dx), float(dy), float(dz))
    return make_frame(ID_ARM_POSITION_CORRECT, payload)
```

**사용 예시**:
- 로봇 위치가 잘못되어 팔 동작이 제대로 안 될 때
- 젯슨이 위치 보정 명령 발행 → Pi Gateway가 STM32로 전달

---

### 3. ID_ARM_WATER (0x09) - 급수 동작

```python
ID_ARM_WATER = 0x09

def make_arm_water_frame(water_action: int) -> bytes:
    """
    로봇팔 급수 프레임 생성.
    
    Args:
        water_action:
            - 0 = 정지/완료 (바퀴 해제, 이동 가능)
            - 1 = 물그릇 집기 (바퀴 잠금)
            - 2 = 물 버리기 (바퀴 잠금)
            - 3 = 물 받기 (바퀴 잠금)
            - 4 = 물그릇 두기 (바퀴 잠금)
    """
    payload = bytes([water_action & 0xFF])
    return make_frame(ID_ARM_WATER, payload)
```

**급수 동작**:
- 강화학습 로봇팔이 자율로 물 뜨고 버리는 전체 과정을 수행합니다.
- 물그릇 위치로 이동 → 물그릇 집기 → 화장실 위치로 이동 → 물 버리기
→ 서스펜서 위치로 이동 → 물 받기 → 물그릇 위치로 이동 → 물그릇 두기

**제어 신호**:
- `water_action = 1`: 급수 시작 (바퀴 잠금, 강화학습 로봇팔이 자율로 수행)
- `water_action = 0`: 급수 완료/정지 (바퀴 해제)

## 🔄 ROS2 브릿지

**파일**: `src/ros_cmdvel.py`

### ArmControlBridge 클래스

젯슨이 발행한 ROS2 토픽을 구독하여 UART로 STM32에 전송합니다.

#### 구독 토픽

1. **`arm/start`** (std_msgs/UInt8)
   - 로봇팔 동작 시작/정지
   - `action_id = 0`: 정지 (바퀴 해제)
   - `action_id != 0`: 시작 (바퀴 잠금)

2. **`arm/position_correct`** (geometry_msgs/Vector3)
   - 로봇 위치 보정
   - `dx, dy, dz`: 이동 거리 (m)

3. **`arm/water`** (std_msgs/UInt8)
   - 급수 제어 (강화학습 로봇팔 자율 수행)
   - `water_action = 0`: 완료/정지 (바퀴 해제)
   - `water_action = 1`: 급수 시작 (바퀴 잠금, 강화학습 로봇팔이 자율로 수행)

#### 주요 메서드

```python
class ArmControlBridge(Node):
    def __init__(self, uart: UartLink, cmdvel_bridge=None):
        # ...
        self.arm_active = False  # arm/start 활성화 상태
        self.arm_water_active = False  # arm/water 활성화 상태
    
    def on_start(self, msg: UInt8):
        """arm/start 토픽 수신 → UART 전송 + 바퀴 잠금"""
        action_id = int(msg.data)
        frame = make_arm_start_frame(action_id)
        self.uart.send(frame)
        
        # 바퀴 잠금 처리
        if action_id != 0:
            self.cmdvel_bridge.set_arm_start_active(True)
        else:
            self.cmdvel_bridge.set_arm_start_active(False)
    
    def on_water(self, msg: UInt8):
        """arm/water 토픽 수신 → UART 전송 + 바퀴 잠금"""
        water_action = int(msg.data)
        frame = make_arm_water_frame(water_action)
        self.uart.send(frame)
        
        # 바퀴 잠금 처리 (action_id != 0일 때만 잠금)
        if water_action != 0:
            self.cmdvel_bridge.set_arm_water_active(True)
        else:
            self.cmdvel_bridge.set_arm_water_active(False)
```

## 🔒 바퀴 잠금 로직

**파일**: `src/ros_cmdvel.py`

### RosCmdVelBridge 클래스

로봇팔 동작 중에는 바퀴 모터를 잠가야 합니다.

#### 바퀴 잠금 상태 관리

```python
class RosCmdVelBridge(Node):
    def __init__(self, ...):
        self.arm_start_active = False  # arm/start 활성화 상태
        self.arm_water_active = False  # arm/water 활성화 상태
    
    @property
    def arm_active(self) -> bool:
        """로봇팔 활성화 상태 (둘 중 하나라도 활성화되면 True)"""
        return self.arm_start_active or self.arm_water_active
    
    def tick(self):
        # ...
        # 로봇팔 동작 중에는 바퀴 모터를 잠가야 함 (cmd_vel을 0으로 전송)
        if self.arm_active:
            self.uart.send(make_cmd_vel_frame(0.0, 0.0, 0.0))
            return
        # ...
```

#### 바퀴 잠금 조건

| 상황 | 바퀴 상태 |
|------|---------|
| `arm/start` (action_id != 0) | 잠금 |
| `arm/start` (action_id = 0) | 해제 |
| `arm/water` (water_action = 1) | 잠금 |
| `arm/water` (water_action = 0) | 해제 |

**중요**: `arm/start`와 `arm/water` 중 하나라도 활성화되면 바퀴 잠금

## 🔄 동작 흐름

### 1. 변 치우기 (arm/start) - 순찰 중 자동 실행

**중요**: 변 치우기는 **순찰 중에만** 자동으로 실행됩니다!

#### 순찰 중 자동 실행 흐름

```
1. 순찰 모드 시작 (mode = "auto")
   → PatrolLoop.tick() 호출
   → mode == "auto"일 때만 action_scheduler.tick() 실행

2. 순찰 시작 시점 체크
   → 순찰 모드로 전환될 때 (mode = "auto") 체크
   → 저녁 시간대(예: 18:00 이후)인지 확인
   → 오늘 이미 실행했는지 확인 (날짜 기반)
   → 조건 만족 시 변 치우기 시작

3. 변 치우기 시작
   PatrolActionScheduler
     ↓ make_arm_start_frame(1) 전송
   STM32
     ↓ ID_ARM_START (action_id=1) 수신 → 바퀴 모터 잠금
     ↓ 서보 모터 제어 (변 치우기)
   로봇팔 강화학습 모델
     ↓ 화장실 4구역 자동 처리

4. 변 치우기 완료
   젯슨
     ↓ ROS2 토픽 발행: arm/start (action_id=0)
   ArmControlBridge
     ↓ UART 프레임 생성: ID_ARM_START (action_id=0)
     ↓ 바퀴 해제 신호: cmdvel_bridge.set_arm_start_active(False)
   RosCmdVelBridge
     ↓ 정상적으로 cmd_vel 전송 재개
   STM32
     ↓ ID_ARM_START (action_id=0) 수신 → 바퀴 모터 해제
   → 순찰 계속 진행
```

#### 수동 실행 흐름 (웹 API 또는 젯슨)

순찰 중이 아니어도 수동으로 실행 가능:

```
젯슨 또는 웹 API
  ↓ ROS2 토픽 발행 또는 POST /arm/start: arm/start (action_id=1)
ArmControlBridge
  ↓ UART 프레임 생성: ID_ARM_START (action_id=1)
  ↓ 바퀴 잠금 신호: cmdvel_bridge.set_arm_start_active(True)
RosCmdVelBridge
  ↓ cmd_vel을 0으로 전송 (바퀴 제어 차단)
STM32
  ↓ ID_ARM_START 수신 → 바퀴 모터 잠금
  ↓ 서보 모터 제어 (변 치우기)
```

### 2. 급수 동작 (arm/water) - 강화학습 로봇팔 자율 수행

급수 동작은 강화학습 로봇팔이 자율로 물 뜨고 버리는 전체 과정을 수행합니다.

#### 급수 시작

```
젯슨
  ↓ ROS2 토픽 발행: arm/water (water_action=1)
ArmControlBridge
  ↓ UART 프레임 생성: ID_ARM_WATER (water_action=1)
  ↓ 바퀴 잠금 신호: cmdvel_bridge.set_arm_water_active(True)
RosCmdVelBridge
  ↓ cmd_vel을 0으로 전송 (바퀴 제어 차단)
STM32
  ↓ ID_ARM_WATER (water_action=1) 수신 → 바퀴 모터 잠금
로봇팔 강화학습 모델
  ↓ 물그릇 위치로 이동 → 물그릇 집기 → 화장실 위치로 이동 → 물 버리기
  → 서스펜서 위치로 이동 → 물 받기 → 물그릇 위치로 이동 → 물그릇 두기
  (전체 과정을 자율로 수행)
```

#### 급수 완료

```
젯슨
  ↓ ROS2 토픽 발행: arm/water (water_action=0)
ArmControlBridge
  ↓ UART 프레임 생성: ID_ARM_WATER (water_action=0)
  ↓ 바퀴 해제 신호: cmdvel_bridge.set_arm_water_active(False)
RosCmdVelBridge
  ↓ 정상적으로 cmd_vel 전송 재개
STM32
  ↓ ID_ARM_WATER (water_action=0) 수신 → 바퀴 모터 해제
  ↓ 로봇 이동 가능
```

## 🔑 핵심 포인트

### 1. 역할 구분
- **젯슨**: 로봇팔 동작 로직 제어 (순서, 타이밍, 각도)
- **Pi Gateway**: 브릿지 + 바퀴 잠금
- **STM32**: 하드웨어 제어

### 2. 바퀴 잠금 규칙
- 로봇팔 동작 중 (`arm/start` 또는 `arm/water` 활성화) → 바퀴 잠금
- 로봇팔 정지/완료 (`action_id = 0`) → 바퀴 해제

### 3. 급수 동작 특수성
- 각 단계(집기, 버리기, 받기, 두기)에서만 바퀴 잠금
- 이동 중(`action_id = 0`)에는 바퀴 해제하여 이동 가능

### 4. 이중 잠금 메커니즘
- **Pi Gateway**: cmd_vel을 0으로 전송하여 바퀴 제어 차단
- **STM32**: UART 프레임 수신하여 하드웨어 레벨 바퀴 잠금

## ✅ 요약

1. **UART 프레임**: `ID_ARM_START`, `ID_ARM_WATER`, `ID_ARM_POSITION_CORRECT`
2. **ROS2 브릿지**: `ArmControlBridge`가 젯슨의 토픽을 구독하여 UART로 전송
3. **바퀴 잠금**: 로봇팔 동작 중 자동으로 바퀴 잠금, 정지 시 해제
4. **급수 동작**: 강화학습 로봇팔이 자율로 수행 (시작 시 바퀴 잠금, 완료 시 해제)
5. **웹 API**: 웹 UI에서도 로봇팔 제어 가능

**모든 로봇팔 제어는 젯슨이 담당하며, Pi Gateway는 브릿지 역할만 수행합니다!** 🔄
